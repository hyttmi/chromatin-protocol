<objective>
Add StorageFull wire message (TransportMsgType = 23) sent when a Data message blob is rejected for capacity, and peer_is_full flag on PeerInfo that suppresses outbound sync blob pushes to full peers. Sync ingest skips blobs silently on capacity rejection and signals StorageFull after sync completes.

Purpose: Peers learn about a node's storage exhaustion and stop sending it blobs they know will be rejected, reducing wasted bandwidth and processing.
Output: Wire protocol extension, peer flag, sync push suppression, E2E tests.
</objective>

    **db/peer/peer_manager.cpp:**
    6. **Data message handler:** In `on_peer_message()` case for `TransportMsgType_Data`, after `engine_.ingest()` returns a rejected result:
       - If `result.error == IngestError::storage_full`, send a StorageFull message (empty payload) on the connection:
         ```cpp
         if (result.error && *result.error == engine::IngestError::storage_full) {
             // Send StorageFull to inform peer
             auto msg = wire::encode_transport(wire::TransportMsgType_StorageFull, {});
             conn->send(msg);
             spdlog::warn("Storage full, notifying peer {}", peer_display_name(conn));
         }
         ```
         Use `co_spawn` if `send()` is async, or direct call if synchronous. Follow the existing pattern for sending responses in the Data handler.

    7. **StorageFull receive handler:** In `on_peer_message()`, add a case for `TransportMsgType_StorageFull`:
       ```cpp
       case wire::TransportMsgType_StorageFull: {
           auto* peer = find_peer(conn);
           if (peer) {
               peer->peer_is_full = true;
               spdlog::info("Peer {} reported storage full, suppressing sync pushes",
                            peer_display_name(conn));
           }
           return;
       }
       ```

    8. **Sync push suppression (Phase C):** In `run_sync_with_peer()`, at the point where blob transfers are initiated (Phase C, where we send BlobTransfer messages TO the peer), add a guard:
       ```cpp
       // Phase C: Exchange blobs
       // Skip outbound blob transfer if peer is full
       auto* peer = find_peer(conn);
       if (peer && peer->peer_is_full) {
           spdlog::debug("Skipping blob push to full peer {}", peer_display_name(conn));
           // Still process INCOMING blob transfers from this peer
       }
       ```
       Important: This suppresses only OUTBOUND blob pushes. The node should still:
       - Send BlobRequests to request blobs FROM the full peer (the full peer can still serve data it has)
       - Process incoming BlobTransfer messages from the full peer
       - Complete Phase A/B normally (namespace lists and hash lists)

    9. **Post-sync StorageFull signal:** After `run_sync_with_peer()` completes and `ingest_blobs()` returned stats with `storage_full_count > 0`, send StorageFull to the sync peer:
       ```cpp
       if (stats.storage_full_count > 0) {
           auto msg = wire::encode_transport(wire::TransportMsgType_StorageFull, {});
           co_await conn->async_send(msg);
           spdlog::warn("Sent StorageFull to sync peer {} ({} blobs rejected)",
                        peer_display_name(conn), stats.storage_full_count);
       }
       ```
       Also do the same in `handle_sync_as_responder()` -- both sides should signal if their ingest_blobs hit capacity.

    10. **No reconnection-based recovery:** Do NOT add a StorageAvailable message. Recovery is reconnection-only -- PeerInfo is recreated fresh on new connections, so peer_is_full naturally resets.
