    1. **storage.h changes:**
       - Add `NamespaceQuota` struct: `uint64_t total_bytes = 0; uint64_t blob_count = 0;`
       - Add public methods:
         - `NamespaceQuota get_namespace_quota(std::span<const uint8_t, 32> ns);` — O(1) read from quota_map
         - `void rebuild_quota_aggregates();` — full scan of blobs_map, rebuild quota_map
       - Update class doc comment: "Manages seven sub-databases" and add quota line:
         `/// - quota: [namespace:32] -> [total_bytes_be:8][blob_count_be:8]`

    2. **storage.cpp Impl changes:**
       - Add `mdbx::map_handle quota_map{0};` to Impl struct
       - Change `operate_params.max_maps` from 7 to 8 (7 named sub-dbs + 1 default). CRITICAL: Pitfall 5 from RESEARCH.md.
       - Add `quota_map = txn.create_map("quota");` after cursor_map
       - Add private helpers in anonymous namespace:
         ```
         static constexpr size_t QUOTA_VALUE_SIZE = 16;
         encode_quota_value(const NamespaceQuota& q) -> std::array<uint8_t, 16>
         decode_quota_value(const uint8_t* data) -> NamespaceQuota
         ```
         Use big-endian uint64 encoding (reuse existing encode_be_u64/decode_be_u64 helpers).

    3. **store_blob (3-arg overload) changes** — inside the write transaction, AFTER the blob is stored and encrypted, BEFORE commit:
       - Check `wire::is_tombstone(blob.data)` — if tombstone, skip quota increment (tombstones exempt per user decision)
       - Read current aggregate from quota_map for this namespace (32-byte key)
       - Increment: `current.total_bytes += encrypted.size()` (actual storage cost), `current.blob_count += 1`
       - Upsert updated aggregate back to quota_map
       - NOTE: This increment is in the SAME write transaction as the blob store — atomic.
       - NOTE: The dedup early-return path (line 368 txn.abort()) must NOT increment — it correctly skips.

    4. **delete_blob_data changes** — inside the write transaction, AFTER confirming blob exists and reading its encrypted size, BEFORE commit:
       - Read the encrypted blob value size from `existing.length()` BEFORE erasing from blobs_map
       - Read current aggregate from quota_map
       - Decrement: `current.total_bytes -= existing.length()` (clamp to 0 to prevent underflow), `current.blob_count -= 1` (clamp to 0)
       - Upsert updated aggregate. If both reach 0, erase the quota_map entry to keep it clean.

    5. **run_expiry_scan changes** — inside the write transaction, for each expired blob deleted:
       - Read the encrypted blob value size from `raw.length()` (already fetched on line 839-841)
       - Read current aggregate, decrement bytes and count, upsert back
       - Same clamping to 0 as delete_blob_data

    6. **rebuild_quota_aggregates implementation:**
       - Start a write transaction
       - Clear the entire quota_map (cursor scan + erase all entries)
       - Cursor scan over blobs_map: for each entry, extract namespace from key (first 32 bytes), add value.length() to that namespace's running total, increment count
       - Write all computed aggregates to quota_map
       - Commit
       - Log: `spdlog::info("Quota rebuild: {} namespaces, {} total blobs", ns_count, total_count)`

    7. **get_namespace_quota implementation:**
       - Start a read transaction
       - Lookup namespace key in quota_map
       - If found (length == 16): decode and return
       - If not found: return {0, 0}

    8. **Constructor addition:**
       - After cursor cleanup / validate_no_unencrypted_data (end of constructor), call `rebuild_quota_aggregates()` — ensures accuracy on every startup.

       - "run_expiry_scan decrements quota aggregate" — store blob with TTL, advance clock, run scan, check aggregate
       - "rebuild_quota_aggregates matches actual storage"
       - "multiple namespaces tracked independently"
