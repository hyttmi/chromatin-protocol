  <action>
    **Implementation approach (per D-01, D-02, D-03, D-09):**

    1. **Internal data structure:** Add `self._kem_ring: list[tuple[int, bytes, oqs.KeyEncapsulation | None]]` to Identity.__init__. Each entry is `(version, kem_public_key, kem_object_or_none)`. The list is sorted by version ascending. The last entry is always the "current" keypair.

    2. **Constructor changes:** Keep backward-compatible constructor. When `kem_ring` is not provided (default None), build a ring of length 1 from existing `kem_public_key` and `kem` parameters with version 0. Add optional `kem_ring` parameter:
       ```python
       def __init__(self, public_key, namespace, signer=None,
                    kem_public_key=None, kem=None, *,
                    kem_ring=None):
       ```
       If `kem_ring` is provided, use it directly; derive `_kem_public_key` and `_kem` from `kem_ring[-1]`.
       If `kem_ring` is None and `kem_public_key` is not None, build `[(0, kem_public_key, kem)]`.
       If both are None, `_kem_ring = []`.

    3. **Properties:**
       - `key_version` -> `int`: Returns `self._kem_ring[-1][0]` if ring non-empty, else 0.
       - `kem_public_key` -> `bytes`: Returns `self._kem_ring[-1][1]` (unchanged behavior, just different source).
       - `has_kem` -> `bool`: Returns `len(self._kem_ring) > 0`.

    4. **rotate_kem(key_path: str | Path) -> None** (per D-09):
       - Raise KeyFileError if `self._signer is None` (verify-only identity cannot rotate)
       - Raise KeyFileError if `self._kem is None` (no KEM to rotate from -- defensive)
       - `key_path = Path(key_path); parent = key_path.parent; stem = key_path.stem`
       - `current_version = self._kem_ring[-1][0]`
       - If first rotation (current_version == 0 and no `{stem}.kem.0` on disk):
         Write `{stem}.kem.0` with `self._kem_ring[-1][2].export_secret_key()` (3168 bytes)
         Write `{stem}.kpub.0` with `self._kem_ring[-1][1]` (1568 bytes)
       - Generate new KEM: `new_kem = oqs.KeyEncapsulation("ML-KEM-1024"); new_pk = bytes(new_kem.generate_keypair())`
       - `new_version = current_version + 1`
       - Write `{stem}.kem.{new_version}` with `new_kem.export_secret_key()` (3168 bytes)
       - Write `{stem}.kpub.{new_version}` with `new_pk` (1568 bytes)
       - Update canonical: `key_path.with_suffix(".kem").write_bytes(new_kem.export_secret_key())`
       - Update canonical: `key_path.with_suffix(".kpub").write_bytes(new_pk)`
       - Append `(new_version, new_pk, new_kem)` to `self._kem_ring`
       - Update `self._kem_public_key = new_pk; self._kem = new_kem`

    5. **load() changes** (per D-02, D-03):
       After reading canonical .kem/.kpub files, glob for numbered files:
       ```python
       numbered = sorted(
           (p for p in parent.glob(f"{stem}.kem.[0-9]*")
            if p.suffix.lstrip('.').isdigit()),
           key=lambda p: int(p.suffix.lstrip('.'))
       )
       ```
       Wait -- pathlib.suffix returns the LAST suffix. For `identity.kem.0`, Path.suffix is `.0` and Path.stem is `identity.kem`. Need different approach.

       Actually `Path("identity.kem.0").suffix` returns `".0"`. And the glob `parent.glob(f"{stem}.kem.[0-9]*")` where stem="identity" will match `identity.kem.0`, `identity.kem.1`, etc. The suffix is `.0`, `.1` etc. So:
       ```python
       numbered_sec = sorted(
           parent.glob(f"{stem}.kem.[0-9]*"),
           key=lambda p: int(p.suffix.lstrip('.'))
       )
       ```
       Filter out non-numeric suffixes (e.g., `.bak`): `[p for p in ... if p.suffix.lstrip('.').isdigit()]`

       If `len(numbered_sec) == 0`: Pre-rotation identity. Build ring with single entry `(0, kem_public, kem)`.
       If `len(numbered_sec) > 0`: Build ring from numbered files:
       ```python
       kem_ring = []
       for sec_path in numbered_sec:
           version = int(sec_path.suffix.lstrip('.'))
           pub_path = parent / f"{stem}.kpub.{version}"
           sk_bytes = sec_path.read_bytes()
           pk_bytes = pub_path.read_bytes()
           # Validate sizes
           if len(sk_bytes) != KEM_SECRET_KEY_SIZE:
               raise KeyFileError(...)
           if len(pk_bytes) != KEM_PUBLIC_KEY_SIZE:
               raise KeyFileError(...)
           kem_obj = oqs.KeyEncapsulation("ML-KEM-1024", secret_key=sk_bytes)
           kem_ring.append((version, pk_bytes, kem_obj))
       ```
       Return `cls(public_key, namespace, signer, kem_ring=kem_ring)` (no separate kem_public_key/kem params when ring is provided).

    6. **save() changes:** When ring length > 1, write numbered files for all entries:
       ```python
       for version, pk, kem_obj in self._kem_ring:
           (parent / f"{stem}.kem.{version}").write_bytes(kem_obj.export_secret_key())
           (parent / f"{stem}.kpub.{version}").write_bytes(pk)
       ```
       Always write canonical .kem/.kpub from latest entry (ring[-1]).
       When ring length == 1 and version == 0 (pre-rotation), do NOT write numbered files (D-03 lazy migration).

    7. **_build_kem_ring_map() -> dict[bytes, oqs.KeyEncapsulation]:**
       ```python
       def _build_kem_ring_map(self) -> dict[bytes, oqs.KeyEncapsulation]:
           return {sha3_256(pk): kem_obj
                   for _, pk, kem_obj in self._kem_ring
                   if kem_obj is not None}
       ```
       Returns empty dict for public-only identities (from_public_keys).

    8. **from_public_keys() change:** Build ring with `[(0, kem_pk, None)]`. The `None` kem_obj means no secret key.

    9. **Ensure all existing tests pass.** The constructor change is backward-compatible: existing callers pass positional args (public_key, namespace, signer, kem_public_key, kem) which still work since kem_ring defaults to None and the old path builds a single-entry ring.
