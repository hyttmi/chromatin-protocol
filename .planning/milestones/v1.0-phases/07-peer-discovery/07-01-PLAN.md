```cpp
void Server::connect_once(const std::string& address) {
    if (draining_) return;
    asio::co_spawn(ioc_, [this, address]() -> asio::awaitable<void> {
        auto [host, port] = parse_address(address);

        asio::ip::tcp::resolver resolver(ioc_);
        auto [ec_resolve, endpoints] = co_await resolver.async_resolve(
            host, port, use_nothrow);
        if (ec_resolve) {
            spdlog::debug("connect_once: failed to resolve {}: {}", address, ec_resolve.message());
            co_return;  // Give up, no reconnect
        }

        asio::ip::tcp::socket socket(ioc_);
        auto [ec_connect, ep] = co_await asio::async_connect(
            socket, endpoints, use_nothrow);
        if (ec_connect) {
            spdlog::debug("connect_once: failed to connect to {}: {}", address, ec_connect.message());
            co_return;  // Give up, no reconnect
        }

        spdlog::info("connect_once: connected to {}", address);

        auto conn = Connection::create_outbound(std::move(socket), identity_);
        connections_.push_back(conn);

        conn->on_close([this](Connection::Ptr c, bool /*graceful*/) {
            remove_connection(c);
            if (on_disconnected_) on_disconnected_(c);
        });

        conn->on_ready([this](Connection::Ptr c) {
            if (on_connected_) on_connected_(c);
        });

        co_await conn->run();
        // Connection ended -- no reconnect for discovered peers
    }, asio::detached);
}
```

**KEY DESIGN NOTES:**
- `connect_once` is fire-and-forget: spawns a coroutine and returns immediately.
- On resolve or connect failure, it just returns (no reconnect_loop call).
- On successful connection + run completion, it does NOT reconnect (unlike `connect_to_peer` which falls into `reconnect_loop`).
- Uses the same `on_ready` callback pattern as existing connection methods so PeerManager gets notified via `on_peer_connected`.

**2. Implement PEX in peer_manager.cpp:**

**Wire encoding** (binary, big-endian, matching sync payload style):
```
encode_peer_list: [uint16_t count][for each: uint16_t addr_len, utf8 addr_bytes]
decode_peer_list: inverse of above
```
Use big-endian uint16_t (same pattern as sync's encode_namespace_list). This is a static method, easily unit-testable.

**build_peer_list_response(exclude_address):**
1. Collect addresses of authenticated peers (from peers_ vector, using `address` field)
2. Filter out: `exclude_address` (the requester), `config_.bind_address` (self)
3. Only include peers we have actively connected to and authenticated (i.e., they are in peers_ and `connection->is_authenticated()`)
4. Shuffle the result using `std::mt19937` + `std::random_device`
5. Truncate to `MAX_PEERS_PER_EXCHANGE` (8)
6. Return the vector of address strings

**handle_peer_list_request(conn):**
1. Build response using `build_peer_list_response(conn->remote_address())`
2. Encode with `encode_peer_list`
3. Send via `conn->send_message(TransportMsgType_PeerListResponse, payload)` -- fire and forget with co_spawn since we're in a non-coroutine callback context. Actually, send_message is awaitable, so wrap in co_spawn:
```cpp
void PeerManager::handle_peer_list_request(net::Connection::Ptr conn) {
    auto addresses = build_peer_list_response(conn->remote_address());
    auto payload = encode_peer_list(addresses);
    spdlog::debug("PEX: sending {} peers to {}", addresses.size(), conn->remote_address());
    asio::co_spawn(ioc_, [conn, payload = std::move(payload)]() -> asio::awaitable<void> {
        co_await conn->send_message(wire::TransportMsgType_PeerListResponse,
                                     std::span<const uint8_t>(payload));
    }, asio::detached);
}
```

**handle_peer_list_response(conn, payload):**
1. Decode payload with `decode_peer_list`
2. For each address in the response:
   - Skip if address is in `known_addresses_` (already know about it)
   - Skip if address == `config_.bind_address` (that's us)
   - Skip if we already have `peers_.size() >= config_.max_peers`
   - Add to `known_addresses_`
3. Collect up to `MAX_DISCOVERED_PER_ROUND` (3) new addresses
4. For each new address, call `server_.connect_once(address)`
5. Log: "PEX: discovered N new peers from {remote}, connecting to M"

**pex_timer_loop():**
Follow exact pattern of sync_timer_loop:
```cpp
asio::awaitable<void> PeerManager::pex_timer_loop() {
    while (!stopping_) {
        asio::steady_timer timer(ioc_);
        timer.expires_after(std::chrono::seconds(PEX_INTERVAL_SEC));
        auto [ec] = co_await timer.async_wait(asio::as_tuple(asio::use_awaitable));
        if (ec || stopping_) co_return;
        co_await request_peers_from_all();
    }
}
```

**request_peers_from_all():**
```cpp
asio::awaitable<void> PeerManager::request_peers_from_all() {
    for (auto& peer : peers_) {
        if (peer.connection->is_authenticated()) {
            send_peer_list_request(peer.connection);
        }
    }
    co_return;
}
```
