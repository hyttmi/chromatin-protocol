  <behavior>
    Sequence numbering (verify store_blob already assigns seq_nums from Plan 01):
    - First blob in a namespace gets seq_num = 1
    - Second blob in same namespace gets seq_num = 2
    - Blobs in different namespaces have independent seq_num counters (both start at 1)
    - Duplicate blob does not consume a seq_num
    - After storing 5 blobs, seq_nums are 1,2,3,4,5 with no gaps

    Range query (get_blobs_by_seq):
    - get_blobs_by_seq(ns, 0) returns all blobs in namespace in seq order
    - get_blobs_by_seq(ns, 3) returns only blobs with seq > 3
    - get_blobs_by_seq(ns, 999) returns empty vector (no blobs past 999)
    - get_blobs_by_seq for empty namespace returns empty vector
    - Returned blobs are in seq_num order (ascending)
    - Blobs from other namespaces are not included

    Expiry index verification:
    - Storing blob with TTL > 0 creates an entry in expiry index
    - Storing blob with TTL = 0 does NOT create an expiry index entry
    - Expiry index key uses big-endian timestamp so entries are sorted by time
  </behavior>
  <action>
    **Read existing code first:** Read `src/storage/storage.h` and `src/storage/storage.cpp` to understand the current implementation from Plan 02-01. The store_blob() method should already be assigning seq_nums and populating the expiry index. This task focuses on:

    1. **Verifying seq_num assignment** via new tests. If store_blob() from Plan 01 already correctly assigns seq_nums (cursor to last entry for namespace, increment), the sequence tests should pass without code changes. If the implementation has issues, fix them.

    2. **Implementing get_blobs_by_seq():** Replace the stub with a real implementation:
       - Start a read transaction
       - Build a lower-bound key: `make_seq_key(ns, since_seq + 1)` -- we want blobs AFTER since_seq
       - Open a cursor on seq_map_, seek to `lower_bound(lower_key)`
       - Iterate forward while the key's namespace prefix (first 32 bytes) matches `ns`
       - For each seq entry: extract the hash (32-byte value), build blob key, fetch full blob from blobs_map_, decode via `wire::decode_blob()`, add to result vector
       - If a seq entry points to a missing blob (deleted by expiry), skip it gracefully
       - Return the collected blobs in seq_num order

    3. **Verifying expiry index population:** Write tests that store blobs with TTL > 0 and TTL = 0, then directly inspect the expiry sub-database to confirm entries exist (or don't). To inspect, open a read transaction, use a cursor on expiry_map_, and iterate to verify.

    **Edge cases to handle:**
    - Cursor reaching end of database (no more entries)
    - Namespace boundary crossing (cursor moves past entries for our namespace into another namespace's entries in seq_map)
    - Empty namespace (no entries at all)

    **NOTE:** If storage.h needs a method to inspect internals for testing (e.g., `uint64_t get_latest_seq(ns)` or `size_t count_expiry_entries()`), add it. Testing internal state is appropriate for storage engine tests.

    Write TDD tests FIRST following the behavior list above. Use the same test fixture pattern from Plan 01 (temp directory, construct Storage, cleanup).
