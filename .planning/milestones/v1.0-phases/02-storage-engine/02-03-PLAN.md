    Expiry scan:
    - run_expiry_scan() with no expired blobs returns 0
    - Store blob with TTL=100 at timestamp=1000 (expires at 1100). Set clock to 1101. run_expiry_scan() returns 1 and blob is gone from get_blob()
    - Store 3 blobs with different expiry times. Advance clock past 2 of them. run_expiry_scan() returns 2, third blob still exists
    - run_expiry_scan() called twice -- second call returns 0 (already purged)
    - Purged blob removed from blobs sub-database (get_blob returns nullopt)
    - Purged blob removed from expiry sub-database (no stale entries)
    - Sequence index entries for purged blobs remain (get_blobs_by_seq skips missing blobs)

    TTL=0 permanent blobs:
    - Store blob with TTL=0. Advance clock far into the future. run_expiry_scan() returns 0. Blob still retrievable via get_blob()
    - Mix of TTL=0 and TTL > 0 blobs: only TTL > 0 blobs are purged when expired

    Injectable clock:
    - Tests construct Storage with a lambda returning a controlled fake_time variable
    - Advancing fake_time triggers different expiry behavior without real-time waits

    Edge cases:
    - Expiry scan on empty database returns 0
    - All blobs expired -- scan purges everything, database is empty but operational
    - Blob stored with very large TTL (e.g., UINT32_MAX) does not expire for a long time
  </behavior>
  <action>
    **Read existing code first:** Read `src/storage/storage.h` and `src/storage/storage.cpp` to understand the current implementation. The `run_expiry_scan()` method should currently be a stub returning 0. Replace it with the real implementation.

    **Implement run_expiry_scan() in storage.cpp:**
    1. Get current time from `clock_()`
    2. Start a write transaction
    3. Open cursor on `expiry_map_`
    4. Move cursor to first entry: `cursor.to_first(/*throw_notfound=*/false)`
    5. Loop while cursor is valid:
       a. Read key: extract expiry_timestamp (first 8 bytes, big-endian decode) and blob_hash (next 32 bytes)
       b. If `expiry_timestamp > now`: break -- all remaining entries are in the future (sorted by big-endian timestamp)
       c. Read value: extract namespace (32 bytes)
       d. Build blob key from namespace + blob_hash
       e. Delete from blobs sub-database: `txn.erase(blobs_map_, blob_key_slice)`
       f. Delete current expiry entry via cursor: `cursor.erase()`
       g. Increment purge counter
       h. Move to next: `cursor.to_next(/*throw_notfound=*/false)` -- NOTE: after erase(), the cursor may need re-positioning. Check libmdbx docs -- `cursor.erase()` typically positions cursor to next entry automatically, but verify.
    6. Commit transaction
    7. Log purged count via spdlog
    8. Return purge count

    **IMPORTANT:** Do NOT delete sequence index entries during expiry scan. Per CONTEXT.md: "Seq_nums will have gaps after expiry -- callers must handle this." The get_blobs_by_seq implementation (Plan 02) should already skip missing blobs gracefully.

    **Test fixture with injectable clock:**
    ```cpp
    // In test file:
    uint64_t fake_time = 1000;  // start at t=1000
    Storage store(temp_dir, [&]() -> uint64_t { return fake_time; });

    // Store blob with TTL=100, timestamp=1000 -> expires at 1100
    wire::BlobData blob;
    blob.timestamp = 1000;
    blob.ttl = 100;
    // ... fill other fields
    store.store_blob(blob);

    // Before expiry
    REQUIRE(store.run_expiry_scan() == 0);
    REQUIRE(store.has_blob(ns, hash));

    // Advance past expiry
    fake_time = 1101;
    REQUIRE(store.run_expiry_scan() == 1);
    REQUIRE_FALSE(store.has_blob(ns, hash));
    ```

The namespace_id should be deterministic per test (use a counter or fixed values) so tests can reliably query by namespace. Signature and pubkey can be arbitrary bytes for storage-only tests (no verification at this layer).
