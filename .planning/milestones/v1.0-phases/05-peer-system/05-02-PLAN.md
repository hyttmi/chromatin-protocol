1. **Extend Server** to support PeerManager callbacks:
   Server needs to let PeerManager control connection acceptance and receive lifecycle events. Add to `Server`:
   ```cpp
   // Callback types for PeerManager integration
   using ConnectionCallback = std::function<void(Connection::Ptr)>;
   using AcceptFilter = std::function<bool()>;  // return false to reject

   void set_on_connected(ConnectionCallback cb);    // Called after successful handshake
   void set_on_disconnected(ConnectionCallback cb);  // Called on connection close
   void set_accept_filter(AcceptFilter cb);          // Called before accepting inbound
   ```

   In `accept_loop()`: before creating Connection, call accept_filter. If returns false, close socket and continue.

   In the connection lifecycle: after `conn->run()` returns true (handshake succeeded), call on_connected callback. In on_close callback, call on_disconnected.

   Server also needs to expose BlobEngine reference for PeerManager to set up message handlers. Add constructor parameter:

   ```cpp
          const identity::NodeIdentity& identity,

   ```

Add `is_bootstrap` tracking: when Server connects to a bootstrap peer address, tag the connection. Actually, since we use pubkeys for identification, and bootstrap peers are identified by address, PeerManager should track by address. Add `std::string remote_address()` to Connection (returns the peer's address string stored at connection time).

2. **Create PeerManager** (`src/peer/peer_manager.h`, `src/peer/peer_manager.cpp`):

   ```cpp
   namespace chromatin::peer {

   struct PeerInfo {
       net::Connection::Ptr connection;
       std::string address;
       bool is_bootstrap = false;
       uint32_t strike_count = 0;
       bool syncing = false;
   };

   class PeerManager {
   public:
       PeerManager(const config::Config& config,
                   identity::NodeIdentity& identity,
                   engine::BlobEngine& engine,
                   storage::Storage& storage,
                   asio::io_context& ioc);

       void start();  // Starts Server + sync timer
       void stop();   // Stops Server + cancels timers

       size_t peer_count() const;
       size_t bootstrap_peer_count() const;

   private:
       // Server callbacks
       void on_peer_connected(net::Connection::Ptr conn);
       void on_peer_disconnected(net::Connection::Ptr conn);
       bool should_accept_connection();

       // Message routing
       void on_peer_message(net::Connection::Ptr conn,
                            wire::TransportMsgType type,
                            std::vector<uint8_t> payload);

       // Sync orchestration
       asio::awaitable<void> sync_with_peer(net::Connection::Ptr conn);
       asio::awaitable<void> sync_all_peers();
       asio::awaitable<void> sync_timer_loop();

       // Sync message handling (responder side)
       asio::awaitable<void> handle_sync_request(net::Connection::Ptr conn);
       asio::awaitable<void> handle_sync_message(net::Connection::Ptr conn,
                                                  wire::TransportMsgType type,
                                                  std::vector<uint8_t> payload);

       // Strike system
       void record_strike(net::Connection::Ptr conn);

       const config::Config& config_;
       identity::NodeIdentity& identity_;
       engine::BlobEngine& engine_;
       storage::Storage& storage_;
       asio::io_context& ioc_;

       net::Server server_;
       sync::SyncProtocol sync_proto_;

       std::vector<PeerInfo> peers_;
       std::set<std::string> bootstrap_addresses_;

       static constexpr uint32_t STRIKE_THRESHOLD = 10;
       static constexpr uint32_t STRIKE_COOLDOWN_SEC = 300;  // 5 minutes
   };
   }
   ```

   Implementation details:

   **Connection limit:** `should_accept_connection()` returns `peers_.size() < config_.max_peers`.

   **Peer tracking:** `on_peer_connected` adds to peers_ vector with is_bootstrap flag based on address lookup. Sets up on_message callback to route through PeerManager. `on_peer_disconnected` removes from peers_.

   **Sync on connect:** `on_peer_connected` spawns `sync_with_peer(conn)` as a coroutine via `asio::co_spawn`.

   **Sync orchestration** (`sync_with_peer`):
   This is the coroutine that drives the sync protocol over a single Connection. Flow:
   ```
   1. Send SyncRequest message
   2. Wait for SyncAccept
   3. Collect our namespace list, send NamespaceList
   4. Receive peer's NamespaceList
   5. For each shared namespace:
      a. Collect our hashes, send HashList
      b. Receive peer's HashList
      c. Diff: find what we need from them
      d. If we need blobs: send BlobRequest, receive BlobTransfer, ingest
      e. If they need blobs: receive BlobRequest, send BlobTransfer
   6. Send SyncComplete
   7. Log summary
   ```

   **Sync timer:** `sync_timer_loop` uses `asio::steady_timer`, wakes every `sync_interval_seconds`, calls `sync_all_peers()` which iterates peers_ and spawns `sync_with_peer` for each authenticated, non-syncing peer.

   **Handling incoming sync:** When a peer sends SyncRequest to us, we handle it as responder (same protocol, but we wait for their messages first then respond). The on_message routing checks for sync message types and delegates to handle_sync_message.

   **Sync concurrency guard:** Set `syncing = true` on PeerInfo before starting sync, clear after. Skip peers already syncing. This prevents overlapping sync rounds on the same connection.

   **Strike system:** `record_strike` increments strike_count on the PeerInfo. If >= STRIKE_THRESHOLD, log at warn level "disconnecting peer {addr}: {count} validation failures", call conn->close_gracefully(). Strike count resets when peer reconnects (on_peer_connected initializes to 0).

   **Logging (per CONTEXT.md locked decisions):**
   - Sync summary at info: `spdlog::info("Synced with peer {}: received {} blobs, sent {} blobs", addr, stats.blobs_received, stats.blobs_sent)`
   - Individual transfers at debug: `spdlog::debug("transferring blob {} to peer {}", hash_hex, addr)`
   - Connect/disconnect at info: `spdlog::info("Connected to peer {}@{}", ns_hex, addr)` / `spdlog::info("Peer {} disconnected ({})", ns_hex, graceful ? "graceful" : "timeout")`
   - Invalid blobs at warn: `spdlog::warn("invalid blob from peer {}: {}", addr, error_detail)`

   **Reconnect policy:** Server's existing reconnect_loop handles bootstrap reconnection. PeerManager's `on_peer_disconnected` does NOT trigger reconnect for non-bootstrap peers (Server only reconnects addresses from `config_.bootstrap_peers`). This is already the behavior -- Server only calls `reconnect_loop` for bootstrap peer addresses in its on_close callback. Verify this behavior is preserved with the PeerManager integration.
