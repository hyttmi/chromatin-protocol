    **schemas/transport.fbs:**
    ```flatbuffers
    namespace chromatin.wire;

    enum TransportMsgType : byte {
        KemPubkey = 1,
        KemCiphertext = 2,
        AuthSignature = 3,
        AuthPubkey = 4,
        Ping = 5,
        Pong = 6,
        Goodbye = 7,
        Data = 8
    }

    table TransportMessage {
        type: TransportMsgType;
        payload: [ubyte];
    }

    root_type TransportMessage;
    ```

    Note: AuthPubkey is needed because during mutual auth, each side sends its ML-DSA-87 public key along with the signature so the peer can verify. Alternatively, the signing public key can be included in the AuthSignature payload. Keep it simple: use a single payload field and let the type disambiguate.

  <behavior>
    Protocol codec:
    - encode(type, payload) -> bytes: produces valid FlatBuffer TransportMessage
    - decode(bytes) -> (type, payload): round-trips correctly for all message types
    - decode of corrupt bytes returns error (nullopt or throws)

    Nonce generation:
    - make_nonce(0) produces 12 bytes: 4 zero bytes + 8 zero bytes
    - make_nonce(1) produces 12 bytes: 4 zero bytes + BE counter 1
    - make_nonce(UINT64_MAX) produces correct encoding
    - Sequential nonces are all unique

    Frame writer (in-memory, no socket):
    - write_frame(plaintext, key, counter) -> [4-byte BE len][ciphertext]
    - Ciphertext length = plaintext + TAG_SIZE (16)
    - Length prefix encodes ciphertext length correctly in big-endian

    Frame reader (in-memory, no socket):
    - read_frame(buffer, key, counter) -> plaintext
    - Round-trip: write_frame then read_frame recovers original plaintext
    - Frame exceeding 16 MB max rejected before allocation
    - Decrypt failure (wrong key) returns error
    - Decrypt failure (wrong counter/nonce desync) returns error

    Integration:
    - Encode a TransportMessage, encrypt+frame it, read frame, decrypt, decode -> matches original
  </behavior>
  <action>
    **src/net/protocol.h/cpp:** Create `chromatin::net::TransportCodec` with static methods:
    - `static std::vector<uint8_t> encode(TransportMsgType type, std::span<const uint8_t> payload)`: Build FlatBuffer TransportMessage, return serialized bytes.
    - `struct DecodedMessage { TransportMsgType type; std::vector<uint8_t> payload; };`
    - `static std::optional<DecodedMessage> decode(std::span<const uint8_t> data)`: Verify FlatBuffer, extract type and payload. Return nullopt on invalid data.

    **src/net/framing.h/cpp:** Create framing utilities in `chromatin::net` namespace:

    ```cpp
    constexpr uint32_t MAX_FRAME_SIZE = 16 * 1024 * 1024; // 16 MB
    constexpr size_t FRAME_HEADER_SIZE = 4; // uint32_t big-endian

    // Counter-based nonce: 4 zero bytes + 8-byte big-endian counter
    std::array<uint8_t, crypto::AEAD::NONCE_SIZE> make_nonce(uint64_t counter);

    // Encrypt plaintext and wrap in length-prefixed frame
    // Returns: [4-byte BE length][ciphertext with AEAD tag]
    // Uses empty associated data (AD = {})
    std::vector<uint8_t> write_frame(
        std::span<const uint8_t> plaintext,
        std::span<const uint8_t> key,
        uint64_t counter);

    // Parse length prefix from buffer, validate size, decrypt ciphertext
    // Returns: plaintext on success, nullopt on decrypt failure
    // Throws: runtime_error if frame exceeds MAX_FRAME_SIZE
    struct FrameResult {
        std::vector<uint8_t> plaintext;
        size_t bytes_consumed; // total bytes consumed from input (header + ciphertext)
    };
    std::optional<FrameResult> read_frame(
        std::span<const uint8_t> buffer,
        std::span<const uint8_t> key,
        uint64_t counter);
    ```

    Frame format: `[uint32_t BE ciphertext_length][ciphertext (plaintext_len + 16 bytes AEAD tag)]`

    The write_frame and read_frame functions work on byte buffers (not sockets) so they are testable without networking. The async socket IO will wrap these in Plan 03.

    Use empty `std::span<const uint8_t>{}` for associated data in AEAD calls. The counter-nonce provides replay protection.
