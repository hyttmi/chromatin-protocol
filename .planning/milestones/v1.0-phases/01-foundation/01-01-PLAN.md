<task type="auto" tdd="true">
  <name>Task 2: ML-DSA-87 signing and ML-KEM-1024 key exchange wrappers</name>
  <files>
    src/crypto/signing.h,
    src/crypto/signing.cpp,
    src/crypto/kem.h,
    src/crypto/kem.cpp,
    tests/crypto/test_signing.cpp,
    tests/crypto/test_kem.cpp
  </files>
  <behavior>
    Signing:
    - generate_keypair() produces pubkey of exactly 2592 bytes and secret key of exactly 4866 bytes
    - sign() produces signature of at most 4627 bytes
    - verify() returns true for valid message+signature+pubkey
    - verify() returns false for tampered message
    - verify() returns false for wrong public key
    - Signer destructor securely erases secret key (uses SecureBytes)
    - Signer supports export_public_key() and export_secret_key() as byte spans
    - Static Signer::from_keypair(pubkey, secret_key) reconstructs from saved key material

    KEM:
    - generate_keypair() produces pubkey of exactly 1568 bytes and secret key of exactly 3168 bytes
    - encaps(pubkey) produces ciphertext (1568 bytes) and shared_secret (32 bytes)
    - decaps(ciphertext, secret_key) produces shared_secret (32 bytes)
    - encaps shared_secret == decaps shared_secret (round-trip)
    - decaps with wrong secret key produces different shared secret (ML-KEM implicit reject)
    - KEM destructor securely erases secret key
  </behavior>
  <action>
    **signing.h/cpp:** Create `chromatin::crypto::Signer` class:
    - Constructor: calls `OQS_SIG_new(OQS_SIG_alg_ml_dsa_87)`, throws if null
    - `generate_keypair()`: calls `OQS_SIG_keypair()`, stores pubkey as `std::vector<uint8_t>`, secret key as `SecureBytes`
    - `sign(std::span<const uint8_t> message) -> std::vector<uint8_t>`: calls `OQS_SIG_sign()`
    - `static verify(std::span<const uint8_t> message, std::span<const uint8_t> signature, std::span<const uint8_t> public_key) -> bool`: creates temporary OQS_SIG, calls `OQS_SIG_verify()`
    - `export_public_key() -> std::span<const uint8_t>`
    - `export_secret_key() -> std::span<const uint8_t>`
    - `static from_keypair(std::span<const uint8_t> pubkey, std::span<const uint8_t> secret_key) -> Signer`
    - Destructor: `OQS_SIG_free()`, SecureBytes auto-zeroes secret key
    - Delete copy, allow move

    **kem.h/cpp:** Create `chromatin::crypto::KEM` class:
    - Constructor: calls `OQS_KEM_new(OQS_KEM_alg_ml_kem_1024)`, throws if null
    - `generate_keypair()`: calls `OQS_KEM_keypair()`
    - `encaps(std::span<const uint8_t> public_key) -> std::pair<std::vector<uint8_t>, SecureBytes>` (ciphertext, shared_secret)
    - `decaps(std::span<const uint8_t> ciphertext, std::span<const uint8_t> secret_key) -> SecureBytes` (shared_secret)
    - Destructor: `OQS_KEM_free()`, SecureBytes auto-zeroes
    - Delete copy, allow move

    Use `OQS_MEM_secure_free()` for any temporary liboqs allocations. Always check `OQS_STATUS` return codes and throw `std::runtime_error` on failure.

    Write TDD tests FIRST following behavior list above.
  </action>
  <verify>
    <automated>cd build && cmake --build . --target chromatindb_tests && ./chromatindb_tests --reporter compact "[signing]" "[kem]"</automated>
  </verify>
  <done>ML-DSA-87 sign/verify round-trips. ML-KEM-1024 encaps/decaps produces matching shared secrets. Wrong key/tampered message correctly detected. All crypto material securely erased.</done>
</task>

<task type="auto" tdd="true">
  <name>Task 3: ChaCha20-Poly1305 AEAD and HKDF-SHA256 KDF wrappers</name>
  <files>
    src/crypto/aead.h,
    src/crypto/aead.cpp,
    src/crypto/kdf.h,
    src/crypto/kdf.cpp,
    tests/crypto/test_aead.cpp,

  <behavior>
    AEAD:
    - encrypt then decrypt round-trips (plaintext matches)
    - encrypt with associated data, decrypt verifies AD (decrypt fails with wrong AD)
    - tampered ciphertext fails decryption (returns error/throws)
    - wrong key fails decryption
    - different nonces produce different ciphertext for same plaintext
    - keygen() produces 32-byte key
    - ciphertext is plaintext_len + 16 bytes (tag appended)

    KDF:
    - extract(salt, ikm) produces PRK (pseudorandom key)
    - expand(prk, context, len) produces derived key of requested length
    - same inputs produce same derived key (deterministic)
    - different contexts produce different derived keys
    - full derive(salt, ikm, context, len) convenience function works
    - derive from ML-KEM shared secret (32 bytes) produces valid AEAD key (32 bytes)
  </behavior>
  <action>
    **aead.h/cpp:** Create `chromatin::crypto::AEAD` namespace with free functions:
    - `keygen() -> SecureBytes`: wraps `crypto_aead_chacha20poly1305_ietf_keygen()`
    - `encrypt(std::span<const uint8_t> plaintext, std::span<const uint8_t> ad, std::span<const uint8_t, 12> nonce, std::span<const uint8_t, 32> key) -> std::vector<uint8_t>`: wraps `crypto_aead_chacha20poly1305_ietf_encrypt()`
    - `decrypt(std::span<const uint8_t> ciphertext, std::span<const uint8_t> ad, std::span<const uint8_t, 12> nonce, std::span<const uint8_t, 32> key) -> std::optional<std::vector<uint8_t>>`: wraps `crypto_aead_chacha20poly1305_ietf_decrypt()`, returns nullopt on auth failure
    - Constants: `KEY_SIZE = 32`, `NONCE_SIZE = 12`, `TAG_SIZE = 16`

    Ensure `sodium_init()` is called before any libsodium operation. Use a static initializer or call in each function with `static bool init = (sodium_init(), true);` pattern.

    **kdf.h/cpp:** Create `chromatin::crypto::KDF` namespace with free functions:
    - `extract(std::span<const uint8_t> salt, std::span<const uint8_t> ikm) -> SecureBytes`: wraps `crypto_kdf_hkdf_sha256_extract()`
    - `expand(std::span<const uint8_t> prk, std::string_view context, size_t output_len) -> SecureBytes`: wraps `crypto_kdf_hkdf_sha256_expand()`
    - `derive(std::span<const uint8_t> salt, std::span<const uint8_t> ikm, std::string_view context, size_t output_len) -> SecureBytes`: convenience that calls extract then expand
    - Uses `crypto_kdf_hkdf_sha256_KEYBYTES` for PRK size

    Write TDD tests FIRST following behavior list above. Include an integration test: ML-KEM shared secret -> HKDF derive -> AEAD encrypt/decrypt round-trip.
