    - Config has fields: bind_address (string), storage_path (string), data_dir (string), bootstrap_peers (vector of string), default_ttl (uint32), log_level (string)

    struct Config {
        std::string bind_address = "0.0.0.0:4200";

        std::string data_dir = "./data";
        std::vector<std::string> bootstrap_peers;
        uint32_t default_ttl = 604800;  // 7 days in seconds
        std::string log_level = "info";
    };

    // Load from JSON file. Returns defaults if file doesn't exist.
    Config load_config(const std::filesystem::path& path);

    // Parse CLI arguments, override config fields

  <name>Task 2: Structured logging and node identity</name>

  <behavior>
    Logging:
    - init(log_level) creates console logger writing to stderr
    - get_logger(name) returns a named logger (e.g., "crypto", "config", "identity")
    - Log level "info" shows info+warn+error, hides debug+trace
    - Log level "debug" shows debug+info+warn+error
    - Log format: [YYYY-MM-DD HH:MM:SS.mmm] [level] [logger_name] message
    - init() can be called safely multiple times (idempotent)

    Identity:
    - NodeIdentity::generate() creates new keypair and derives namespace
    - namespace is SHA3-256(pubkey) = exactly 32 bytes
    - Same pubkey always produces same namespace (deterministic)
    - save_to(data_dir) writes node.key and node.pub as raw binary files
    - load_from(data_dir) reads node.key and node.pub, reconstructs identity
    - save then load round-trips: namespace matches
    - load_or_generate(data_dir) loads if files exist, generates + saves if not
    - node.key file is exactly 4866 bytes (ML-DSA-87 secret key)
    - node.pub file is exactly 2592 bytes (ML-DSA-87 public key)
    - NodeIdentity exposes namespace_id(), public_key(), and can sign messages
  </behavior>
  <action>
    **logging.h/cpp:** Create `chromatin::logging` namespace:
    ```cpp
    void init(const std::string& level = "info");
    std::shared_ptr<spdlog::logger> get_logger(const std::string& name);
    ```
    Use `spdlog::stderr_color_mt` for console sink. Set pattern to `[%Y-%m-%d %H:%M:%S.%e] [%l] [%n] %v`. Map level string to `spdlog::level::level_enum`. Use `spdlog::get()` to retrieve existing loggers by name (creates if not found).

    **identity.h/cpp:** Create `chromatin::identity::NodeIdentity` class:
    ```cpp
    class NodeIdentity {
        chromatin::crypto::Signer signer_;
        std::array<uint8_t, 32> namespace_id_;
    public:
        // Generate fresh keypair + derive namespace
        static NodeIdentity generate();

        // Load from data directory (node.key + node.pub files)
        static NodeIdentity load_from(const std::filesystem::path& data_dir);

        // Load if key files exist, otherwise generate + save
        static NodeIdentity load_or_generate(const std::filesystem::path& data_dir);

        // Save key files to data directory
        void save_to(const std::filesystem::path& data_dir) const;

        // Accessors
        std::span<const uint8_t, 32> namespace_id() const;
        std::span<const uint8_t> public_key() const;

        // Sign a message using the node's keypair
        std::vector<uint8_t> sign(std::span<const uint8_t> message) const;
    };
    ```

    Key I/O: Use `std::ofstream` with `std::ios::binary` to write raw bytes. Use `std::ifstream` with `std::ios::binary` to read. Verify file sizes on load (pubkey must be 2592, secret key must be 4866).

    Namespace derivation: `namespace_id_ = chromatin::crypto::sha3_256(signer_.export_public_key())`

    Write TDD tests FIRST. Use Catch2 with `[identity]` tag. Create temporary directories for key file I/O tests.

    **Logging tests are NOT needed** -- spdlog is a trusted dependency. Just verify it initializes without throwing.
