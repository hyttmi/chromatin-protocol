get_blob_count() {
    local container="$1"
    docker kill -s USR1 "$container" >/dev/null 2>&1 || true
    sleep 2
    local count
    count=$(docker logs "$container" 2>&1 | grep "metrics:" | tail -1 | grep -oP 'blobs=\K[0-9]+' || echo "0")
    echo "$count"
}

run_verify() {
    local container="$1"
    shift
    docker exec -i "$container" chromatindb_verify "$@"
}

run_loadgen() {
    local target_host="$1"
    shift
    docker run --rm --network "$NETWORK" \
        --entrypoint chromatindb_loadgen \
        "$IMAGE" \
        --target "${target_host}:4200" "$@" \
        2>/dev/null
}
```

**helpers.sh -- Fix get_blob_count for large log volumes:**
Change `docker logs "$container" 2>&1` to `docker logs --tail 200 "$container" 2>&1` in the `get_blob_count` function. This limits log output to the last 200 lines, preventing grep failures and stale results when debug logs are large (10K+ blobs generate thousands of log lines). The metrics line is always recent, so tail 200 is more than sufficient. This fix is prerequisite for RECON-01's 10K-blob baseline.

**test_net03_large_blob_integrity.sh -- Add hash verification after sync:**
Modify the test to capture blob field values via `--verbose-blobs` and verify hash integrity on node2 using `chromatindb_verify hash-fields`. Specifically:

1. For the FIRST 3 size tiers (1K, 100K, 1M): change the loadgen call to capture stderr with `--verbose-blobs`. Parse the BLOB_FIELDS JSON from stderr to extract namespace_id, data_hex, ttl, timestamp, and signing_digest. After sync to node2, run `chromatindb_verify hash-fields` inside node2 (via `run_verify`) with those field values and assert the signing_digest matches.

2. For the LAST 2 size tiers (10M, 100M): do NOT use `--verbose-blobs` because the data_hex output would be 20-200 MB of hex chars. The blob count verification already proves AEAD-authenticated transfer (corrupted data fails AEAD decryption and is never stored). Add a comment explaining this tradeoff.

    --data-hex "$DATA_HEX" \

    --timestamp "$TS" 2>/dev/null)
ACTUAL_DIGEST=$(echo "$VERIFY_JSON" | jq -r '.signing_digest')
# Compare ACTUAL_DIGEST to EXPECTED_DIGEST from BLOB_FIELDS
```

Note: Use `docker run --rm --entrypoint chromatindb_verify "$IMAGE"` (standalone container, not exec into node2) for the hash-fields verification since the tool operates on field values not stored data.

3. Create a temp dir for capturing verbose output. Add cleanup to the trap.

**test_net06_late_joiner.sh -- Add hash integrity check on late-joiner:**
