**test_recon01_diff_scaling.sh (RECON-01: O(diff) scaling):**

Uses tcpdump traffic capture via nicolaka/netshoot to measure wire traffic.

Test flow:
1. Start 2-node topology via docker-compose.recon.yml. Wait healthy.
2. Preload 10,000 blobs on node1 and wait for full sync to node2:
   `run_loadgen 172.28.0.2 --count 10000 --size 256 --rate 200 --ttl 3600`
   Use wait_sync with 300s timeout (10K blobs at 200/sec = 50s ingest + sync time).
3. Wait for sync to fully settle (additional 30s after wait_sync returns).
4. Stop node2 to prevent sync during blob injection: `docker stop chromatindb-test-node2`
5. Inject exactly 10 new blobs to node1: `run_loadgen 172.28.0.2 --count 10 --size 256 --ttl 3600`
6. Start tcpdump capture on the test network (nicolaka/netshoot container):
   ```bash
   docker run -d --name recon01-capture \
       --network chromatindb-test_test-net \
       --cap-add NET_ADMIN \
       nicolaka/netshoot \
       tcpdump -i any -w /tmp/diff-sync.pcap -s 0 port 4200
   ```
7. Restart node2: `docker start chromatindb-test-node2`. Wait healthy.
8. Wait for sync to node2 (10010 blobs, 120s timeout).
9. Stop tcpdump capture: `docker kill --signal SIGINT recon01-capture`, copy pcap file.
10. Measure traffic: `stat -c%s diff-sync.pcap`
11. Compare against expected: The diff sync traffic should be MUCH less than a full 10K-blob transfer. A 10K-blob transfer at 256 bytes/blob (plus protocol overhead ~200 bytes/blob) would be ~4.5 MB. The 10-blob diff should be well under 500 KB. Use threshold: diff traffic < 500000 bytes. This is very conservative (10 blobs * 256 bytes + overhead should be ~10 KB).
12. Verify blob count on node2 == 10010.

Cleanup: Remove netshoot container and pcap file in trap.

**test_recon02_empty_skip.sh (RECON-02: identical namespace skip):**

10. Also check cursor_misses did NOT increase (no changes means no misses).
11. Optionally check logs for "sync cursor hit" debug message.

**test_recon03_version_compat.sh (RECON-03: version byte forward compat):**

Since all wire traffic is AEAD-encrypted after handshake, we cannot inject a malformed ReconcileInit with a wrong version byte from outside. The version byte rejection (decode_reconcile_init returns nullopt for version != 0x01) is verified by unit tests.

The Docker integration test verifies the broader robustness property: the sync responder handles any malformed/failed decode via `continue` (no crash, no state corruption). Test flow:

1. Start 2-node topology. Wait healthy.
2. Ingest 50 blobs. Wait for sync (50 blobs on both nodes).
3. Use netshoot container to send raw garbage bytes to node1's port 4200 via socat/netcat. This tests that the connection handler does not crash on garbage input:
   ```bash
   docker run --rm --network chromatindb-test_test-net \
       nicolaka/netshoot \
       bash -c 'echo "GARBAGE_VERSION_0xFF_INVALID_RECONCILE" | nc -w 2 172.28.0.2 4200'
   ```
4. Wait 10s. Verify node1 is still healthy (TCP probe works).
5. Verify node1 still has the correct blob count (no state corruption).
6. Ingest 10 more blobs. Wait for sync to node2 (60 blobs). This proves sync still works after receiving garbage.
7. Document in test output: "Version byte decode rejection verified by unit test decode_reconcile_init; integration test confirms node resilience to invalid input on port 4200"

**test_net06_late_joiner.sh (NET-06: late-joiner at 10K blobs, multi-namespace):**

Uses manual container creation (like ACL tests) for a 3-node topology where the late-joiner arrives after data is established.

Topology:
- Create network: `docker network create --driver bridge --subnet 172.32.0.0/16 chromatindb-late-test-net`
- Node1: 172.32.0.2 (seed)
- Node2: 172.32.0.3 (bootstrap to node1)
- Node3: 172.32.0.4 (late joiner -- started later, bootstraps to node1)
- Use 172.32.0.0/16 to avoid conflicts with other test networks
- Override NETWORK for run_loadgen: `NETWORK="chromatindb-late-test-net"`
- All configs: sync_interval_seconds=5, full_resync_interval=0, inactivity_timeout_seconds=0

Test flow:
1. Start node1 and node2. Wait healthy.
2. Generate 3 different identities for multi-namespace testing. Write config files, then ingest:
   ```bash
   # Identity 1: 3334 blobs
   TMPDIR1=$(mktemp -d)
   run_loadgen 172.32.0.2 --count 1 --size 256 --ttl 3600 --identity-save "$TMPDIR1/id1"
   # Discard the 1 blob used for identity discovery, or count it in the total
   run_loadgen 172.32.0.2 --count 3333 --size 256 --rate 200 --ttl 3600 --identity-file "$TMPDIR1/id1"

   # Identity 2: 3333 blobs
   TMPDIR2=$(mktemp -d)
   run_loadgen 172.32.0.2 --count 1 --size 256 --ttl 3600 --identity-save "$TMPDIR2/id2"
   run_loadgen 172.32.0.2 --count 3332 --size 256 --rate 200 --ttl 3600 --identity-file "$TMPDIR2/id2"

   # Identity 3: 3333 blobs
   TMPDIR3=$(mktemp -d)
   run_loadgen 172.32.0.2 --count 1 --size 256 --ttl 3600 --identity-save "$TMPDIR3/id3"
   run_loadgen 172.32.0.2 --count 3332 --size 256 --rate 200 --ttl 3600 --identity-file "$TMPDIR3/id3"
   ```
   Total: 3334 + 3333 + 3333 = 10000 blobs across 3 namespaces.
   NOTE: The --identity-save/--identity-file flags may work differently. The loadgen run that saves the identity also injects blobs. So the first run with --identity-save already creates blobs in that namespace. Verify loadgen behavior. If --identity-save saves the identity AND injects --count blobs, then:
   ```bash
   run_loadgen 172.32.0.2 --count 3334 --size 256 --rate 200 --ttl 3600 --identity-save "$TMPDIR1/id1"
   run_loadgen 172.32.0.2 --count 3333 --size 256 --rate 200 --ttl 3600 --identity-save "$TMPDIR2/id2"
   run_loadgen 172.32.0.2 --count 3333 --size 256 --rate 200 --ttl 3600 --identity-save "$TMPDIR3/id3"
   ```
   This creates 10000 blobs total across 3 namespaces. The --identity-save creates a fresh identity and saves it. Subsequent --identity-file reuses it.

3. Wait for sync to node2 (10000 blobs, 300s timeout).
4. Verify node2 has 10000 blobs.
5. Start node3 (late joiner) with bootstrap to node1. Wait healthy.
6. Wait for sync to node3 (10000 blobs, 300s timeout). The late joiner must catch up on all 3 namespaces.
7. Verify node3 has 10000 blobs.
8. Compare blob counts across all 3 nodes -- they must match.
9. Verify via SIGUSR1 metrics that node3's cursor_misses shows it synced all namespaces (3 namespaces, so at least 3 cursor_misses on initial sync).

Cleanup: Remove all containers, network, temp dirs in trap.
