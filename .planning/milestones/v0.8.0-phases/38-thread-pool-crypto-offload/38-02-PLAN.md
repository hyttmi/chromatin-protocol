<objective>
Convert BlobEngine::ingest() and delete_blob() to coroutines that offload CPU-bound crypto (SHA3-256 hashing, ML-DSA-87 verification) to the thread pool, then update all callers (PeerManager, SyncProtocol) to co_await the new async methods.

Purpose: The event loop never blocks on ML-DSA-87 verify or SHA3-256 hash during blob ingest/delete — other connections continue processing I/O while crypto runs on worker threads.
Output: Async BlobEngine with two-dispatch ingest pattern (blob_hash first, build_signing_input+verify bundled second). All callers updated. All tests pass.
</objective>

**engine.cpp ingest() — two-dispatch pattern per CONTEXT.md:**

The ingest method keeps its existing validation pipeline order but offloads the two expensive crypto steps:

1. Steps 0, 0b, 1: Size check, capacity check, structural checks — remain synchronous (cheap integer comparisons, no offload needed).

2. Step 2: Namespace ownership check — `crypto::sha3_256(blob.pubkey)` offloaded to pool. This is a small hash (~2592 bytes) but we offload ALL sha3_256 per the user's locked decision for a uniform model. Use the offload helper:
   ```cpp
   auto derived_ns = co_await crypto::offload(pool_, co_await asio::this_coro::executor, [&blob]() {
       return crypto::sha3_256(blob.pubkey);
   });
   ```
   Continue with ownership/delegation checks synchronously (they are just comparisons and storage lookups — cheap).

3. Steps 2a: Quota check — synchronous (storage lookup, no crypto).

4. Step 2.5: **First dispatch** — blob_hash. Encode blob (synchronous, fast), then offload the SHA3-256 hash:
   ```cpp
   auto encoded = wire::encode_blob(blob);
   auto content_hash = co_await crypto::offload(pool_, co_await asio::this_coro::executor, [&encoded]() {
       return wire::blob_hash(encoded);
   });
   ```

5. Dedup check: `storage_.has_blob()` — synchronous on event loop. If duplicate, return early (no second dispatch). **This is the key optimization: duplicates pay only ONE thread pool round-trip.**

6. Step 3: **Second dispatch** — build_signing_input + verify BUNDLED:
   ```cpp
   auto [signing_input, verify_ok] = co_await crypto::offload(pool_, co_await asio::this_coro::executor, [&blob]() {
       auto si = wire::build_signing_input(blob.namespace_id, blob.data, blob.ttl, blob.timestamp);
       bool ok = crypto::Signer::verify(si, blob.signature, blob.pubkey);
       return std::pair{si, ok};
   });
   if (!verify_ok) {
       // ... rejection ...
   }
   ```

7. Steps 3.5, 4: Tombstone handling, store — synchronous (storage ops, must run on event loop since Storage is NOT thread-safe).

**CRITICAL SAFETY RULES:**
- Storage operations (has_blob, store_blob, delete_blob_data, has_tombstone_for, has_valid_delegation, get_namespace_quota) MUST stay on the event loop — Storage is not thread-safe.
- AEAD state is in Connection, not BlobEngine — no risk here, but verify no encrypt/decrypt touches.
- Captures in offload lambdas: use `[&blob]` or `[&encoded]` — the blob/encoded are alive on the coroutine frame throughout the co_await. Do NOT capture `this` in the offload lambda.

**engine.cpp delete_blob() — single dispatch:**

delete_blob has the same pipeline but simpler (no dedup, no blob_hash):

1. Step 1: Structural checks — synchronous.
2. Step 2: Namespace ownership — offload sha3_256(pubkey), same pattern as ingest.
3. Step 3: **Single dispatch** — build_signing_input + verify bundled (same pattern as ingest Step 3).
4. Steps 4-5: Delete target, store tombstone — synchronous.

**NOTE on the namespace sha3_256 offload:** Per the locked decision "ALL sha3_256() calls offloaded — including small ones like pubkey-to-namespace derivation (uniform model)." The pubkey is 2592 bytes — the SHA3 itself is very fast, but we offload it anyway for uniformity. If this causes measurable overhead in benchmarks (Phase 41), it can be revisited with the PERF-11 future requirement (per-size threshold), but that is explicitly deferred beyond v1.0.0.

if (type == wire::TransportMsgType_Delete) {
    asio::co_spawn(ioc_, [this, conn, payload = std::move(payload)]() -> asio::awaitable<void> {
        try {
            auto blob = wire::decode_blob(payload);
            // ... namespace filter check ...
            auto result = co_await engine_.delete_blob(blob);
            if (result.accepted && result.ack.has_value()) {
                // Build and send DeleteAck — can now co_await directly
                auto ack = result.ack.value();
                std::vector<uint8_t> ack_payload(41);
                std::memcpy(ack_payload.data(), ack.blob_hash.data(), 32);
                for (int i = 7; i >= 0; --i) {
                    ack_payload[32 + (7 - i)] = static_cast<uint8_t>(ack.seq_num >> (i * 8));
                }
                ack_payload[40] = (ack.status == engine::IngestStatus::stored) ? 0 : 1;
                co_await conn->send_message(wire::TransportMsgType_DeleteAck,

            record_strike(conn, e.what());
