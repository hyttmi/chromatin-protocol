    **Step 4: Add `reset_topology_fastsync()` helper.** Add a new helper function in `run-benchmark.sh` (near the existing `reset_topology()`) that does `docker compose -f $COMPOSE_FILE -f "$SCRIPT_DIR/docker-compose.fastsync.yml" -p chromatindb down -v` then `up -d --wait` with the same fallback polling pattern as `reset_topology()`.     **Step 5: Add `run_scenario_reconciliation_scaling()` function.** Add after the existing tombstone scenarios, before `main()`. Follow the exact pattern from the research:

    The scenario has two phases:
    1. **Preload phase:** Reset topology with fast-sync configs via `reset_topology_fastsync()`. Ingest 1000 blobs (1 KiB each) to node1 at rate 100. Wait for node2 convergence at 1000 (timeout 300s). This establishes a large shared namespace.
    2. **Delta phase:** Add 10 more blobs (1 KiB) to node1. Immediately start timing. Wait for node2 convergence at 1010 (timeout 120s). The convergence time for 10 blobs on a 1000-blob namespace should be similar to syncing 10 blobs on an empty namespace (O(diff) behavior).

    Use constants at the top of the function: `local LARGE_COUNT=1000` and `local DELTA_COUNT=10`.

    Capture stats before preload ("pre" "reconcile-scaling") and after delta convergence ("post" "reconcile-scaling").

    Write result JSON to `$RESULTS_DIR/scenario-reconcile-scaling.json` with fields: `scenario`, `preload_blobs`, `delta_blobs`, `preload_convergence_ms`, `delta_sync_ms`, `sync_interval_seconds` (2).

    The key metric is `delta_sync_ms`. With sync_interval=2s, the delta should converge within ~1-2 intervals (~2000-4000ms) regardless of the 1000-blob namespace size. If it were O(N), the sync would take significantly longer due to hash list exchange overhead.

    Log clearly at each phase. Use the standard `capture_stats` pattern.

    **Important:** After delta convergence measurement, clean up by calling `docker compose ... down -v` to remove the fastsync topology. The next scenario will use `reset_topology()` which starts the standard config.

    **Step 6: Wire into main().** Add `run_scenario_reconciliation_scaling` call in `main()` BEFORE the `generate_report` call, after the tombstone scenarios. Add a comment `# Reconciliation scenario (Plan 41)`.
