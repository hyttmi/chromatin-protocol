<objective>
Implement the lightweight handshake protocol for trusted connections, branching do_handshake() between PQ and lightweight paths based on IP trust.

Purpose: Connections between localhost peers or configured trusted peers skip the expensive ML-KEM-1024 key exchange while maintaining mutual identity verification and encrypted transport. Non-trusted connections use the existing PQ handshake with zero changes.

Output: Connection::do_handshake() branches based on trust. Lightweight path exchanges TrustedHello messages (nonce + signing pubkey), derives session keys via HKDF, and both sides verify each other's identity. Mismatch falls back to PQ.
</objective>

Protocol design (from CONTEXT.md decisions + RESEARCH.md analysis):
```
INITIATOR-FIRST DESIGN (avoids deadlock):

LIGHTWEIGHT (both trust):
  Initiator -> Responder: TrustedHello (nonce_i, signing_pk_i)
  Responder reads TrustedHello, checks own trust -> trusted
  Responder -> Initiator: TrustedHello (nonce_r, signing_pk_r)
  Both derive: HKDF(nonce_i||nonce_r, signing_pk_i||signing_pk_r) -> SessionKeys
  Begin encrypted communication immediately (no separate auth step -- pubkeys in TrustedHello)

PQ (initiator doesn't trust):
  Initiator -> Responder: KemPubkey (kem_pk, signing_pk_i)   [unchanged]
  Responder -> Initiator: KemCiphertext (kem_ct, signing_pk_r)
  Initiator -> Responder: AuthSignature (encrypted)
  Responder -> Initiator: AuthSignature (encrypted)

MISMATCH (initiator trusts, responder doesn't):
  Initiator -> Responder: TrustedHello (nonce_i, signing_pk_i)
  Responder reads TrustedHello, checks own trust -> NOT trusted
  Responder -> Initiator: PQRequired (empty payload)
  Initiator receives PQRequired, generates KEM, sends KemPubkey
  Normal PQ handshake continues (KemCiphertext, AuthSig, AuthSig)

TrustedHello payload: [nonce:32][signing_pubkey:2592] = 2624 bytes
PQRequired payload: empty (msg type is the signal)

Key derivation (lightweight):
  IKM = nonce_initiator || nonce_responder  (64 bytes)
  Salt = signing_pk_initiator || signing_pk_responder  (5184 bytes)
  PRK = HKDF-Extract(salt=Salt, ikm=IKM)
  init_to_resp = HKDF-Expand(PRK, "chromatin-init-to-resp-v1", 32)
  resp_to_init = HKDF-Expand(PRK, "chromatin-resp-to-init-v1", 32)
  fingerprint = SHA3-256(IKM || Salt)

"Server-driven" preserved: responder has final say — if it doesn't trust, it sends PQRequired and forces PQ.
```

    1. **db/net/connection.cpp**: Rewrite `do_handshake()` to branch between PQ and lightweight paths. Add `#include <sodium.h>` for `randombytes_buf`.

       **New do_handshake() structure:**
       ```cpp
       asio::awaitable<bool> Connection::do_handshake() {
           // Determine trust based on remote IP
           asio::error_code ec;
           auto remote_ep = socket_.remote_endpoint(ec);
           bool peer_is_trusted = !ec && trust_check_ && trust_check_(remote_ep.address());

           if (is_initiator_) {
               if (peer_is_trusted) {
                   co_return co_await do_handshake_initiator_trusted();
               } else {
                   co_return co_await do_handshake_initiator_pq();
               }
           } else {
               // Responder: read first message, then decide
               auto first_msg = co_await recv_raw();
               if (!first_msg) co_return false;

               auto decoded = TransportCodec::decode(*first_msg);
               if (!decoded) co_return false;

               if (decoded->type == wire::TransportMsgType_TrustedHello) {
                   if (peer_is_trusted) {
                       co_return co_await do_handshake_responder_trusted(decoded->payload);
                   } else {
                       // Mismatch: initiator trusts us, we don't trust them -> force PQ
                       co_return co_await do_handshake_responder_pq_fallback(decoded->payload);
                   }
               } else if (decoded->type == wire::TransportMsgType_KemPubkey) {
                   co_return co_await do_handshake_responder_pq(*first_msg);
               } else {
                   spdlog::warn("handshake: unexpected first message type");
                   co_return false;
               }
           }
       }
       ```

    2. **Add private helper methods to Connection** (declared in connection.h, implemented in connection.cpp):

       **do_handshake_initiator_trusted():**
       - Generate 32-byte nonce via `randombytes_buf()`
       - Build TrustedHello payload: [nonce:32][signing_pubkey:2592]
       - Encode as TransportMsgType_TrustedHello, send via send_raw
       - Read response via recv_raw
       - If response is TrustedHello: parse peer nonce + signing_pk, call derive_lightweight_session_keys, set session_keys_, set peer_pubkey_, log "handshake complete (initiator, lightweight)"
       - If response is PQRequired: mismatch fallback — proceed as PQ initiator (generate KEM, send KemPubkey, receive KemCiphertext, exchange auth signatures). This adds 1 round trip but avoids connection failure.
       - Return authenticated_ = true on success

       **do_handshake_initiator_pq():**
       - Extract the existing PQ initiator handshake code from the current do_handshake() (lines 124-209). This is the unchanged PQ path. Log "handshake complete (initiator, PQ)".

       **do_handshake_responder_trusted(payload):**
       - Parse initiator's TrustedHello payload: [nonce_i:32][signing_pk_i:2592]
       - Validate payload size == 32 + 2592
       - Generate own 32-byte nonce via `randombytes_buf()`
       - Build our TrustedHello: [nonce_r:32][signing_pk_r:2592]
       - Send via send_raw
       - Call derive_lightweight_session_keys(nonce_i, nonce_r, signing_pk_i, signing_pk_r, false)
       - Set session_keys_, peer_pubkey_ = initiator's signing_pk
       - Log "handshake complete (responder, lightweight)"
       - Return authenticated_ = true

       **do_handshake_responder_pq_fallback(payload):**
       - Initiator sent TrustedHello but we don't trust them
       - Send PQRequired (empty payload) via send_raw
       - Wait for KemPubkey via recv_raw
       - Process as standard PQ responder (existing HandshakeResponder logic)
       - Log "handshake complete (responder, PQ, trust mismatch)"

       **do_handshake_responder_pq(first_msg):**
       - First message already received (KemPubkey). Pass to HandshakeResponder::receive_kem_pubkey()
       - Continue with existing PQ responder logic (auth exchange)
       - Log "handshake complete (responder, PQ)"

       **IMPORTANT:** The current do_handshake() inline implementation for both initiator PQ and responder PQ must be refactored into the helper methods. The connection.cpp send_encrypted/recv_encrypted for auth exchange in PQ path can be reused as-is. The auth message encode/decode logic should remain inline (it's Connection-level, not in HandshakeInitiator/HandshakeResponder state machines).

The `do_handshake_responder_pq(first_msg)` method receives the FULL raw first message bytes and passes them directly to `HandshakeResponder::receive_kem_pubkey()`.
