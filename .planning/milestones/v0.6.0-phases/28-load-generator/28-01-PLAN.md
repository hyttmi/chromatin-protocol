```cpp
static chromatindb::wire::BlobData make_signed_blob(
    const chromatindb::identity::NodeIdentity& id,
    const std::vector<uint8_t>& data,
    uint32_t ttl, uint64_t timestamp)
{
    chromatindb::wire::BlobData blob;
    std::memcpy(blob.namespace_id.data(), id.namespace_id().data(), 32);
    blob.pubkey.assign(id.public_key().begin(), id.public_key().end());
    blob.data = data;
    blob.ttl = ttl;
    blob.timestamp = timestamp;
    auto signing_input = chromatindb::wire::build_signing_input(
        blob.namespace_id, blob.data, blob.ttl, blob.timestamp);
    blob.signature = id.sign(signing_input);
    return blob;
}
```

<task type="auto">
  <name>Task 1: Create loadgen binary with full protocol-compliant load generation</name>
  <files>loadgen/loadgen_main.cpp, CMakeLists.txt</files>
  <action>
Create `loadgen/` directory and `loadgen/loadgen_main.cpp` implementing the complete load generator.

**CLI interface** (use simple argc/argv parsing, same pattern as main daemon):
- `--target HOST:PORT` (required) - target node address
- `--count N` (default: 100) - total blobs to send
- `--rate N` (default: 10) - blobs per second
- `--size N` (default: 1024) - blob data size in bytes (ignored in mixed mode)
- `--mixed` (flag) - enable mixed-size mode: 70% 1 KiB, 20% 100 KiB, 10% 1 MiB
- `--ttl N` (default: 3600) - blob TTL in seconds
- `--drain-timeout N` (default: 5) - seconds to wait for notification ACKs after last send

**Architecture** (single file, all in anonymous namespace except main):

1. **Blob pool pre-generation:** Before the timed run, pre-generate random data buffers for each size class (1 KiB, 100 KiB, 1 MiB -- one buffer each, reused with unique timestamps). For each blob to send, pre-compute the size class (using a seeded mt19937 and the 70/20/10 distribution for mixed mode, or fixed size otherwise). Do NOT pre-sign all blobs -- sign during send to get unique timestamps. Pre-signing all would use excessive memory for large blob counts.

2. **Connection setup:** Generate a fresh `NodeIdentity` (loadgen is its own peer). Resolve target address, create TCP socket, connect, create `Connection::create_outbound()`. Set up `on_message` callback for Notification matching. Set up `on_close` callback. The `on_ready` callback should: (a) subscribe to own namespace via `PeerManager::encode_namespace_list`, (b) start the send coroutine via `asio::co_spawn`.

3. **Notification ACK matching:** Maintain an `std::unordered_map<std::string, std::chrono::steady_clock::time_point>` keyed by hex-encoded blob_hash. When a Data message is sent, compute the blob_hash of the encoded FlatBuffer bytes and store the *scheduled* send time. When a Notification (type 22) is received, parse the 77-byte payload, extract blob_hash (bytes 32-63), look up in the map, compute latency = now - scheduled_time. Store latency in a `std::vector<double>` (milliseconds).

4. **Timer-driven scheduling (anti-coordinated-omission):** Pre-compute all scheduled send times at startup: `scheduled[i] = start_time + i * interval` where `interval = 1s / rate`. Use `asio::steady_timer` that expires_at each scheduled time. The loop:
   ```
   for i in 0..count:
     timer.expires_at(scheduled[i])
     co_await timer.async_wait(use_nothrow)
     // construct blob with current timestamp, sign, encode, send
     // record blob_hash -> scheduled[i] in pending map
   ```
   This ensures that if a send is slow, the next timer fires immediately (catching up), and latency is measured from the *intended* time.

5. **Send coroutine:** For each blob: pick size class, construct BlobData with `make_signed_blob` pattern (memcpy namespace_id, assign pubkey, set data/ttl/timestamp, build_signing_input, sign). Encode with `wire::encode_blob()`. Compute `wire::blob_hash()` on the encoded bytes. Store hash->scheduled_time. Send via `conn->send_message(TransportMsgType_Data, encoded)`. On send failure, increment error counter and continue (don't abort). If `StorageFull` was received, stop sending and report partial results.

6. **Drain phase:** After the last send, wait up to `drain_timeout` seconds for remaining notifications. Use a steady_timer for the timeout. When all expected notifications are received OR timeout fires, proceed to stats.

7. **Statistics and JSON output:** Sort the latencies vector. Compute:
   - `total_blobs` = count (or actual sent if stopped early)
   - `duration_sec` = wall-clock time from first send to last send
   - `blobs_per_sec` = total_blobs / duration_sec
   - `mib_per_sec` = total_bytes_sent / (1024*1024) / duration_sec
   - p50/p95/p99/min/max/mean from sorted latencies vector (use nth_element for efficiency, not full sort)
   - `blob_sizes` breakdown (count per size class)
   - `errors` = send failures + missing notifications
   - `notifications_received` vs `notifications_expected`

   Output JSON to stdout using nlohmann/json. Use spdlog for progress/debug logging to stderr (so JSON on stdout is clean).

8. **StorageFull handling:** In the on_message callback, if type == TransportMsgType_StorageFull, set a flag. The send coroutine checks this flag before each send and stops if set. Log a warning via spdlog.

**Important implementation notes:**
- Use `asio::co_spawn(ioc, conn->run(), asio::detached)` to start the connection message loop. Then `co_spawn` the send coroutine separately. Both share the same io_context.
- `ioc.run()` drives everything. After the send coroutine and drain complete, call `conn->close()` and `ioc.stop()`.
- Use `std::chrono::steady_clock` everywhere for timing. Never system_clock.
- Use the `chromatindb::net::use_nothrow` completion token for all async operations.
- The loadgen needs its own namespace to subscribe to. `NodeIdentity::generate()` creates a fresh keypair each run -- this is fine, the namespace is ephemeral.
- For the namespace_id array needed by encode_namespace_list, copy from `identity.namespace_id()` (which returns `span<const uint8_t, 32>`) into a `std::array<uint8_t, 32>`.
