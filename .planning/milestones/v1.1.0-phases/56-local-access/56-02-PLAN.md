Test cases:

1. `TEST_CASE("UdsAcceptor binds and accepts connection", "[uds]")`:
   - Generate two identities (server and client).
   - Create UdsAcceptor on a temp socket path.
   - Set on_connected callback that captures the peer pubkey.
   - Start the acceptor.
   - Create a `local::stream_protocol::socket`, async_connect to the path.
   - Move the client socket into a Connection via `Connection::create_uds_outbound` (created in Plan 01 Task 1 — the client-side/initiator UDS factory).
   - Run both connections (co_spawn), verifying handshake completes.

   Test structure using `ioc.run_for(std::chrono::seconds(5))` pattern:
   ```cpp
   asio::io_context ioc;
   auto server_id = identity::NodeIdentity::generate();
   auto client_id = identity::NodeIdentity::generate();
   auto path = temp_socket_path();

   net::UdsAcceptor acceptor(path, server_id, ioc);

   bool connected = false;
   net::Connection::Ptr server_conn;
   acceptor.set_on_connected([&](net::Connection::Ptr c) {
       connected = true;
       server_conn = c;
   });
   acceptor.start();

   // Client connects
   asio::local::stream_protocol::socket client_sock(ioc);
   client_sock.connect(asio::local::stream_protocol::endpoint(path));
   auto client_conn = net::Connection::create_uds_outbound(std::move(client_sock), client_id);

   asio::co_spawn(ioc, [&]() -> asio::awaitable<void> {
       co_await client_conn->run();
   }, asio::detached);

   ioc.run_for(std::chrono::seconds(3));

   REQUIRE(connected);
   REQUIRE(client_conn->is_authenticated());
   REQUIRE(client_conn->is_uds());

   // Cleanup
   acceptor.stop();
   std::filesystem::remove(path);
   ```

**PROTOCOL.md update:**
- Read current PROTOCOL.md, find the Transport section.
- Add a new subsection "## Unix Domain Socket Transport" (or similar, consistent with existing doc style).
- Content:
  - UDS is an alternative transport for local process communication.
  - Enabled by setting `uds_path` in config (absolute path, empty = disabled).
  - UDS connections use the same wire protocol as TCP (length-prefixed frames, same message types).
  - UDS connections skip PQ key exchange — they always use the TrustedHello handshake path (local = trusted).
  - Session keys are derived via lightweight HKDF (same as trusted TCP peers).
  - UDS connections receive identical enforcement: ACL gating, rate limiting, quotas, max_peers.
  - Socket permissions: 0660 (owner + group). Stale socket unlinked on startup.
  - NOT SIGHUP-reloadable: changing uds_path requires restart.
