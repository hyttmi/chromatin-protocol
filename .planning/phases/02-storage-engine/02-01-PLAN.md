    set(MDBX_BUILD_CXX ON CACHE BOOL "" FORCE)
    set(MDBX_BUILD_TOOLS OFF CACHE BOOL "" FORCE)
    set(MDBX_ENABLE_TESTS OFF CACHE BOOL "" FORCE)

    **storage.h:** Create in `src/storage/` with:
    - `enum class StoreResult { Stored, Duplicate, Error }` -- return type for store operations
    - `using Clock = std::function<uint64_t()>` -- injectable clock for testable expiry
    - `uint64_t system_clock_seconds()` -- default clock implementation
    - `class Storage` in `chromatin::storage` namespace:
      - Constructor: `Storage(const std::string& data_dir, Clock clock = system_clock_seconds)`
      - Move constructor and move assignment (delete copy)
      - `StoreResult store_blob(const wire::BlobData& blob)` -- store a blob, returns Stored/Duplicate/Error
      - `std::optional<wire::BlobData> get_blob(std::span<const uint8_t, 32> ns, std::span<const uint8_t, 32> hash)` -- retrieve by namespace + hash
      - `bool has_blob(std::span<const uint8_t, 32> ns, std::span<const uint8_t, 32> hash)` -- existence check
      - `std::vector<wire::BlobData> get_blobs_by_seq(std::span<const uint8_t, 32> ns, uint64_t since_seq)` -- range query (stub returning empty, implemented in Plan 02)
      - `size_t run_expiry_scan()` -- expiry scan (stub returning 0, implemented in Plan 03)
    - Private members: `mdbx::env_managed env_`, `mdbx::map_handle blobs_map_`, `mdbx::map_handle seq_map_`, `mdbx::map_handle expiry_map_`, `Clock clock_`
    - Private helpers: `make_blob_key()`, `make_seq_key()`, `make_expiry_key()`, `encode_be_u64()`, `decode_be_u64()`

    **storage.cpp:** Implement:
    - Constructor: Create data_dir if missing (`std::filesystem::create_directories`). Open libmdbx env with `mdbx::env_managed`. Set geometry with large upper bound (e.g., 64GB virtual, start at 1MB, grow 1MB at a time). Use `mdbx::env::mode::write_mapped_io` for performance. Set `max_maps(3)`. Open all 3 DBI handles in a single startup write transaction using `txn.create_map("blobs", ...)`, `txn.create_map("sequence", ...)`, `txn.create_map("expiry", ...)`.
    - `store_blob()`: Encode blob via `wire::encode_blob()`, compute hash via `wire::blob_hash()`, build blob key, start write txn, check dedup with `txn.get()`, if exists return Duplicate. Otherwise: put blob, assign seq_num (cursor to last seq entry for namespace, increment), put seq entry, put expiry entry (if TTL > 0, expiry_time = timestamp + ttl), commit, return Stored. Log via spdlog on store/duplicate.
    - `get_blob()`: Start read txn, get by blob key, if found decode via `wire::decode_blob()`, return BlobData. If not found, return nullopt.
    - `has_blob()`: Start read txn, get by blob key, return whether found.
    - Key helpers: `make_blob_key(ns, hash) -> std::array<uint8_t, 64>`, `make_seq_key(ns, seq) -> std::array<uint8_t, 40>`, `make_expiry_key(expiry_ts, hash) -> std::array<uint8_t, 40>`. Use big-endian encoding for uint64 fields.

    **IMPORTANT libmdbx C++ API notes:**
    - `mdbx::slice` is constructed from `(void*, size_t)` -- use `.data()` and `.size()` from arrays
    - `txn.get()` returns a `mdbx::slice` that is empty/falsy if key not found
    - `txn.put()` accepts key and value as `mdbx::slice`
    - `txn.create_map()` is used to create/open a named sub-database
    - `cursor.to_key_lesser_or_equal()` or `cursor.lower_bound()` for seeking
    - Environment creation: use `mdbx::env_managed()` default-constructed, then call `env_.operate(path, create_params, operate_params)` or similar open method

    Write TDD tests FIRST with a test fixture that creates a temporary directory (use `std::filesystem::temp_directory_path() / "chromatindb_test_XXXXXX"` with unique naming), constructs Storage, and cleans up after. Tests:
    - Open/close without error
    - Store a blob and retrieve it -- verify all fields match
    - Store same blob twice -- second call returns Duplicate
    - has_blob returns true for stored blob, false for unknown blob
    - get_blob returns nullopt for unknown blob
    - Data directory created automatically
    - Crash recovery: store blob, destroy Storage without calling close explicitly, reopen from same path, verify blob is still there
