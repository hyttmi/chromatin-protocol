Subscription aggregation with reference counting and notification fan-out from node to WebSocket clients.

Purpose: Clients subscribe to namespace changes via JSON. The relay aggregates subscriptions so the node sees one Subscribe per namespace regardless of client count. Notifications from the node are translated once and fanned out to all subscribed clients. StorageFull/QuotaExceeded broadcasts reach all connected sessions.

Output: SubscriptionTracker class, WsSession interception for Subscribe/Unsubscribe, UdsMultiplexer notification fan-out, comprehensive unit tests.
