1. Modify `db/peer/peer_manager.h`:
   - Add `#include "db/acl/access_control.h"` and `#include "db/crypto/hash.h"`

   - Add `const acl::AccessControl& acl() const { return acl_; }` public accessor (for testing)

2. Modify `db/peer/peer_manager.cpp`:
   - Update constructor to accept and store `acl` reference
   - In `on_peer_connected()`, add ACL check as the FIRST thing (before any peer tracking):
     ```cpp
     // Derive peer namespace hash
     auto peer_ns = crypto::sha3_256(conn->peer_pubkey());
     std::array<uint8_t, 32> peer_ns_arr;
     std::copy(peer_ns.begin(), peer_ns.end(), peer_ns_arr.begin());

     if (!acl_.is_allowed(peer_ns_arr)) {
         auto full_hex = to_hex(std::span<const uint8_t>(peer_ns_arr), 32);
         spdlog::warn("access denied: namespace={} ip={}", full_hex, conn->remote_address());
         conn->close();  // Silent close, no goodbye
         return;
     }
     ```
   - Modify the `to_hex` helper at the top of the file: change default `max_len` parameter from 8 to 32 (or add overload), since ACL logging needs full 64-char hex. Actually, better: add a second call with max_len=32 for ACL logging. Keep existing calls unchanged.

3. Guard ALL PEX code paths with `acl_.is_closed_mode()`:
   - In `pex_timer_loop()`: at the start of the while loop body, after the timer wait, add `if (acl_.is_closed_mode()) continue;` to skip the PEX round entirely.
   - In `handle_pex_as_responder()`: at the very start (before any sync_inbox reads), add: `if (acl_.is_closed_mode()) { peer->syncing = false; co_return; }`. This makes the node ignore incoming PeerListRequest in closed mode.
   - In `run_sync_with_peer()`: guard the inline PEX block (the block that sends PeerListRequest after sync) with `if (!acl_.is_closed_mode()) { ... }`. The PEX block starts with the comment `// PEX exchange: send PeerListRequest`.
   - In `handle_sync_as_responder()`: guard the inline PEX block (the block that waits for PeerListRequest after sync) with `if (!acl_.is_closed_mode()) { ... }`. The PEX block starts with the comment `// PEX exchange: wait for PeerListRequest`.

4. In `start()`, log access control mode:
   ```cpp
   if (acl_.is_closed_mode()) {
       spdlog::info("access control: closed mode ({} allowed keys)", acl_.allowed_count());
   } else {
       spdlog::info("access control: open mode");
   }
   ```

5. Modify `db/main.cpp`:
   - Add `#include "db/acl/access_control.h"` and `#include "db/crypto/hash.h"`
   - In `cmd_run()`, after creating `identity`, construct AccessControl:
     ```cpp
     chromatindb::acl::AccessControl acl(config.allowed_keys, identity.namespace_id());
     ```
   - Pass `acl` to PeerManager constructor:
     ```cpp
     chromatindb::peer::PeerManager pm(config, identity, engine, storage, ioc, acl);
     ```
