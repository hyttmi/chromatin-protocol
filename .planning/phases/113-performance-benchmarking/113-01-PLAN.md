Create `tools/relay_perf_test.sh` by adapting `tools/relay_asan_test.sh` for Release benchmarking. The script follows the same structural pattern (setup, cleanup trap, build, config gen, node start, relay start, benchmark, summary) but differs in these specific ways per locked decisions D-01 through D-07:

**Header and setup (per D-01):**
- Shebang: `#!/usr/bin/env bash`
- Comment block: "Performance benchmark harness for chromatindb_relay"
- `set -euo pipefail`
- Variables: SCRIPT_DIR, REPO_ROOT, BUILD_DIR=`$REPO_ROOT/build-release`, WORK_DIR="", NODE_PID="", RELAY_PID="", SKIP_BUILD=false
- Parse `--skip-build` flag from `$@`
- `log()` function: `echo ">>> $*"`
- NO pass/fail/RESULTS tracking (per D-07 -- no thresholds, no pass/fail)

**Cleanup trap (same pattern as ASAN script):**
- `cleanup()` function: kill RELAY_PID with TERM then KILL, kill NODE_PID with TERM then KILL, rm -rf WORK_DIR
- `trap cleanup EXIT`
- `WORK_DIR=$(mktemp -d /tmp/relay_perf_test.XXXXXX)`

**Dependency check (early guard):**
```bash
python3 -c "import httpx, flatbuffers, oqs" 2>/dev/null || {
    echo "ERROR: Missing Python dependencies (httpx, flatbuffers, oqs)"
    exit 1
}
```

**Build section (per D-02 -- Release, no sanitizer):**
```bash
if [ "$SKIP_BUILD" = true ] && [ -x "$BUILD_DIR/chromatindb" ] && [ -x "$BUILD_DIR/chromatindb_relay" ]; then
    log "Skipping build (--skip-build, binaries exist)"
else
    log "Building Release binaries..."
    cmake -B "$BUILD_DIR" -S "$REPO_ROOT" \
        -DCMAKE_BUILD_TYPE=Release \
        -DBUILD_TESTING=OFF 2>&1 | tail -5
    cmake --build "$BUILD_DIR" --target chromatindb chromatindb_relay -j"$(nproc)" 2>&1 | tail -10
fi
```
- Verify binaries exist: `[ -x "$BUILD_DIR/chromatindb" ] && [ -x "$BUILD_DIR/chromatindb_relay" ]`
- Exit 1 with message if not found.

**Config generation:**
- Node config: `$WORK_DIR/node.json` with `bind_address: "127.0.0.1:4280"`, `uds_path: "$WORK_DIR/chromatindb.sock"`, `log_level: "info"`. Use port 4280 (different from ASAN's 4290 for parallel safety).
- Relay config: `$WORK_DIR/relay.json` with `bind_address: "127.0.0.1"`, `bind_port: 4281`, `uds_path: "$WORK_DIR/chromatindb.sock"`, `identity_key_path: "$WORK_DIR/relay.key"`, `log_level: "info"`, `rate_limit_messages_per_sec: 0`, `request_timeout_seconds: 0`, `max_blob_size_bytes: 0`, `max_connections: 1024`
- Use port 4281 for relay (different from ASAN's 4291).

**Node startup (same pattern as ASAN script, no ASAN_OPTIONS/LSAN_OPTIONS):**
```bash
"$BUILD_DIR/chromatindb" run \
    --config "$WORK_DIR/node.json" \
    --data-dir "$WORK_DIR/node_data" \
    >"$WORK_DIR/node.log" 2>&1 &
NODE_PID=$!
```
- UDS wait loop: 30s timeout, check `-S "$WORK_DIR/chromatindb.sock"`, check `kill -0 "$NODE_PID"`, cat node.log on failure.

**Relay startup (same pattern, no ASAN env vars):**
```bash
"$BUILD_DIR/chromatindb_relay" \
    --config "$WORK_DIR/relay.json" \
    >"$WORK_DIR/relay.log" 2>&1 &
RELAY_PID=$!
```
- TCP wait loop: 15s timeout, check `/dev/tcp/127.0.0.1/$RELAY_PORT`, cat relay.log on failure.
- `RELAY_PORT=4281`

**Run benchmark (per D-03, D-04 -- all defaults, all 4 workloads):**
```bash
REPORT_PATH="$REPO_ROOT/tools/benchmark_report.md"
log "Running all 4 benchmark workloads (PERF-01 through PERF-04)..."
log "This will take approximately 10-20 minutes."

python3 "$REPO_ROOT/tools/relay_benchmark.py" \
    --relay-url "http://127.0.0.1:$RELAY_PORT" \
    --output "$REPORT_PATH"
```
- All other flags use relay_benchmark.py defaults per D-03: warmup=5, iterations=100, concurrency-levels="1,10,100", blob-sizes="1048576,10485760,52428800,104857600", mixed-duration=30, skip-uds=true.
- No `|| true` -- if benchmark fails, script fails (set -e).

**Metadata append (per D-05, D-06):**
After benchmark completes, append git hash and build type to the report:
```bash
GIT_HASH=$(git -C "$REPO_ROOT" rev-parse --short HEAD 2>/dev/null || echo "unknown")
CPU_MODEL=$(grep -m1 'model name' /proc/cpuinfo 2>/dev/null | cut -d: -f2 | xargs || uname -m)
OS_INFO=$(uname -srm)

{
    echo ""
    echo "---"
    echo ""
    echo "## Environment"
    echo ""
    echo "| Field | Value |"
    echo "|-------|-------|"
    echo "| Git hash | \`$GIT_HASH\` |"
    echo "| Build type | Release |"
    echo "| CPU | $CPU_MODEL |"
    echo "| OS | $OS_INFO |"
    echo "| Date | $(date '+%Y-%m-%d %H:%M:%S %Z') |"
} >> "$REPORT_PATH"
```

**Summary output:**
```bash

echo "============================================================"
echo "  Performance Benchmark Complete"
echo "============================================================"
echo "  Report: $REPORT_PATH"
echo "  Git hash: $GIT_HASH"

```
