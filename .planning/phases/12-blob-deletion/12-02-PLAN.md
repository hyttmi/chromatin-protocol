    Replace the existing liboqs FetchContent configuration block in CMakeLists.txt. The current block has `OQS_BUILD_ONLY_LIB` and `OQS_MINIMAL_BUILD`. Replace with comprehensive algorithm-stripping flags.

    **liboqs CMake flag reference (0.15.0):**
    liboqs uses `OQS_ENABLE_*` flags to control which algorithms are compiled. By default all are ON. We disable everything, then enable only what we use.

    Add these CMake cache variables BEFORE the FetchContent_Declare for liboqs:

    ```cmake
    # -- liboqs 0.15.0 (ML-DSA-87, ML-KEM-1024, SHA3-256)
    # Strip all unused algorithms to reduce build time and binary size.
    set(OQS_BUILD_ONLY_LIB ON CACHE BOOL "" FORCE)
    set(OQS_MINIMAL_BUILD "" CACHE STRING "" FORCE)

    # Disable all KEM algorithms, then enable only ML-KEM-1024
    set(OQS_ENABLE_KEM_BIKE OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_KEM_FRODOKEM OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_KEM_HQC OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_KEM_KYBER OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_KEM_ML_KEM ON CACHE BOOL "" FORCE)
    set(OQS_ENABLE_KEM_NTRUPRIME OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_KEM_CLASSIC_MCELIECE OFF CACHE BOOL "" FORCE)

    # Disable all signature algorithms, then enable only ML-DSA-87
    set(OQS_ENABLE_SIG_DILITHIUM OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_SIG_ML_DSA ON CACHE BOOL "" FORCE)
    set(OQS_ENABLE_SIG_FALCON OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_SIG_SPHINCS OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_SIG_MAYO OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_SIG_CROSS OFF CACHE BOOL "" FORCE)
    set(OQS_ENABLE_SIG_SLH_DSA OFF CACHE BOOL "" FORCE)

    # Disable liboqs tests and benchmarks
    set(OQS_ENABLE_TEST_SPEED OFF CACHE BOOL "" FORCE)
    set(OQS_BUILD_ONLY_LIB ON CACHE BOOL "" FORCE)
    ```

    **IMPORTANT:** Check the actual flag names by reading the liboqs 0.15.0 CMakeLists.txt (it's in build/_deps/liboqs-src/CMakeLists.txt after first build). The flag names may differ slightly (e.g., `OQS_ENABLE_KEM_ML_KEM` vs `OQS_ENABLE_KEM_ml_kem`). Read the liboqs CMakeLists.txt first to get exact flag names, then write the correct flags.

    Also check if there are additional KEM/SIG families added in 0.15.0 that aren't listed above. The goal is to disable ALL except ML-DSA and ML-KEM.

    After updating, do a clean rebuild to verify:


    Note: The clean rebuild of liboqs dependencies is needed because CMake cache variables from the previous build may persist. After the first clean rebuild, subsequent builds will be faster.
