**Test class 1: TestBackoffDelay** (CONN-03 backoff formula):
- `test_attempt_1_range`: 1000 samples of backoff_delay(1), all in [0, 1.0]
- `test_attempt_2_range`: 1000 samples of backoff_delay(2), all in [0, 2.0]
- `test_attempt_5_range`: 1000 samples of backoff_delay(5), all in [0, 16.0]
- `test_cap_at_30`: 1000 samples of backoff_delay(10), all in [0, 30.0]
- `test_custom_base_and_cap`: backoff_delay(1, base=2.0, cap=10.0) in [0, 2.0]; backoff_delay(5, base=2.0, cap=10.0) in [0, 10.0]
- `test_jitter_is_not_constant`: 100 samples, assert len(set of rounded values) > 1 (not all same)

**Test class 2: TestConnectionState**:
- `test_enum_values`: verify all 4 values: disconnected, connecting, connected, closing
- `test_member_count`: exactly 4 members

**Test class 3: TestInvokeCallback** (CONN-05 callback safety):
- `test_sync_callback`: sync function called with args, verified via mock
- `test_async_callback`: async function awaited with args, verified via mock
- `test_none_callback`: invoke_callback(None) does not raise
- `test_exception_suppressed`: callback that raises RuntimeError does not propagate; returns normally
- `test_async_exception_suppressed`: async callback that raises does not propagate

**Test class 4: TestClientStateTransitions** (CONN-03 state machine):
Test using direct Transport injection (ChromatinClient(transport)) with mocked transport:
- `test_direct_init_is_connected`: client = ChromatinClient(mock_transport) -> state is CONNECTED
- `test_connect_classmethod_initial_state`: client = ChromatinClient.connect(...) -> state is DISCONNECTED (before __aenter__)
- `test_connection_state_property`: verify it returns ConnectionState enum value

**Test class 5: TestReconnectLoop** (CONN-03 reconnect behavior):
These tests need to mock `_do_connect` to avoid real TCP. Strategy:
1. Create client with connect() classmethod
2. Manually set client state and transport
3. Mock `_do_connect` to succeed or fail
4. Call `_on_connection_lost()` and verify behavior

Tests:
- `test_reconnect_triggers_on_connection_lost`: Set state=CONNECTED, mock transport.closed=True. Call _on_connection_lost(). Verify state transitions to DISCONNECTED, then (via mocked _do_connect success) back to CONNECTED.
- `test_reconnect_retries_on_failure`: Mock _do_connect to fail twice then succeed. Verify 3 attempts total. Use asyncio.Event to synchronize.
- `test_close_stops_reconnect`: Start reconnect loop. Call close (set state=CLOSING). Verify reconnect loop exits. The reconnect_task should be done.
- `test_reconnect_not_triggered_when_auto_reconnect_false`: Create client with auto_reconnect=False. Call _on_connection_lost(). Verify _reconnect_task is None.

For mocking _do_connect, use `unittest.mock.AsyncMock` patched onto the client instance.

**Test class 6: TestSubscriptionRestore** (CONN-04):
- `test_subscriptions_resent_after_reconnect`: Create client, add namespaces to _subscriptions set, mock transport, call _restore_subscriptions(). Verify transport.send_message called with Subscribe type for each namespace.
- `test_subscribe_failure_logged_not_fatal`: Mock transport.send_message to raise. Call _restore_subscriptions(). Verify it completes without raising (subscriptions still in set).

**Test class 7: TestCallbacks** (CONN-05):
- `test_on_disconnect_fires`: Set on_disconnect callback (AsyncMock). Trigger _on_connection_lost(). Verify callback was awaited (give asyncio a tick).
- `test_on_reconnect_fires_with_args`: Set on_reconnect callback (AsyncMock). Mock _do_connect success. Trigger _on_connection_lost(). Wait for reconnect. Verify callback was awaited with (attempt_count: int, downtime: float).
- `test_callback_exception_does_not_kill_reconnect`: Set on_reconnect to raise. Mock _do_connect success. Trigger _on_connection_lost(). Verify state ends at CONNECTED despite callback failure.

**Test class 8: TestCloseNoReconnect** (CONN-03/CONN-05 close suppression):
- `test_close_sets_closing_state`: Verify __aexit__ sets state to CLOSING.
- `test_closing_state_prevents_reconnect`: Set state to CLOSING. Call _on_connection_lost(). Verify no reconnect task spawned.

**Test class 9: TestWaitConnected**:
- `test_returns_true_when_connected`: state=CONNECTED -> wait_connected() returns True immediately
- `test_returns_false_when_closing`: state=CLOSING -> wait_connected() returns False immediately
- `test_times_out`: state=DISCONNECTED, event not set -> wait_connected(timeout=0.1) returns False
- `test_unblocks_on_reconnect`: state=DISCONNECTED, spawn wait_connected(timeout=5.0) task, then set _connected_event, verify returns True

All async tests use `@pytest.mark.asyncio` decorator. All tests use `pytest` assertions (no unittest.TestCase).

IMPORTANT: Use `from unittest.mock import AsyncMock, MagicMock, patch` for mocking. For transport mocks, create a simple mock with `.closed` property, `.send_message` as AsyncMock, `.stop` as AsyncMock, `.notifications` as `asyncio.Queue()`.
