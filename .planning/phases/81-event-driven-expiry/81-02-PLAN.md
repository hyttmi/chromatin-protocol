    **Step 1: Rewrite expiry_scan_loop() (peer_manager.cpp, replacing lines 2873-2888):**

    Replace the entire function body with event-driven timing. The coroutine reads `next_expiry_target_` to determine how long to sleep, runs the scan, queries `get_earliest_expiry()` for rearm, and exits when no more expiring blobs exist.

    ```cpp
    asio::awaitable<void> PeerManager::expiry_scan_loop() {
        expiry_loop_running_ = true;
        while (!stopping_) {
            uint64_t target = next_expiry_target_;
            if (target == 0) break;  // No expiry target -- exit loop

            // Convert wall-clock expiry to steady_timer duration (Pitfall 3: underflow guard)
            uint64_t now = storage::system_clock_seconds();
            auto duration = (target > now)
                ? std::chrono::seconds(target - now)
                : std::chrono::seconds(0);  // Already expired, fire immediately

            asio::steady_timer timer(ioc_);
            expiry_timer_ = &timer;
            timer.expires_after(duration);
            auto [ec] = co_await timer.async_wait(
                asio::as_tuple(asio::use_awaitable));
            expiry_timer_ = nullptr;
            if (stopping_) break;
            if (ec) continue;  // Cancelled (rearmed by on_blob_ingested) -- loop back to read new target

            // D-05: Process ALL expired blobs in one scan
            auto purged = storage_.run_expiry_scan();
            if (purged > 0) {
                spdlog::info("expiry scan: purged {} blobs", purged);
            }

            // D-01/MAINT-02: Rearm to next earliest expiry from storage (source of truth)
            auto next = storage_.get_earliest_expiry();
            if (next.has_value()) {
                next_expiry_target_ = *next;
            } else {
                next_expiry_target_ = 0;
                break;  // No more expiring blobs -- exit loop
            }
        }
        expiry_loop_running_ = false;
    }
    ```

    Key design points:
    - `expiry_loop_running_` flag prevents double co_spawn (Pitfall 1)
    - Duration underflow guard: check `target > now` before subtraction (Pitfall 3)
    - On cancel (ec is truthy from operation_aborted): `continue` loops back to re-read `next_expiry_target_` which was updated by `on_blob_ingested()` before the cancel (Pitfall 4)
    - Per D-02: if timer fires and nothing expired, `run_expiry_scan()` returns 0, `get_earliest_expiry()` finds the actual next entry
    - On empty: sets `next_expiry_target_ = 0` and breaks. `on_blob_ingested()` will re-spawn the loop if a new expiring blob arrives.
