Replace the `generate_report()` stub with the full implementation. This is the main deliverable of Phase 31.

The function produces two outputs:
- `$RESULTS_DIR/REPORT.md` -- structured markdown report (OBS-02)
- `$RESULTS_DIR/benchmark-summary.json` -- combined machine-readable JSON (OBS-03)

**Helper functions** (add before generate_report):

1. `get_peak_stats()` -- Extract peak CPU% and memory for a container from docker-stats files for a scenario. Use jq with rtrimstr("%") for CPU parsing and split(" / ")[0] for memory extraction. Return "N/A" if no stats files found. Pattern: `$RESULTS_DIR/docker-stats-${scenario}*.json` glob for finding stats files.

2. `format_ingest_row()` -- Format one ingest scenario row (1k, 100k, or 1m). Check if source JSON exists, return "[not run]" row if missing. Extract blobs_per_sec, mib_per_sec, p50, p95, p99 with jq rounding (multiply, round, divide for 1-2 decimal places).

3. `compute_overhead()` -- Compute PQ overhead % from two values. Guard against division by zero with jq conditional (if $tr == 0 then "N/A").

**generate_report() implementation:**

Step 1: Collect provenance
- timestamp: `date -u +"%Y-%m-%dT%H:%M:%SZ"`
- commit: `git -C "$REPO_ROOT" rev-parse --short HEAD 2>/dev/null || echo "unknown"`
- version: `git -C "$REPO_ROOT" describe --tags --always 2>/dev/null || echo "dev"`

Step 2: Load hardware info
- Read from `$RESULTS_DIR/hardware-info.json` with jq. If missing, use "unknown" placeholders.

Step 3: Extract executive summary values
- Peak ingest blobs/sec: max blobs_per_sec across the three ingest JSONs (check each exists)
- Worst p99 latency: max latency_ms.p99 across the three ingest JSONs
- PQ overhead %: compute from scenario-trusted-vs-pq.json (pq_mode.latency_ms.p50 vs trusted_mode.latency_ms.p50). Use compute_overhead(). "[not run]" if missing.
- Late-joiner catch-up: from scenario-latejoin.json catchup_ms. "[not run]" if missing.

Step 4: Write REPORT.md using heredoc (cat > "$RESULTS_DIR/REPORT.md" <<EOF)

Report sections in order per CONTEXT.md:
1. **Header:** `# chromatindb Benchmark Report` + provenance line
2. **Executive Summary:** 4-row table (Peak ingest throughput, Worst p99 latency, PQ vs trusted overhead, Late-joiner catch-up)
3. **System Profile:** Table from hardware-info.json (CPU, Cores, RAM, Disk, Kernel, Docker)
4. **Topology:** ASCII art: `node1 --> node2 --> node3 --> [node4 late-joiner]` with sync_interval note
5. **Benchmark Parameters:** Table with BLOB_COUNT, RATE, drain timeout (10s), sync interval (10s)
6. **Ingest Throughput:** Header row (Size | blobs/sec | MiB/sec | p50 ms | p95 ms | p99 ms), then 3 data rows via format_ingest_row. Resource summary sub-table for peak CPU/memory from docker-stats-ingest-* files.
7. **Sync Latency:** Table with 1-hop ms, 2-hop ms, sync-interval multiple. Graceful if scenario-sync-latency.json missing. Resource summary.
8. **Late-Joiner Catch-Up:** Table with catch-up time in ms. Graceful if missing. Resource summary.
9. **Trusted vs PQ:** Side-by-side table: Metric | PQ | Trusted | Delta %. Rows for blobs/sec, p50, p95, p99. Use compute_overhead() for deltas. Graceful if missing. Resource summary.
10. **Caveats:** Three bullets: sync latency includes 10s sync_interval, single-run (no statistical significance), Docker overhead vs bare metal.
11. **Raw Data:** List JSON files in results directory.

Step 5: Build benchmark-summary.json
- Use jq -n to construct the combined JSON per the schema in RESEARCH.md
- Include: generated timestamp, commit, version, hardware (slurp hardware-info.json), parameters (blob_count, rate, drain_timeout, sync_interval)
- scenarios object: for each scenario, slurp the JSON file if it exists, otherwise null
- Write to `$RESULTS_DIR/benchmark-summary.json`

Step 6: Log completion
- `log "Report written: $RESULTS_DIR/REPORT.md"`
- `log "Summary written: $RESULTS_DIR/benchmark-summary.json"`

**Important implementation details:**
- ALL jq extractions must handle missing files gracefully (check -f before reading)
- Use `jq -r` for values embedded in heredoc (raw output, no quotes)
- For numeric formatting: `jq -r '. * 10 | round / 10'` for 1 decimal, `jq -r '. * 100 | round / 100'` for 2 decimals
- Docker stats CPU: `rtrimstr("%") | tonumber` before arithmetic
- Docker stats memory: `split(" / ")[0]` for usage portion
- Division by zero guard in compute_overhead: `if $tr == 0 then "N/A"`
- The heredoc for REPORT.md must use unquoted EOF (variable expansion needed)
- For the benchmark-summary.json construction, use `--slurpfile` or `--argjson` with file contents
