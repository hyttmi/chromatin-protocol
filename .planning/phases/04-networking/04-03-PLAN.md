    - After handshake: recv_message() reads a length-prefixed frame, decrypts, and returns decoded message
    - Ping/pong: periodic timer sends Ping every 30s, expects Pong within 30s, closes if timeout
    - Goodbye: send_goodbye() sends a Goodbye message and closes cleanly
    - Close: cancel all pending async ops, close socket

    Loopback integration test:

    ```cpp
    namespace chromatin::net {

    class Connection : public std::enable_shared_from_this<Connection> {
    public:
        using Ptr = std::shared_ptr<Connection>;

        /// Create from an accepted inbound socket (responder role).
        static Ptr create_inbound(asio::ip::tcp::socket socket,
                                   const identity::NodeIdentity& identity,
                                   asio::any_io_executor executor);

        /// Create for outbound connection (initiator role).
        static Ptr create_outbound(const std::string& address,
                                    const identity::NodeIdentity& identity,
                                    asio::any_io_executor executor);

        /// Run the connection lifecycle: handshake -> message loop -> cleanup.
        /// This is the main coroutine, spawned via co_spawn.
        asio::awaitable<void> run();

        /// Send a transport message (encrypted).
        asio::awaitable<void> send_message(TransportMsgType type,
                                            std::span<const uint8_t> payload);

        /// Send goodbye and close.
        asio::awaitable<void> close_gracefully();

        /// Force close (cancel all ops).
        void close();

        /// Whether the connection has completed handshake.
        bool is_authenticated() const;

        /// Peer's signing public key (available after auth).
        std::span<const uint8_t> peer_pubkey() const;

        /// Set callback for received data messages.
        using MessageCallback = std::function<void(TransportMsgType, std::vector<uint8_t>)>;
        void on_message(MessageCallback cb);

        /// Set callback for connection close/error.
        using CloseCallback = std::function<void(std::shared_ptr<Connection>)>;
        void on_close(CloseCallback cb);

    private:
        // ...
    };

    } // namespace chromatin::net
    ```

    **Implementation details:**
    - Use `asio::as_tuple(asio::use_awaitable)` for all async operations to avoid exceptions on expected errors (connection reset, timeout).
    - Handshake phase:
      - For inbound: run HandshakeResponder steps, reading/writing raw frames for KEM exchange, then encrypted frames for auth.
      - For outbound: resolve address, connect, run HandshakeInitiator steps.
      - 10-second timeout on handshake: use `asio::steady_timer` with `co_await` using the `||` operator from `<asio/experimental/awaitable_operators.hpp>` or a simple timer + cancel pattern.
      - On handshake failure: log at warn level, close socket silently (no error sent to peer per CONTEXT.md).
    - Message loop:
      - After handshake, enter a loop: read frame -> decrypt -> decode -> dispatch via callback.
      - Handle Ping by replying Pong immediately.
      - Handle Pong by resetting dead-peer timer.
      - Handle Goodbye by logging and closing cleanly.
      - Handle Data by invoking message callback.
    - Heartbeat:
      - 30-second steady_timer fires Ping. If no Pong received within another 30s, close connection.
      - Cancel heartbeat timer on close/shutdown.
    - Counters: send_counter_ increments on each write_frame, recv_counter_ increments on each read_frame. These drive the counter-based nonces.
    - For the async frame IO over TCP: adapt the in-memory write_frame/read_frame to use `asio::async_write` and `asio::async_read`. Write the 4-byte header + ciphertext in one write. Read the 4-byte header first, validate size, then read the ciphertext.

    **Test:** Loopback test using asio::io_context. Create a tcp::acceptor on 127.0.0.1:0 (random port), connect an outbound Connection to it, accept the inbound socket, run both connections. Verify handshake completes, exchange a Data message, verify it arrives correctly. Test auth failure by using a different identity.

    **src/net/server.h:**

    ```cpp
    namespace chromatin::net {

    class Server {
    public:
        Server(const config::Config& config,
               const identity::NodeIdentity& identity,
               asio::io_context& ioc);

        /// Start accepting connections and connect to bootstrap peers.
        void start();

        /// Trigger graceful shutdown (non-blocking, starts drain).
        void stop();

        /// Number of active authenticated connections.
        size_t connection_count() const;

        /// Whether the server is in shutdown/draining state.
        bool is_draining() const;

    private:
        asio::awaitable<void> accept_loop();
        asio::awaitable<void> connect_to_peer(const std::string& address);
        asio::awaitable<void> reconnect_loop(const std::string& address);
        asio::awaitable<void> drain(std::chrono::seconds timeout);

        const config::Config& config_;
        const identity::NodeIdentity& identity_;
        asio::io_context& ioc_;
        asio::ip::tcp::acceptor acceptor_;
        asio::signal_set signals_;
        std::vector<Connection::Ptr> connections_;
        bool draining_ = false;
    };

    } // namespace chromatin::net
    ```

    **Implementation details:**

    **Accept loop:**
    ```cpp
    asio::awaitable<void> Server::accept_loop() {
        while (!draining_) {
            auto [ec, socket] = co_await acceptor_.async_accept(
                asio::as_tuple(asio::use_awaitable));
            if (ec) {
                if (draining_) break;
                spdlog::warn("accept error: {}", ec.message());
                continue;
            }
            auto conn = Connection::create_inbound(std::move(socket), identity_, ioc_.get_executor());
            connections_.push_back(conn);
            conn->on_close([this](auto c) { remove_connection(c); });
            asio::co_spawn(ioc_, conn->run(), asio::detached);
        }
    }
    ```

    **Outbound connect:**
    - Parse "host:port" from address string.
    - Use `asio::ip::tcp::resolver` to resolve, then `socket.async_connect()`.
    - On success: create Connection::create_outbound, add to connections_, spawn run().
    - On failure: enter reconnect_loop with exponential backoff.

    **Reconnect loop:**
    - Use `asio::steady_timer` for backoff delay.
    - Start at 1s, double each attempt, cap at 60s.
    - Reset delay on successful connection.
    - If peer sent Goodbye before disconnect, start with 0s delay (immediate reconnect).
    - Cancel timer when draining_ is set.

    **Signal handling:**
    ```cpp
    signals_.async_wait([this](auto ec, int sig) {
        if (ec) return;
        if (draining_) {
            spdlog::info("second signal received ({}), forcing exit", sig);
            std::_Exit(1);
        }
        spdlog::info("signal {} received, starting graceful shutdown", sig);
        asio::co_spawn(ioc_, drain(std::chrono::seconds(5)), asio::detached);
        // Re-register for second signal
        signals_.async_wait([](auto, int) { std::_Exit(1); });
    });
    ```

    **Drain:**
    1. Set draining_ = true.
    2. Cancel acceptor (stops accept loop).
    3. Send goodbye to all connections: `for (auto& c : connections_) co_await c->close_gracefully();`
    4. Log "goodbye sent to N peers".
    5. Start 5-second drain timer. When timer fires or all connections closed, stop io_context.
    6. Log "shutdown complete".

    **Tests:**
    - Use `asio::io_context` with `ioc.run_for(std::chrono::seconds(5))` for bounded test execution.
    - Create two servers on 127.0.0.1 with random ports (port 0 lets OS assign).
    - Configure server A's bootstrap_peers to include server B's address and vice versa.
    - Run io_context, verify both sides connect and authenticate.
    - Test shutdown: call server.stop(), verify drain completes within 5s.
    - Test reconnect: forcefully close one connection, verify reconnect attempt occurs.
    - All tests MUST be deterministic and not rely on timing (use timers/events, not sleep).

    Write TDD tests FIRST.
