    **relay/core/relay_session.h** in namespace `chromatindb::relay::core`:

    ```cpp
    #pragma once

    #include "db/identity/identity.h"
    #include "db/net/connection.h"

    #include <asio.hpp>
    #include <asio/local/stream_protocol.hpp>

    #include <functional>
    #include <memory>
    #include <string>

    namespace chromatindb::relay::core {

    /// Manages a single client session through the relay.
    /// Pairs a PQ-authenticated client TCP connection with a TrustedHello UDS
    /// connection to the chromatindb node. Forwards allowed messages bidirectionally.
    class RelaySession : public std::enable_shared_from_this<RelaySession> {
    public:
        using Ptr = std::shared_ptr<RelaySession>;
        using CloseCallback = std::function<void(RelaySession::Ptr)>;

        /// Create a relay session for an authenticated client connection.
        /// client_conn must already be post-handshake (authenticated).
        static Ptr create(chromatindb::net::Connection::Ptr client_conn,
                          const std::string& uds_path,
                          const chromatindb::identity::NodeIdentity& identity,
                          asio::io_context& ioc);

        /// Start the session: connect UDS to node, begin bidirectional forwarding.
        /// Returns false if UDS connection fails (per D-03: refuse client).
        asio::awaitable<bool> start();

        /// Tear down both connections.
        void stop();

        /// Set callback for when session ends (both directions closed).
        void on_close(CloseCallback cb) { close_cb_ = std::move(cb); }

        /// Client's public key hash hex string (for logging per D-06).
        const std::string& client_pubkey_hex() const { return client_pk_hex_; }

        /// Client's remote address (for logging per D-06).
        const std::string& client_address() const;

    private:
        RelaySession(chromatindb::net::Connection::Ptr client_conn,
                     std::string uds_path,
                     const chromatindb::identity::NodeIdentity& identity,
                     asio::io_context& ioc);

        /// Forward messages from client to node (with filtering per RELAY-03).
        void handle_client_message(chromatindb::net::Connection::Ptr conn,
                                   chromatindb::wire::TransportMsgType type,
                                   std::vector<uint8_t> payload);

        /// Forward messages from node to client (no filtering -- node only sends client types).
        void handle_node_message(chromatindb::net::Connection::Ptr conn,
                                 chromatindb::wire::TransportMsgType type,
                                 std::vector<uint8_t> payload);

        /// Handle client disconnect.
        void handle_client_close(chromatindb::net::Connection::Ptr conn, bool graceful);

        /// Handle node UDS disconnect (per D-04: disconnect client immediately).
        void handle_node_close(chromatindb::net::Connection::Ptr conn, bool graceful);

        /// Shared teardown logic.
        void teardown(const std::string& reason);

        chromatindb::net::Connection::Ptr client_conn_;
        chromatindb::net::Connection::Ptr node_conn_;
        std::string uds_path_;
        const chromatindb::identity::NodeIdentity& identity_;
        asio::io_context& ioc_;
        std::string client_pk_hex_;
        bool stopped_ = false;

        CloseCallback close_cb_;
    };

    } // namespace chromatindb::relay::core
    ```

    **relay/core/relay_session.cpp** implementation:

    - **Constructor**: Store client_conn, uds_path, identity ref, ioc ref. Compute client_pk_hex_ from client_conn->peer_pubkey() using SHA3-256 + hex encoding (reuse the pattern from relay_main.cpp's to_hex, or include a local to_hex helper in an anonymous namespace).

    - **create()**: `return Ptr(new RelaySession(client_conn, uds_path, identity, ioc));`

    - **start()**: This is a coroutine (returns `asio::awaitable<bool>`).
      1. Open a UDS socket: `asio::local::stream_protocol::socket uds_socket(ioc_);`
      2. Connect to node: `auto [ec] = co_await uds_socket.async_connect(asio::local::stream_protocol::endpoint(uds_path_), chromatindb::net::use_nothrow);`
      3. If connect fails, log error and `co_return false;` (per D-03)
      4. Create node connection: `node_conn_ = chromatindb::net::Connection::create_uds_outbound(std::move(uds_socket), identity_);`
      5. Set up message callbacks on BOTH connections:
         - `client_conn_->on_message(...)` -> `handle_client_message` (captures shared_from_this)
         - `node_conn_->on_message(...)` -> `handle_node_message` (captures shared_from_this)
         - `client_conn_->on_close(...)` -> `handle_client_close`
         - `node_conn_->on_close(...)` -> `handle_node_close`
      6. IMPORTANT: The client connection is already running (run() was called by the accept loop). DO NOT call client_conn_->run() again. Only the client's on_message callback needs to be updated to forward.
      7. Spawn node connection run: `asio::co_spawn(ioc_, node_conn_->run(), asio::detached);` -- this performs TrustedHello handshake with the node.
      8. `co_return true;`

      WAIT: There is a subtlety. The client_conn->on_message and on_close were set BEFORE run() by the accept loop. But on_ready fires after handshake -- that's where RelaySession::start() gets called. By the time start() runs, the message loop hasn't started yet (on_ready fires between handshake and message_loop in Connection::run()). So the client callbacks WILL be set in time.

      Actually, re-reading Connection::run(): after handshake succeeds, it calls on_ready, then enters message_loop. So setting on_message in on_ready (which calls start()) is perfect -- callbacks are in place before messages arrive.

      But the node_conn_ needs its own run() to perform TrustedHello. And its on_ready fires after TrustedHello completes. There's a timing issue: client messages could arrive before node handshake finishes. This is fine -- handle_client_message can queue or the Connection will buffer. Actually, Connection::send_message on node_conn_ before it's authenticated would fail. We need to wait for node handshake.

      Better approach: In start(), after spawning node_conn_->run(), set node_conn_->on_ready() to log "UDS connection to node established". The client messages will arrive but we need node_conn_ to be ready. Since TrustedHello is very fast (no PQ crypto, just nonce exchange), this is practically instant. But to be safe, we can set the client_conn_->on_message INSIDE node_conn_->on_ready callback, ensuring messages only flow after the node is ready.

      Revised start() flow:
      1-4: Same as above (connect UDS, create node_conn_)
      5. Set node_conn_->on_close -> handle_node_close
      6. Set node_conn_->on_ready callback that:
         a. Sets client_conn_->on_message -> handle_client_message
         b. Sets node_conn_->on_message -> handle_node_message
         c. Logs "session active" at info
      7. Set client_conn_->on_close -> handle_client_close
      8. Spawn node_conn_->run() -- TrustedHello handshake + message loop
      9. co_return true

    - **handle_client_message()**: Check `is_client_allowed(type)`. If allowed, spawn a coroutine to `node_conn_->send_message(type, payload)`. If blocked, log warning with type_name and client_pk_hex_ (per D-07), then call teardown("blocked message type") to disconnect (per D-01).

    - **handle_node_message()**: Spawn a coroutine to `client_conn_->send_message(type, payload)`. No filtering needed -- node only sends types the client understands.

    - **handle_client_close()**: Log "client disconnected" at info with reason (graceful/error) and client_pk_hex_ (per D-06). Call teardown("client disconnected").

    - **handle_node_close()**: Log "node UDS connection lost" at info. Call teardown("node disconnected") (per D-04: disconnect client immediately).

    - **teardown()**: If already stopped_, return. Set stopped_ = true. Close both connections (client_conn_->close(), node_conn_->close() if non-null). Fire close_cb_ with shared_from_this(). Log the reason.

    - **client_address()**: Return client_conn_->remote_address().

    For the coroutine spawning in handle_client_message and handle_node_message, use:
    ```cpp
    auto self = shared_from_this();
    auto t = type;
    auto p = std::move(payload);
    asio::co_spawn(ioc_, [self, conn = node_conn_, t, p = std::move(p)]() -> asio::awaitable<void> {
        co_await conn->send_message(t, p);
    }, asio::detached);
    ```
    This is necessary because on_message is a synchronous callback but send_message is a coroutine.
