Create dist/install.sh as a POSIX-compatible shell script (#!/bin/sh, not bash). Must be executable (chmod +x).

**Usage:** `sudo dist/install.sh [--uninstall] <chromatindb_binary> <chromatindb_relay_binary>`

**Script structure:**

1. **Header and constants:**
```sh
#!/bin/sh
set -eu

BINDIR="/usr/local/bin"
UNITDIR="/usr/lib/systemd/system"
SYSUSERSDIR="/usr/lib/sysusers.d"
TMPFILESDIR="/usr/lib/tmpfiles.d"
CONFDIR="/etc/chromatindb"
DATADIR="/var/lib/chromatindb"
SCRIPTDIR="$(cd "$(dirname "$0")" && pwd)"
```

2. **Argument parsing:**
- If first arg is `--uninstall`, run uninstall mode (no binary args needed)
- Otherwise, exactly 2 positional args required: path to chromatindb binary, path to chromatindb_relay binary
- Validate both binary paths exist and are executable files
- Print usage to stderr and exit 1 on bad args: `Usage: install.sh [--uninstall] <chromatindb> <chromatindb_relay>`
- Check running as root (id -u == 0), exit 1 with error if not

3. **Install mode** (default, no flags):
```
a. Copy binaries:
   install -m 0755 "$1" "$BINDIR/chromatindb"
   install -m 0755 "$2" "$BINDIR/chromatindb_relay"

b. Install sysusers.d and create user:
   install -m 0644 "$SCRIPTDIR/sysusers.d/chromatindb.conf" "$SYSUSERSDIR/chromatindb.conf"
   systemd-sysusers

c. Install tmpfiles.d and create directories:
   install -m 0644 "$SCRIPTDIR/tmpfiles.d/chromatindb.conf" "$TMPFILESDIR/chromatindb.conf"
   systemd-tmpfiles --create chromatindb.conf

d. Install configs (PRESERVE existing -- per D-08 reinstall safety):
   if [ ! -f "$CONFDIR/node.json" ]; then
     install -m 0644 "$SCRIPTDIR/config/node.json" "$CONFDIR/node.json"
   fi
   if [ ! -f "$CONFDIR/relay.json" ]; then
     install -m 0644 "$SCRIPTDIR/config/relay.json" "$CONFDIR/relay.json"
   fi

e. Install systemd units:
   install -m 0644 "$SCRIPTDIR/systemd/chromatindb.service" "$UNITDIR/chromatindb.service"
   install -m 0644 "$SCRIPTDIR/systemd/chromatindb-relay.service" "$UNITDIR/chromatindb-relay.service"
   systemctl daemon-reload

f. Generate identity keys if missing (per D-06):
   if [ ! -f "$DATADIR/node.key" ]; then
     "$BINDIR/chromatindb" keygen --data-dir "$DATADIR"
     chown chromatindb:chromatindb "$DATADIR/node.key" "$DATADIR/node.pub"
   fi
   if [ ! -f "$DATADIR/relay.key" ]; then
     "$BINDIR/chromatindb_relay" keygen --output "$DATADIR/relay.key"
     chown chromatindb:chromatindb "$DATADIR/relay.key" "$DATADIR/relay.pub"
   fi
```

Per D-07: Script does NOT call `systemctl enable` or `systemctl start`. Operator does that manually.
Per D-09: No echo/printf on success. Only `>&2` error messages on failure.

4. **Uninstall mode** (`--uninstall` flag, per D-08):
```
a. Stop and disable services (if active):
   systemctl stop chromatindb-relay.service 2>/dev/null || true
   systemctl stop chromatindb.service 2>/dev/null || true
   systemctl disable chromatindb-relay.service 2>/dev/null || true
   systemctl disable chromatindb.service 2>/dev/null || true

b. Remove installed files:
   rm -f "$UNITDIR/chromatindb.service"
   rm -f "$UNITDIR/chromatindb-relay.service"
   rm -f "$SYSUSERSDIR/chromatindb.conf"
   rm -f "$TMPFILESDIR/chromatindb.conf"
   rm -f "$BINDIR/chromatindb"
   rm -f "$BINDIR/chromatindb_relay"

c. Reload systemd:
   systemctl daemon-reload

d. Preserve data and configs (per D-08: preserve data dir and configs):
   Do NOT remove /var/lib/chromatindb, /etc/chromatindb, or /var/log/chromatindb
```

5. **Error handling:**
- `set -eu` at top (exit on error, exit on undefined var)
- All error messages to stderr: `echo "error: message" >&2`
- Exit 1 on any validation failure
- Use `|| true` only on systemctl stop/disable in uninstall (services might not be running)
