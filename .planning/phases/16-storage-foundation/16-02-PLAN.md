<objective>
Add configurable storage capacity limit (`max_storage_bytes`) with Step 0 enforcement in the ingest pipeline. When the database exceeds the configured limit, ingest returns `IngestError::storage_full` before any crypto operations.

Purpose: Prevent nodes from consuming unbounded disk space. Operators configure a limit; the node enforces it cheaply at the earliest possible point in the ingest path.
Output: Config field, IngestError enum value, Step 0 capacity gate in BlobEngine::ingest().
</objective>
