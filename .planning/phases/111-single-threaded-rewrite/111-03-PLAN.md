**--- relay/tests/test_http_router.cpp ---**
Per D-11:

1. Delete the `using Strand = asio::strand<asio::io_context::executor_type>;` line (line 15).

2. **Rewrite `run_async_dispatch` helper (lines 248-260):** Remove strand creation and set_strand call. The new version:
   ```cpp
   static HttpResponse run_async_dispatch(HttpRouter& router, const HttpRequest& req,
                                           const std::vector<uint8_t>& body, TokenStore& store) {
       asio::io_context ioc;
       HttpResponse result;
       asio::co_spawn(ioc, [&]() -> asio::awaitable<void> {
           result = co_await router.dispatch_async(req, body, store);
       }, asio::detached);
       ioc.run();
       return result;
   }
   ```
   Key changes: no `auto strand = asio::make_strand(ioc);`, no `router.set_strand(&strand);`, co_spawn on `ioc` directly.

3. **Update all 4 auth endpoint test cases** that create a local strand for register_auth_routes:
   - "POST /auth/challenge returns 200 with nonce" (lines 262-282)
   - "POST /auth/verify with invalid nonce returns 401" (lines 284-306)
   - "POST /auth/verify with bad JSON returns 400" (lines 308-323)
   - "POST /auth/verify with missing fields returns 400" (lines 325-342)

   In each test case:
   - Remove `auto strand = asio::make_strand(ioc);`
   - Add `asio::thread_pool pool(1);` (1 thread is enough for tests)
   - Change `register_auth_routes(router, authenticator, store, strand);` to `register_auth_routes(router, authenticator, store, pool, ioc);`
