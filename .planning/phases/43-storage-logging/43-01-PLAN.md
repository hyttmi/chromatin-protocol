    1. Rewrite db/logging/logging.h: Change init() signature to:
       ```cpp

                 const std::string& log_file = "",

                 uint32_t max_files = 3,
                 const std::string& log_format = "text");
       ```
       Keep get_logger() signature unchanged.

    2. Rewrite db/logging/logging.cpp completely:
       - Remove std::call_once (RESEARCH pitfall 2: init() is only called once at startup, call_once prevents re-init with new params)
       - Add module-level `static std::vector<spdlog::sink_ptr> shared_sinks;` to store sinks
       - Add `#include <spdlog/sinks/rotating_file_sink.h>` and keep existing stdout include
       - In init():
         a. Create stderr_color_sink_mt as console sink, add to sinks vector
         b. If log_file is non-empty, try to create rotating_file_sink_mt(log_file, max_size_mb * 1024 * 1024, max_files). Catch spdlog::spdlog_ex and warn to stderr (fmt::print(stderr, ...)), fall back to console-only. Per user decision: node still starts.
         c. Create default logger from sinks vector: `std::make_shared<spdlog::logger>("", sinks.begin(), sinks.end())`
         d. Set pattern based on log_format:
            - "json": `R"({"ts":"%Y-%m-%dT%H:%M:%S.%e","level":"%l","logger":"%n","msg":"%v"})"`
            - "text": `"[%Y-%m-%d %H:%M:%S.%e] [%l] [%n] %v"` (same as current)
         e. Set level via parse_level(), call spdlog::set_default_logger()
       - Fix get_logger() (RESEARCH pitfall 4): Create new loggers using shared_sinks instead of spdlog::stderr_color_mt():
         ```cpp
         auto logger = spdlog::get(name);
         if (!logger) {
             logger = std::make_shared<spdlog::logger>(name, shared_sinks.begin(), shared_sinks.end());
             logger->set_level(global_level);
             spdlog::register_logger(logger);
         }
         return logger;
         ```
       - Both sinks use the SAME formatter (text or JSON). Do NOT mix formats between sinks (RESEARCH anti-pattern).

    3. Update db/main.cpp cmd_run(): Change the logging::init() call (currently line 108) to pass new config fields:
