## Decisions Made
- Used validation_failed path instead of malformed_payload: the relay's JSON-to-binary translation has strict field validation (HEX_32 checks exact length, etc.), so truly malformed payloads are rejected at the relay before reaching the node. Instead, we send structurally valid requests with semantic errors (empty arrays, inverted time ranges) that pass relay translation but trigger node-side validation.
- Selected BatchExistsRequest, BatchReadRequest, and TimeRangeRequest: these all have clear validation rules in the node (count==0 rejection, start_ts > end_ts rejection) that produce deterministic ErrorResponse codes.
