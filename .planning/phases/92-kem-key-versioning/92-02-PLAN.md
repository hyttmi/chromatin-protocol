    **Implementation changes to _directory.py (per D-04, D-05, D-06, D-07):**

    1. **Bump version constant:**
       ```python
       USERENTRY_VERSION: int = 0x02
       ```
       Update USERENTRY_MIN_SIZE to account for key_version:4 bytes:
       ```python
       USERENTRY_MIN_SIZE: int = 4 + 1 + PUBLIC_KEY_SIZE + KEM_PUBLIC_KEY_SIZE + 2 + 4 + 1
       #                                                                        name_len  kv  kem_sig_min
       ```

    2. **DirectoryEntry -- add key_version field:**
       ```python
       @dataclass(frozen=True)
       class DirectoryEntry:
           identity: Identity
           display_name: str
           blob_hash: bytes
           key_version: int = 0
       ```

    3. **encode_user_entry changes (D-05, D-06):**
       - Read `key_version = identity.key_version`
       - Sign `(kem_public_key + struct.pack(">I", key_version))` instead of just `kem_public_key`
       - Insert `struct.pack(">I", key_version)` between name bytes and kem_sig in the output:
       ```python
       return (
           USERENTRY_MAGIC
           + struct.pack("B", USERENTRY_VERSION)  # 0x02
           + identity.public_key
           + identity.kem_public_key
           + struct.pack(">H", len(name_bytes))
           + name_bytes
           + struct.pack(">I", key_version)  # NEW: 4 bytes big-endian
           + kem_sig
       )
       ```

    4. **decode_user_entry changes (D-05):**
       Return 5-tuple: `(signing_pk, kem_pk, display_name, key_version, kem_sig)`.
       After parsing display_name:
       ```python
       if offset + 4 > len(data):
           return None
       key_version = struct.unpack(">I", data[offset : offset + 4])[0]
       offset += 4
       kem_sig = data[offset:]
       ```

    5. **verify_user_entry changes (D-05):**
       Add `key_version: int` parameter. Signed message becomes `kem_pk + struct.pack(">I", key_version)`:
       ```python
       def verify_user_entry(signing_pk: bytes, kem_pk: bytes, key_version: int, kem_sig: bytes) -> bool:
           signed_data = kem_pk + struct.pack(">I", key_version)
           return Identity.verify(signed_data, kem_sig, signing_pk)
       ```

    6. **_populate_cache changes (D-07):**
       After decoding UserEntry (now 5-tuple with key_version):
       ```python
       signing_pk, kem_pk, display_name, key_version, kem_sig = parsed
       if not verify_user_entry(signing_pk, kem_pk, key_version, kem_sig):
           ...continue
       pk_hash = sha3_256(signing_pk)
       identity = Identity.from_public_keys(signing_pk, kem_pk)
       entry = DirectoryEntry(
           identity=identity,
           display_name=display_name,
           blob_hash=blob_ref.blob_hash,
           key_version=key_version,
       )
       # Highest-version-wins (D-07):
       existing = by_pubkey_hash.get(pk_hash)
       if existing is None or key_version > existing.key_version:
           cache[blob_ref.blob_hash] = entry
           by_name[display_name] = entry
           by_pubkey_hash[pk_hash] = entry
       ```
       Note: When a higher-version entry replaces a lower-version one, also need to remove the old blob_hash from the `cache` dict if the old entry's blob_hash differs. Actually, `cache` is keyed by blob_hash so both entries stay in it but `by_name` and `by_pubkey_hash` point to the higher version. This is acceptable -- cache is used for `list_users()` which should show the latest only. Clean approach: if replacing, remove old blob_hash from cache:
       ```python
       if existing is not None and key_version > existing.key_version:
           cache.pop(existing.blob_hash, None)
       ```

    7. **resolve_recipient() method on Directory (D-07):**

       ```python
       async def resolve_recipient(self, display_name: str) -> Identity | None:

           entry = await self.get_user(display_name)
           return entry.identity if entry is not None else None
       ```
