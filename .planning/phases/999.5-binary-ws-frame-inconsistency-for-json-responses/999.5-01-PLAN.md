<objective>
Remove the binary WebSocket frame send path from the relay. ReadResponse and BatchReadResponse currently go through send_binary() which prepends a 0x02 marker byte, but the payload is JSON text -- clients must special-case two frame types for no benefit. After this change, all JSON responses use send_json() and text frames uniformly.

Purpose: Eliminate unnecessary complexity in the WS send path and consistent frame type for all JSON responses.
Output: Simplified relay with ~30 lines removed, no binary send path, all tests passing.
</objective>
