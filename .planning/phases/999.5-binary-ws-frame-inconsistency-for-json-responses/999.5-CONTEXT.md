# Phase 999.5: Binary WS Frame Inconsistency for JSON Responses - Context

**Gathered:** 2026-04-12
**Status:** Ready for planning

<domain>
## Phase Boundary

ReadResponse(32) and BatchReadResponse(54) are sent as binary WebSocket frames (opcode 0x2) but their payload is JSON text. All other responses use text frames. Remove the binary frame path entirely — send all JSON responses as text frames.

</domain>

<decisions>
## Implementation Decisions

### Fix
- **D-01:** Remove `send_binary()` from WsSession. All responses use `send_json()`.
- **D-02:** Remove `is_binary_response()` from translator.h.
- **D-03:** Remove binary/text branching in `route_response()` — just call `send_json()`.
- **D-04:** Remove the 0x02 binary marker prefix detection in the send queue write_frame() path.
- **D-05:** No backward compat. Pre-MVP, clean break.
- **D-06:** Update smoke test if it checks frame types for read responses.

### Claude's Discretion
- Whether OPCODE_BINARY constant stays for WS frame decoding on the receive side

</decisions>
