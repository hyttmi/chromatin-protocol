Make 5 hardcoded sync/peer constants configurable via config.json with SIGHUP reload for 3 of them, add validation, update all component classes to use config-loaded member fields instead of static constexprs, and write a pidfile on daemon startup.

Purpose: Operators can tune node behavior without recompiling, and 3 of the 5 constants can be changed at runtime via SIGHUP without node restart. The pidfile enables Plan 02's peer management subcommands to signal the running node.

Output: Config struct with 5 new fields, component classes using member fields, SIGHUP reload wired for 3 fields, pidfile lifecycle in main.cpp, tests passing.
