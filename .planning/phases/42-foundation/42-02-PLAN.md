  <action>
1. In `db/config/config.h`, add after the `validate_trusted_peers` declaration:
   ```cpp
   /// Validate all config field values (ranges, types, formats).
   /// Accumulates all errors and throws std::runtime_error with all of them.
   /// @throws std::runtime_error if any validation fails.
   void validate_config(const Config& cfg);
   ```

2. In `db/config/config.cpp`, implement `validate_config()`:
   - Create `std::vector<std::string> errors;`
   - Check each numeric field per the validation table from RESEARCH.md:
     - `max_peers >= 1`
     - `sync_interval_seconds >= 1`
     - `max_storage_bytes == 0 OR >= 1048576` (1 MiB)
     - `rate_limit_bytes_per_sec == 0 OR >= 1024`
     - If `rate_limit_bytes_per_sec > 0`: `rate_limit_burst >= rate_limit_bytes_per_sec`
     - `full_resync_interval >= 1`
     - `cursor_stale_seconds >= 60`
     - `worker_threads == 0 OR 1-256`
     - `sync_cooldown_seconds`: no constraint (0 = disabled is valid)
     - `max_sync_sessions >= 1`
   - Validate `log_level`: must be one of trace, debug, info, warn, warning, error, err, critical (case-sensitive, matching spdlog levels)
   - Validate `bind_address`: must contain ':', port part must parse as integer 1-65535
   - Do NOT validate `namespace_quota_bytes` or `namespace_quota_count` (0 = unlimited, any value valid)
   - If `!errors.empty()`, build message: "Configuration errors:\n" + "  - " + each error + "\n", throw `std::runtime_error(msg)`
   - Error messages must include field name and actual value, e.g.: `"max_peers must be >= 1 (got 0)"`

3. In `load_config()`, wrap the `j.value()` block in a try-catch for `nlohmann::json::type_error`. Catch it and rethrow as `std::runtime_error` with a human-readable message. The approach: wrap the entire block, and on type_error, the exception message from nlohmann includes enough context. Rethrow as: `"Config type error: " + e.what() + " (check field types in config file)"`.

4. In `load_config()`, after parsing all known fields but before return, add unknown key detection:
   - Define a `static const std::set<std::string> known_keys` containing all recognized field names
   - Iterate `j.items()`, for any key not in known_keys, call `spdlog::warn("unknown config key '{}' (ignored)", key)`
   - Add `#include <spdlog/spdlog.h>` to config.cpp if not already present
   - Do NOT throw on unknown keys (pre-1.0, forward compatibility)

5. Add comprehensive tests in `db/tests/config/test_config.cpp` (append to existing file):
   - Test validate_config with default Config (should pass)
   - Test each invalid field individually (should throw with field name in message)
   - Test multiple simultaneous errors (should throw with all errors)
   - Test valid edge cases (max_storage_bytes=0, rate_limit_bytes_per_sec=0, worker_threads=0)
   - Test type mismatch via load_config with wrong JSON types
   - Test all valid log_level values
   - Test bind_address validation (valid and invalid)

CRITICAL: Do not add constraints to `sync_cooldown_seconds` (0 = disabled is valid) or quota fields (any value valid). Set minimums to lowest operationally valid value to avoid breaking existing test configs that use sync_interval_seconds=1.
