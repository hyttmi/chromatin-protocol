    FragmentAssembler class:
    ```cpp
    enum class AssemblyResult { INCOMPLETE, COMPLETE, CONTROL, ERROR };
    struct AssembledMessage {
        AssemblyResult result;
        uint8_t opcode;           // Original opcode (text/binary) or control opcode
        std::vector<uint8_t> data;
        uint16_t close_code;      // Only for close frames
        std::string error_reason;
    };
    class FragmentAssembler {
    public:
        // Feed a decoded frame (already unmasked). Returns assembly state.
        AssembledMessage feed(uint8_t opcode, bool fin, std::span<const uint8_t> payload);
        void reset();
    private:
        std::vector<uint8_t> buffer_;
        uint8_t start_opcode_ = 0;
        bool in_fragment_ = false;
    };
    ```
    Fragment reassembly logic per RFC 6455 Section 5.4:
    - Non-control FIN=0 with opcode != 0 starts a new fragmented message (save opcode, accumulate payload).
    - Continuation frames (opcode=0) with FIN=0 append payload.
    - Continuation frame with FIN=1 completes the message.
    - Control frames (ping/pong/close) during fragmentation return immediately as CONTROL result without disturbing fragment state.
    - Check accumulated size against max: text uses MAX_TEXT_MESSAGE_SIZE, binary uses MAX_BINARY_MESSAGE_SIZE. Exceeding returns ERROR.
    - Unexpected continuation without active fragment returns ERROR.
    - Non-control frame while fragment in progress (opcode != 0 && in_fragment_) returns ERROR.
