<canonical_layouts>
NAME payload (D-03): `[NAME_MAGIC:4][name_len:2 BE][name:N bytes][target_content_hash:32]`
  - name_len is 2-byte big-endian, range [0, 65535]
  - MIN size = 4 + 2 + 0 + 32 = 38 bytes (empty-name edge case allowed structurally per VALIDATION.md / RESEARCH Example 1)
  - names are opaque bytes (D-04) — use memcmp, not string comparison

BOMB payload (D-05): `[BOMB_MAGIC:4][count:4 BE][target_hash:32 × count]`
  - count is 4-byte big-endian, range [0, UINT32_MAX] subject to MAX_BLOB_DATA_SIZE
  - MIN size = 4 + 4 = 8 bytes (count==0 is accepted as structurally valid no-op per Claude-discretion recommendation A2)
  - data.size() MUST equal 8 + count*32 exactly

Magic bytes (NO COLLISIONS — verify against codec.h:92/112/135 and cli/src/wire.h:268/271/274/277/282):
  - NAME = 0x4E 0x41 0x4D 0x45 ("NAME")
  - BOMB = 0x42 0x4F 0x4D 0x42 ("BOMB")
</canonical_layouts>

           uint64_t timestamp = TS_AUTO)
