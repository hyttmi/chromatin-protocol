```cpp
/// Encode a chunked header: [flags:1][type:1][request_id:4BE][total_size:8BE] = 14 bytes.
/// Optional extra_metadata appended after the 14-byte header (e.g., status byte for ReadResponse).
std::vector<uint8_t> encode_chunked_header(uint8_t type, uint32_t request_id,
                                            uint64_t total_payload_size,
                                            std::span<const uint8_t> extra_metadata = {});

/// Decoded chunked header.
struct ChunkedHeader {
    uint8_t type;
    uint32_t request_id;
    uint64_t total_payload_size;
    std::vector<uint8_t> extra_metadata;  // bytes after the 14-byte header
};

/// Decode a chunked header from raw bytes. Returns nullopt if too short or flags != CHUNKED_BEGIN.
std::optional<ChunkedHeader> decode_chunked_header(std::span<const uint8_t> data);
```

Implementation for encode: allocate vector of 14 + extra_metadata.size() bytes. Write CHUNKED_BEGIN at [0], type at [1], request_id as 4-byte BE at [2..5], total_payload_size as 8-byte BE at [6..13], copy extra_metadata at [14..].

Implementation for decode: check data.size() >= 14, check data[0] == CHUNKED_BEGIN, read type from [1], request_id from [2..5] as u32 BE, total_payload_size from [6..13] as u64 BE. If data.size() > 14, copy remainder into extra_metadata.

ChunkQueue (per D-04) — backpressure queue using asio::steady_timer as signal:
```cpp
struct ChunkQueue {
    explicit ChunkQueue(asio::any_io_executor executor)
        : signal_(executor) {}

    std::deque<std::vector<uint8_t>> chunks;
    asio::steady_timer signal_;
    bool closed = false;

    /// Producer: push chunk, co_await if queue full. Returns false if closed.
    asio::awaitable<bool> push(std::vector<uint8_t> chunk);

    /// Consumer: pop chunk, co_await if empty. Returns nullopt if closed and empty.
    asio::awaitable<std::optional<std::vector<uint8_t>>> pop();

    /// Close the queue (wakes all waiters).
    void close();
};
```

push() implementation: while (chunks.size() >= CHUNK_QUEUE_MAX_DEPTH && !closed), set signal_.expires_at(max), co_await signal_.async_wait(use_nothrow), check closed. If closed, return false. Push chunk, signal_.cancel() to wake consumer. Return true.

pop() implementation: while (chunks.empty() && !closed), set signal_.expires_at(max), co_await signal_.async_wait(use_nothrow). If chunks empty and closed, return nullopt. Move front chunk, pop_front, signal_.cancel() to wake producer. Return chunk.

close() implementation: set closed = true, signal_.cancel().

**Node-side chunked receive support** — the node must reassemble chunked sub-frames into a single complete message before dispatching to message_cb_ (per research recommendation: pre-MVP, node reassembles rather than streaming to storage).

**A. Add chunked detection to `Connection::message_loop()`:**

After `recv_encrypted()` returns a decrypted frame, before calling `TransportCodec::decode()`, check if the first byte is `CHUNKED_BEGIN` (0x01). FlatBuffer messages never start with 0x01 (they start with a 4-byte size prefix followed by the root table offset), so this is unambiguous.

Modified message_loop logic:
```cpp
asio::awaitable<void> Connection::message_loop() {
    while (!closed_) {
        auto msg = co_await recv_encrypted();
        if (!msg) {
            if (!closed_) spdlog::info("connection lost (read failed)");
            break;
        }

        last_recv_time_ = std::chrono::steady_clock::now();

        // Check for chunked sub-frame mode (first byte == 0x01)
        if (!msg->empty() && (*msg)[0] == 0x01) {
            // Chunked mode: reassemble all sub-frames into complete payload
            auto reassembled = co_await recv_chunked(*msg);
            if (!reassembled) {
                spdlog::warn("chunked reassembly failed");
                break;
            }
            // Dispatch reassembled message through message_cb_
            if (!authenticated_) {
                spdlog::error("received chunked message before auth from {}", remote_addr_);
                break;
            }
            if (message_cb_) {
                message_cb_(shared_from_this(),
                            static_cast<wire::TransportMsgType>(reassembled->type),
                            std::move(reassembled->payload),
                            reassembled->request_id);
            }
            continue;
        }

        // Existing non-chunked path (unchanged)
        auto decoded = TransportCodec::decode(*msg);
        if (!decoded) {
            spdlog::warn("received invalid transport message");
            continue;
        }
        // ... rest of existing switch(type) logic unchanged ...
    }
}
```

   ```cpp

       auto chunk = co_await recv_encrypted();
       if (!chunk) co_return std::nullopt;
       last_recv_time_ = std::chrono::steady_clock::now();
       // Zero-length sentinel = end of chunked sequence
       if (chunk->empty()) break;
       payload.insert(payload.end(), chunk->begin(), chunk->end());

       if (payload.size() > total_payload_size + 64) {  // small tolerance for metadata
           spdlog::error("chunked reassembly: exceeded declared size");
           co_return std::nullopt;
       }
   }
   ```
5. Return ReassembledChunked{type, request_id, std::move(payload)}.

**C. Add chunked send support to Connection for responses:**

The node also needs to SEND chunked responses (e.g., ReadResponse for large blobs). Add `send_message_chunked()`:

Declaration in `connection.h`:
```cpp
/// Send a large message as chunked sub-frames.
/// Used for payloads >= STREAMING_THRESHOLD.
asio::awaitable<bool> send_message_chunked(wire::TransportMsgType type,
                                            std::span<const uint8_t> payload,
                                            uint32_t request_id = 0,
                                            std::span<const uint8_t> extra_metadata = {});
```

Implementation in `connection.cpp`:
1. Build 14-byte chunked header: [0x01][type][request_id 4BE][total_size 8BE] + extra_metadata. Use chromatindb::util::store_u32_be and store_u64_be for encoding.
2. Enqueue the header frame via `enqueue_send` (which goes through `drain_send_queue` -> `send_encrypted`).
3. Loop over payload in STREAMING_THRESHOLD-sized chunks:
   ```cpp
   constexpr size_t CHUNK = STREAMING_THRESHOLD;  // 1 MiB
   size_t offset = 0;
   while (offset < payload.size()) {
       size_t len = std::min(CHUNK, payload.size() - offset);
       // Must encode chunk for enqueue_send (TransportCodec format not used — raw chunk)
       std::vector<uint8_t> chunk_data(payload.data() + offset, payload.data() + offset + len);
       bool ok = co_await enqueue_send(std::move(chunk_data));
       if (!ok) co_return false;
       offset += len;
   }
   ```
4. Send zero-length sentinel: `co_await enqueue_send(std::vector<uint8_t>{});`

Wait — `enqueue_send` currently expects pre-encrypted data that goes through drain_send_queue -> send_encrypted. But the drain_send_queue currently does `send_encrypted(msg.encoded)` where msg.encoded is a TransportCodec::encode() result. We need to make enqueue_send work with raw chunks that will be encrypted.

Actually, looking at the code more carefully: `enqueue_send` puts a `PendingMessage{encoded, ...}` on the deque, and `drain_send_queue` calls `send_encrypted(msg.encoded)`. The `send_encrypted` call does AEAD encrypt + send_raw. So enqueue_send's `encoded` field is the PRE-AEAD plaintext (NOT pre-encrypted — it gets encrypted by send_encrypted). This means we CAN use enqueue_send for raw chunk data — each chunk will be independently AEAD encrypted by send_encrypted with its own nonce.

However, there is a subtlety: `send_message()` currently calls `TransportCodec::encode()` and then `enqueue_send()`. For chunked mode, we skip TransportCodec and directly enqueue raw bytes (header, chunks, sentinel). Each of these goes through send_encrypted individually, consuming one nonce each. This correctly implements per-chunk AEAD (D-08) with shared nonce counter (D-09).

Modify `send_message()` to automatically use chunked mode for large payloads:
```cpp
asio::awaitable<bool> Connection::send_message(wire::TransportMsgType type,
                                                std::span<const uint8_t> payload,
                                                uint32_t request_id) {
    if (payload.size() >= STREAMING_THRESHOLD) {
        co_return co_await send_message_chunked(type, payload, request_id);
    }
    // Existing path: TransportCodec::encode() + enqueue_send()
    auto encoded = TransportCodec::encode(type, payload, request_id);
    co_return co_await enqueue_send(std::move(encoded));
}
```
