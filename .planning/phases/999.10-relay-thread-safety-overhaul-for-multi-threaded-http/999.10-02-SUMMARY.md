## Decisions Made

- Heavy work (JSON parsing, FlatBuffer encoding, hex decoding) intentionally stays off-strand per D-16 -- only shared state access enters the strand
- check_auth split into sync and async versions: sync for test dispatch path (no strand needed in single-threaded tests), async for production dispatch_async with strand posting
- dispatch_async falls back to check_auth_sync if strand_ is null, providing backward compatibility for tests that don't set up a strand
- ML-DSA-87 signature verification in /auth/verify runs on-strand after ChallengeStore::consume -- while CPU-heavy, the verify latency is small relative to I/O and avoids the complexity of re-posting to strand after off-strand verify
