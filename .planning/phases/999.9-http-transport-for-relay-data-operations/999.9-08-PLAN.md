    **SessionDispatch for HTTP** (key wiring):
    ```cpp
    core::SessionDispatch dispatch;
    dispatch.send_json = [&token_store, &promise_map](uint64_t session_id, const nlohmann::json& msg) {
        // Two paths: request/response or notification/broadcast
        // For request/response: the msg contains a request_id field (from UDS route_response)
        // But actually, the UDS multiplexer's route_response already resolved the request_id
        // and the send_json is for the session_id. We need to push to either:
        // (a) The SSE writer (for notifications) -- if the session has an sse_writer
        // (b) This case shouldn't happen for request/response -- those go through ResponsePromise

        // Actually, re-think: In the HTTP model, request/response goes through ResponsePromise,
        // NOT through send_json. The UDS multiplexer's route_response calls dispatch_.send_json
        // for the resolved response. We need to intercept this and resolve the ResponsePromise instead.

        // Better approach: Modify UdsMultiplexer's route_response to first check the ResponsePromiseMap.
        // If the relay_rid has a pending promise -> resolve it. Otherwise, fall through to send_json.
        // This means send_json is only called for notifications and broadcasts.

        auto* session = token_store.lookup_by_id(session_id);
        if (!session) return;
        if (session->sse_writer) {
            // Notification: push to SSE stream
            static uint64_t event_counter = 0;  // Per-session counter would be better
            session->sse_writer->push_event(msg, ++event_counter);
        }
        // If no SSE writer, notification is dropped (client hasn't connected SSE yet)
    };

    dispatch.broadcast = [&token_store](const nlohmann::json& msg) {
        token_store.for_each_mut([&msg](const std::string&, HttpSessionState& session) {
            if (session.sse_writer) {
                static uint64_t bc_counter = 0;
                session.sse_writer->push_broadcast(msg);
            }
        });
    };

    dispatch.send_error = dispatch.send_json;
    ```

    **CRITICAL**: The UdsMultiplexer::route_response currently calls send_json for ALL responses. For HTTP, request/response pairs need to go through ResponsePromise, not send_json. Modify route_response to first attempt ResponsePromiseMap::resolve(relay_rid, type, payload). If that returns true (promise found and resolved), skip the send_json call. If no promise found, fall through to dispatch_.send_json (for notifications/broadcasts).

    This means UdsMultiplexer needs a pointer to ResponsePromiseMap. Add: `void set_response_promises(http::ResponsePromiseMap* promises);` and a `http::ResponsePromiseMap* response_promises_ = nullptr;` member. In route_response: `if (response_promises_ && response_promises_->resolve(relay_rid, type, payload)) return;`.
