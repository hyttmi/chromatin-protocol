## Decisions Made
- HttpResponse implemented as header-only (inline methods) since it's pure data construction with no complex dependencies
- HttpRequest.version field stores raw version string to avoid bool flag ambiguity in finalize_headers
- parse_request_line uses rfind for version separator to handle edge cases with spaces in URIs
- TokenStore NOT thread-safe by design (same strand model as existing SessionManager)
