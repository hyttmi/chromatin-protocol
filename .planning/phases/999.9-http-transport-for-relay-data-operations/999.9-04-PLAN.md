<objective>
Create the ResponsePromise mechanism that bridges async UDS responses to synchronous-per-request HTTP handler coroutines.

Purpose: In WebSocket, responses arrive asynchronously via send_json() on the WsSession. In HTTP, each handler coroutine must wait for its specific response. The ResponsePromise is a one-shot awaitable: the handler registers it, co_awaits it, and the UDS response routing resolves it. This is the key architectural bridge between the async UDS model and the per-request HTTP model (RESEARCH Pitfall 3).
Output: response_promise.h (header-only), unit test.

    ```cpp
    #pragma once
    #include <asio.hpp>
    #include <cstdint>
    #include <optional>
    #include <vector>

    namespace chromatindb::relay::http {

    struct ResponseData {
        uint8_t type;
        std::vector<uint8_t> payload;
    };

    /// One-shot awaitable for HTTP handler coroutines waiting for a UDS response.
    /// Pattern: create -> register with pending map -> co_await wait() -> UDS route calls resolve()
    class ResponsePromise {
    public:
        explicit ResponsePromise(asio::any_io_executor executor)
            : timer_(executor) {
            timer_.expires_at(asio::steady_timer::time_point::max());
        }

        /// Wait for resolution or timeout. Returns nullopt on timeout/cancel.
        asio::awaitable<std::optional<ResponseData>> wait(std::chrono::seconds timeout) {
            timer_.expires_after(timeout);
            auto [ec] = co_await timer_.async_wait(asio::as_tuple(asio::use_awaitable));
            // ec == operation_aborted means timer was cancelled by resolve() or cancel()
            if (resolved_) co_return std::move(data_);
            co_return std::nullopt;  // Timeout or cancel without data
        }

        /// Resolve with response data. Wakes the waiting coroutine.
        void resolve(uint8_t type, std::vector<uint8_t> payload) {
            data_ = ResponseData{type, std::move(payload)};
            resolved_ = true;
            timer_.cancel();
        }

        /// Cancel without data (for disconnect cleanup).
        void cancel() {
            timer_.cancel();
        }

        bool is_resolved() const { return resolved_; }

    private:
        asio::steady_timer timer_;
        std::optional<ResponseData> data_;
        bool resolved_ = false;
    };

    /// Registry mapping relay_rid -> ResponsePromise for pending HTTP requests.
    /// NOT thread-safe -- accessed from io_context strand only.
    class ResponsePromiseMap {
    public:
        /// Register a promise for a relay request ID.
        void register_promise(uint32_t relay_rid, ResponsePromise* promise) {
            promises_[relay_rid] = promise;
        }

        /// Resolve a promise by relay_rid. Returns true if found and resolved.
        bool resolve(uint32_t relay_rid, uint8_t type, std::vector<uint8_t> payload) {
            auto it = promises_.find(relay_rid);
            if (it == promises_.end()) return false;
            it->second->resolve(type, std::move(payload));
            promises_.erase(it);
            return true;
        }

        /// Remove a promise (on timeout or cancel).
        void remove(uint32_t relay_rid) {
            promises_.erase(relay_rid);
        }

        /// Cancel all promises for cleanup.
        void cancel_all() {
            for (auto& [rid, promise] : promises_) {
                promise->cancel();
            }
            promises_.clear();
        }

        size_t size() const { return promises_.size(); }

    private:
        std::unordered_map<uint32_t, ResponsePromise*> promises_;
    };

    } // namespace chromatindb::relay::http
    ```

    The key insight: ResponsePromise stores a non-owning pointer in ResponsePromiseMap. The promise is stack-allocated by the HTTP handler coroutine. The coroutine's lifetime spans the full request, so the promise is valid for the duration. When the UDS response arrives, dispatch_.send_json resolves the promise. The HTTP handler wakes up and can now send the HTTP response.
