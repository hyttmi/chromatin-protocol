## Decisions Made
- Used std::function callbacks in a struct rather than a virtual base class -- simpler composition, no vtable overhead, easily constructed inline in relay_main.cpp
- send_error is separate from send_json to allow future transports to handle errors differently (e.g., HTTP 500 vs SSE error event), but defaults to send_json when not set
- Bridge in relay_main.cpp captures session_manager by reference, same lifetime semantics as before
