    **relay/http/sse_writer.h**:
    ```cpp
    class SseWriter {
    public:
        // Constructor takes a reference to the stream (TLS or plain, via type-erased async_write)
        // and the io_context for the heartbeat timer.
        using WriteFn = std::function<asio::awaitable<bool>(std::string_view data)>;

        SseWriter(asio::any_io_executor executor, WriteFn write_fn);

        /// Push a notification event. Thread-safe (posts to executor).
        void push_event(const nlohmann::json& data, uint64_t event_id);

        /// Push a broadcast event (no event ID).
        void push_broadcast(const nlohmann::json& data);

        /// Main drain + heartbeat coroutine. Call via co_spawn.
        asio::awaitable<void> run();

        /// Stop the writer.
        void close();

        bool is_closed() const { return closed_; }

        /// Format an SSE event string.
        static std::string format_event(const nlohmann::json& data, uint64_t event_id);
        static std::string format_heartbeat();
    };
    ```

    **relay/http/sse_writer.cpp**: Drain loop pattern (same as core::Session):
    1. While not closed:
       a. If events in queue, write them all
       b. If no events, set timer to 30s
       c. co_await timer (wakes on new event OR timer expiry)
       d. If timer expired (not cancelled) and queue empty -> send heartbeat
       e. If write fails -> set closed, break
    2. On exit: signal closure

    Event queue: `std::deque<std::string> events_`. Signal: `asio::steady_timer signal_`. Cap: 256 events (per existing send queue pattern). Overflow -> close (same as core::Session overflow behavior).

    WriteFn is a type-erased async write function. This decouples SseWriter from the TLS/plain stream variant -- the HttpConnection provides a WriteFn that internally does the get_if branching.

    **Test file**: Test format_event output format (SSE spec compliance). Test format_heartbeat. Test that push_event queues correctly. Test cap overflow. These are unit tests of formatting -- the actual async drain is harder to test in isolation, but format correctness is critical.

    **POST /subscribe handler** (D-22):
    1. Parse JSON body: `{"namespaces": ["hex64", ...]}`
    2. Validate: each namespace must be 64-char hex. Max 256 per session (per existing cap).
    3. Convert hex strings to Namespace32 arrays.
    4. Call tracker_.subscribe(session->session_id, namespaces).
    5. If new namespaces (first subscriber), forward Subscribe to node via UDS (same u16BE encoding as WsSession).
    6. Return HttpResponse::json(200, {"status": "ok", "subscribed": N}).

    **POST /unsubscribe handler** (D-23):
    1. Parse JSON body: `{"namespaces": ["hex64", ...]}`
    2. Call tracker_.unsubscribe(session->session_id, namespaces).
    3. If namespaces became empty (last subscriber), forward Unsubscribe to node via UDS.
    4. Return HttpResponse::json(200, {"status": "ok", "unsubscribed": N}).

    **GET /events?token=<token> handler** (D-24):
    This is NOT a normal request/response -- it's a long-lived SSE connection. It needs special handling in HttpConnection:

    1. Extract token from query string.
    2. Look up session in TokenStore (this endpoint is special -- auth is via query param, not header).
    3. If invalid token -> 401.
    4. Send SSE response headers:
       ```
       HTTP/1.1 200 OK\r\n
       Content-Type: text/event-stream\r\n
       Cache-Control: no-cache\r\n
       Connection: keep-alive\r\n
       \r\n
       ```
    5. Create SseWriter with a write function bound to the connection's stream.
    6. Register the SseWriter pointer in HttpSessionState (add `SseWriter* sse_writer = nullptr` field to HttpSessionState in token_store.h).
    7. co_spawn the SseWriter::run() coroutine.
    8. co_await the SseWriter to finish (blocks until disconnect or close).
    9. On exit: set session->sse_writer = nullptr, clean up subscriptions.

    **Update HttpSessionState** in token_store.h: Add `SseWriter* sse_writer = nullptr;` field. This is a non-owning pointer -- the SseWriter lives on the HttpConnection's coroutine stack.

    **Wire notification fan-out**: The SessionDispatch::send_json callback (set up in relay_main.cpp, to be updated in Plan 08) will look up the session by ID, check if it has an sse_writer, and call sse_writer->push_event(json, event_id). If no sse_writer -> the notification is dropped (the client hasn't connected an SSE stream yet).

    **Route registration**:
    ```
    POST /subscribe    -> handle_subscribe (authenticated)
    POST /unsubscribe  -> handle_unsubscribe (authenticated)
    GET  /events       -> handle_events (public -- auth via query param)
    ```

    The /events endpoint is marked as public_route in the router because auth happens via query param (D-04), not via Bearer header. The handler itself validates the token.
