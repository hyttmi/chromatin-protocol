    struct HttpRequest {
        std::string method;      // "GET", "POST", "DELETE"
        std::string path;        // "/blob/abc/def" (no query string)
        std::string query;       // "foo=bar" (after '?', empty if none)
        std::unordered_map<std::string, std::string> headers;  // lowercase keys
        size_t content_length = 0;
        bool keep_alive = true;  // HTTP/1.1 default
    };

    - `bool finalize_headers(HttpRequest& req)` -- set keep_alive based on Connection header + HTTP version

    **relay/http/http_response.h**: Header-only. Define HttpResponse struct:
    ```
    struct HttpResponse {
        uint16_t status = 200;
        std::string status_text = "OK";
        std::vector<std::pair<std::string, std::string>> headers;  // ordered pairs (not map -- order matters)
        std::vector<uint8_t> body;

        static HttpResponse json(uint16_t status, const nlohmann::json& j);
        static HttpResponse binary(uint16_t status, std::vector<uint8_t> data);
        static HttpResponse error(uint16_t status, const std::string& code, const std::string& message = "");
        static HttpResponse not_found();
        static HttpResponse no_content();  // 204

        std::string serialize() const;  // Full HTTP response as string
    };
    ```
    Status text lookup: just a switch for 200/201/204/400/401/403/404/405/409/413/500/502/503. Default "Unknown".
    serialize() produces: "HTTP/1.1 {status} {text}\r\n{headers}\r\n\r\n{body}"
    json() and error() auto-set Content-Type: application/json and Content-Length.
    binary() auto-sets Content-Type: application/octet-stream and Content-Length.

    **relay/http/http_parser.cpp**: Implement parse_request_line and parse_header_line. Use string_view::find for splitting (no regex). Header keys lowercased via std::transform. Connection header value also lowercased for comparison.

    Keep it minimal per D-03 and RESEARCH "Don't Hand-Roll" guidance. This is NOT a general-purpose HTTP parser -- it only handles the known request shapes from authenticated chromatindb clients.

    **relay/http/token_store.h**: Define HttpSessionState and TokenStore:
    ```
    struct HttpSessionState {
        uint64_t session_id;
        std::vector<uint8_t> client_pubkey;
        std::array<uint8_t, 32> client_namespace;
        core::RateLimiter rate_limiter;
        std::chrono::steady_clock::time_point last_activity;
        // SSE writer pointer set later (Plan 07)
    };

    class TokenStore {
    public:
        /// Create a new session. Returns hex token (64 chars).
        std::string create_session(std::vector<uint8_t> pubkey,
                                   std::array<uint8_t, 32> ns_hash,
                                   uint32_t rate_limit = 0);

        /// Look up session by token. Updates last_activity. Returns nullptr if not found.
        HttpSessionState* lookup(const std::string& token);

        /// Look up session by session_id. Returns nullptr if not found.
        HttpSessionState* lookup_by_id(uint64_t session_id);

        /// Get token for a session_id (for reverse lookup).
        const std::string* get_token(uint64_t session_id) const;

        /// Remove session by ID. Returns removed namespaces (empty for now).
        void remove_session(uint64_t session_id);

        /// Remove session by token.
        void remove_by_token(const std::string& token);

        /// Reap sessions idle longer than timeout. Returns number reaped.
        size_t reap_idle(std::chrono::seconds timeout);

        /// Number of active sessions.
        size_t count() const;

        /// Iterate all sessions.
        template<typename Fn>
        void for_each(Fn&& fn) const;

        /// Iterate all sessions (mutable).
        template<typename Fn>
        void for_each_mut(Fn&& fn);
    };
    ```
    Internal: `unordered_map<string, HttpSessionState> tokens_` (token -> state), `unordered_map<uint64_t, string> id_to_token_` (session_id -> token), `uint64_t next_id_ = 1`.

    **relay/http/token_store.cpp**: Implement. Token generation: `RAND_bytes(32)` -> `util::to_hex()`. The rate_limiter in HttpSessionState is constructed with the provided rate_limit parameter. last_activity set to steady_clock::now() on create and on lookup.

    NOT thread-safe (same strand model as SessionManager).
