## Decisions Made
- SseWriter is decoupled from transport via WriteFn (type-erased async write) -- HttpConnection provides the write function that handles TLS/plain branching
- SSE heartbeat uses SSE comment format (": heartbeat\n\n") which is ignored by EventSource but keeps connection alive
- GET /events endpoint is marked as public_route on the router because auth happens via token query parameter, not Bearer header -- the handler itself validates the token
- Subscribe/unsubscribe handlers are synchronous (not async) since SubscriptionTracker is in-process and doesn't need UDS round-trip for tracking
