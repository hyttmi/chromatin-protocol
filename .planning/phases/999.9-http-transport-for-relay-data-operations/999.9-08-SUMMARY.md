- `relay/core/metrics_collector.h` - Simplified: removed ioc, accept_loop, handle_connection, start/stop, set_metrics_bind

- `relay/core/uds_multiplexer.h` - Added set_response_promises() and response_promises_ member

- `relay/tests/test_metrics_collector.cpp` - Updated for simplified constructor and new metric names
- `relay/ws/session_manager.cpp` - Updated metric field names from ws_ to http_ prefix

## Decisions Made
- MetricsCollector constructor simplified to just `(const atomic<bool>& stopping)` -- no ioc or metrics_bind needed
- Metric field names renamed: ws_connections_total -> http_connections_total, ws_disconnections_total -> http_disconnections_total, ws_connections_active -> http_connections_active
- UdsMultiplexer::route_response checks ResponsePromiseMap BEFORE translate+send_json -- HTTP handlers receive raw type+payload and translate themselves
- Idle session reaper uses 60s check interval with 600s (10 min) idle timeout
- SIGTERM sequence: stop acceptor, cancel all promises, 2s drain timer, stop ioc
