<objective>
Decouple UdsMultiplexer from ws::SessionManager so it can work with any transport layer (HTTP or WebSocket).

Purpose: UdsMultiplexer currently calls ws::SessionManager::get_session() and WsSession::send_json() directly. This creates a hard dependency on the WebSocket transport. By introducing an abstract SessionDispatch interface, the same UdsMultiplexer works with both the old WS sessions (during transition) and the new HTTP sessions.
Output: session_dispatch.h interface, modified uds_multiplexer.h/cpp using the interface.
</objective>

    **Step 1: Create relay/core/session_dispatch.h**

    Define a transport-agnostic callback interface (per RESEARCH "UdsMultiplexer Decoupling Strategy"):
    ```cpp
    #pragma once
    #include <nlohmann/json_fwd.hpp>
    #include <cstdint>
    #include <functional>

    namespace chromatindb::relay::core {

    /// Transport-agnostic session dispatch interface.
    /// UdsMultiplexer calls these to route responses and notifications
    /// without knowing whether the transport is WebSocket or HTTP.
    struct SessionDispatch {
        /// Send JSON to a specific session. No-op if session not found.
        std::function<void(uint64_t session_id, const nlohmann::json& msg)> send_json;

        /// Broadcast JSON to ALL active sessions (for StorageFull, QuotaExceeded).
        std::function<void(const nlohmann::json& msg)> broadcast;

        /// Send JSON to a specific session by ID, with error context.
        /// Same as send_json but named distinctly for error paths.
        /// Falls back to send_json if not set.
        std::function<void(uint64_t session_id, const nlohmann::json& err)> send_error;
    };

    } // namespace chromatindb::relay::core
    ```

    **Step 2: Modify uds_multiplexer.h**

    Replace `#include "relay/ws/session_manager.h"` with `#include "relay/core/session_dispatch.h"`.
    Change constructor: replace `ws::SessionManager& sessions` with `SessionDispatch dispatch`.
    Change member: replace `ws::SessionManager& sessions_` with `SessionDispatch dispatch_`.
    Remove the forward declaration of ws::SessionManager if present.

    **Step 3: Modify uds_multiplexer.cpp**

    Remove `#include "relay/ws/ws_session.h"` and any ws/ includes.
    Replace every `sessions_.get_session(id)->send_json(json)` with `dispatch_.send_json(id, json)`.
    Replace every `sessions_.for_each(...)` broadcast with `dispatch_.broadcast(json)`.
    Replace error sends with `dispatch_.send_error(id, err)` (or dispatch_.send_json if send_error is null).

    Specifically replace these patterns:
    1. `sessions_.for_each([&json_opt](uint64_t, const auto& session) { session->send_json(*json_opt); })` -> `dispatch_.broadcast(*json_opt)`
    2. `auto session = sessions_.get_session(id); if (!session) return; session->send_json(json)` -> `dispatch_.send_json(id, json)`
    3. Error path sends -> `if (dispatch_.send_error) dispatch_.send_error(id, err); else dispatch_.send_json(id, err);`

    For handle_notification fan-out (subscribers loop): replace `sessions_.get_session(sid)->send_json(json)` with `dispatch_.send_json(sid, json)`.

    **Step 4: Update relay_main.cpp** (minimal bridge)

    The relay_main.cpp currently creates UdsMultiplexer with session_manager. Create a SessionDispatch that bridges to the existing ws::SessionManager:
    ```cpp
    core::SessionDispatch dispatch;
    dispatch.send_json = [&session_manager](uint64_t id, const nlohmann::json& msg) {
        auto session = session_manager.get_session(id);
        if (session) session->send_json(msg);
    };
    dispatch.broadcast = [&session_manager](const nlohmann::json& msg) {
        session_manager.for_each([&msg](uint64_t, const auto& session) {
            session->send_json(msg);
        });
    };
    dispatch.send_error = dispatch.send_json;  // Same behavior for now
    ```
    Pass `dispatch` to UdsMultiplexer constructor instead of `session_manager`.

    This ensures the existing WS-based relay still works identically after the refactor. In Plan 03+, the HTTP transport will provide its own SessionDispatch implementation.
