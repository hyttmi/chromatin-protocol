<objective>
Implement the data operation HTTP endpoints: blob write, read, delete, and batch read. These are the primary value of HTTP transport -- raw binary bodies eliminate base64 overhead.

Purpose: Data operations are the reason this phase exists. POST /blob sends raw FlatBuffer bytes to the node (no JSON wrapping). GET /blob returns raw bytes (no base64 encoding). This eliminates the base64 overhead and text frame size limit that made WS unsuitable for large blobs.
Output: handlers_data.h/cpp with blob write/read/delete/batch_read handlers, unit tests.
</objective>

    NOTE: These handlers need to be coroutines (they co_await ResponsePromise). The signature should actually be `asio::awaitable<HttpResponse>` or the router needs to support async handlers. Design decision: make handlers return `asio::awaitable<HttpResponse>` and have the router/connection await them. Update HttpRouter::Handler typedef to `std::function<asio::awaitable<HttpResponse>(...)>` if needed. If this is too disruptive, use a simpler approach: handlers directly produce the response after co_awaiting the promise, and HttpConnection calls them as coroutines.

    Actually, simpler approach: the handlers ARE called from within the HttpConnection coroutine, so they can co_await. Make the handler return type `asio::awaitable<HttpResponse>` and adjust the router dispatch to support this.

    **POST /blob handler** (per D-07):
    1. Check Content-Type is application/octet-stream (or accept anything -- the body IS the blob)
    2. Check body size against max_blob_size atomic (per FEAT-02). If too large -> 413 response.
    3. Body is raw FlatBuffer-encoded Data blob. Wire type = 8 (Data).
    4. Register request with RequestRouter: `relay_rid = router_.register_request(session->session_id, 0, 8)`
    5. Create ResponsePromise on stack, register in ResponsePromiseMap with relay_rid
    6. Encode via TransportCodec::encode(TransportMsgType_Data, body, relay_rid)
    7. Send via uds_mux_.send()
    8. co_await promise.wait(timeout) -- timeout from config atomic
    9. On response: expect WriteAck (type 30). Translate via binary_to_json(30, payload) -> JSON
    10. Record blob_hash in WriteTracker for source exclusion: extract blob_hash from the JSON ack, call write_tracker_.record(blob_hash, session->session_id)
    11. Return HttpResponse::json(200, json_ack)
    12. On timeout: return HttpResponse::error(504, "timeout")
    13. Cleanup: remove promise from map

    **GET /blob/{ns}/{hash} handler** (per D-08):
    1. Extract namespace (64-char hex) and hash (64-char hex) from path after "/blob/"
    2. Validate: both must be 64-char hex strings. If invalid -> 400.
    3. Build ReadRequest binary: namespace(32 bytes) + hash(32 bytes) = 64 bytes. Wire type = 9.
    4. Register request, create promise, encode via TransportCodec, send via UDS.
    5. co_await promise.wait(timeout)
    6. On response: expect ReadResponse (type 10). Check if the response indicates not-found (status byte).
       If not found -> HttpResponse::not_found()
       If found -> HttpResponse::binary(200, payload) -- return the raw FlatBuffer blob bytes as-is (application/octet-stream)
    7. On timeout: 504.

    **DELETE /blob/{ns}/{hash} handler** (per D-09):
    1. Extract namespace and hash from path (same as GET).
    2. Body is raw FlatBuffer-encoded tombstone blob. Wire type = 17 (Delete).
    3. Validate body is non-empty.
    4. Register, promise, encode TransportCodec::encode(TransportMsgType_Delete, body, relay_rid)
    5. co_await promise.wait(timeout)
    6. On response: expect DeleteAck (type 18). Translate via binary_to_json -> JSON.
    7. Record in WriteTracker for source exclusion (same as write).
    8. Return HttpResponse::json(200, json_ack)

    **POST /batch/read handler** (per D-14):
    1. Parse JSON body: `{"namespace": "<hex>", "hashes": ["<hex>", ...], "cap_bytes": N}`
    2. Build BatchReadRequest binary via json_to_binary (translator handles encoding).
    3. Register, promise, encode, send via UDS.
    4. co_await promise.wait(timeout)
    5. On response: expect BatchReadResponse (type 55). Translate via binary_to_json -> JSON.
    6. Per RESEARCH discretion recommendation: JSON with base64-encoded blobs. The translator already produces this format.
    7. Return HttpResponse::json(200, json_response)

    Path extraction helper: `extract_path_params(path, "/blob/")` splits remaining on '/' to get namespace and hash. Validate each is exactly 64 hex chars via a simple loop.
