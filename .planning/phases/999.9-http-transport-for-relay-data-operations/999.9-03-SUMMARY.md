## Decisions Made
- Body passed as separate `std::vector<uint8_t>` to handler (not inside HttpRequest) to enable zero-copy for large blob payloads in future plans
- ChallengeStore reaps expired challenges on each new challenge request rather than using a background timer -- adequate for auth traffic volume
- Auth verify runs synchronously (not thread pool offloaded) since ML-DSA-87 verify is ~2ms and HTTP request context is already per-connection
- Used `std::tuple` return type for async I/O methods (Asio `as_tuple` returns `std::tuple`, not `std::pair`)
