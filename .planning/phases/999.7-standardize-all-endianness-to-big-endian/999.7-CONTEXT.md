# Phase 999.7: Standardize All Endianness to Big-Endian - Context

**Gathered:** 2026-04-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Two LE encoding sites remain in the codebase while everything else uses BE. Pre-MVP, no backward compatibility needed, no SDK. Standardize both to BE, update PROTOCOL.md, and ensure all tests pass. The relay already uses BE everywhere — relay needs no endian changes but the translator's JSON schema field encodings for ttl/timestamp fields may reference the old LE format in comments/docs.

</domain>

<decisions>
## Implementation Decisions

### Signing Input (codec.cpp)
- **D-01:** Change `build_signing_input()` in `db/wire/codec.cpp` to encode ttl as BE uint32 and timestamp as BE uint64 instead of LE. Use `store_u32_be()` and `store_u64_be()` from `db/util/endian.h`.
- **D-02:** This changes the SHA3-256 hash of every signing input, which means all blob hashes change. Pre-MVP, no deployed data — this is acceptable.

### Auth Payload (auth_helpers.h)
- **D-03:** Change `encode_auth_payload()` to write pubkey_size as BE uint32 instead of LE. Use `store_u32_be()` or manual BE encoding.
- **D-04:** Change `decode_auth_payload()` to read pubkey_size as BE uint32 instead of LE.

### PROTOCOL.md
- **D-05:** Update PROTOCOL.md to reflect the change: remove all references to `ttl_le32`, `timestamp_le64`, and `pubkey_size (4B LE)`. Replace with `ttl_be32`, `timestamp_be64`, and `pubkey_size (4B BE)`. Update the endianness summary section.

### Relay
- **D-06:** Relay endian.h is already all BE — no relay code changes needed for the endian switch itself.
- **D-07:** The relay's JSON schema and translator use BE encoding for all wire fields (including ttl/timestamp in client-facing JSON translation). No changes needed in relay translation code — the relay never sees the signing input or auth payload internals.

### Tests
- **D-08:** All unit tests that verify signing input byte layout (test_codec.cpp) or auth payload encoding (test_auth_helpers.cpp) must be updated to expect BE byte order.
- **D-09:** Integration tests and test helpers (test_helpers.h, test_daemon.cpp, test_engine.cpp, test_sync_protocol.cpp) all call `build_signing_input()` — they don't hardcode byte order, so they should pass automatically after the codec change.
- **D-10:** The relay smoke test (tools/relay_smoke_test.cpp) connects through the relay which handles translation — no byte-order assumptions to update there.

### Comments and Documentation
- **D-11:** Remove all "IMPORTANT: Uses little-endian" comments from auth_helpers.h. Remove "little-endian" comments from codec.cpp. The codebase should have zero LE references after this phase.

### Claude's Discretion
- Whether to add a static_assert or compile-time check that the codebase has no LE encoding patterns
- Whether to delete any LE helper functions if they exist (none found — endian.h is BE-only)

</decisions>
