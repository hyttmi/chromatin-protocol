1. Create `bench/bench_main.cpp` as a standalone benchmark program (no Catch2, no external benchmark framework -- just raw timing with `<chrono>`).

2. Structure the benchmark binary with these benchmark groups:

**Crypto Operations:**
- SHA3-256 hash: 64 bytes, 1 KiB, 64 KiB, 1 MiB input sizes
- ML-DSA-87 keygen: single run (slow op)
- ML-DSA-87 sign: 64-byte message, pre-generated keypair
- ML-DSA-87 verify: 64-byte message, pre-generated keypair + signature
- ML-KEM-1024 keygen: single run
- ML-KEM-1024 encaps: pre-generated keypair
- ML-KEM-1024 decaps: pre-generated keypair + ciphertext
- ChaCha20-Poly1305 encrypt: 64 bytes, 1 KiB, 64 KiB, 1 MiB plaintext
- ChaCha20-Poly1305 decrypt: same sizes

**Data Path:**
- Blob ingest: sign + validate + store, with 1 KiB and 64 KiB data payloads
- Blob retrieval: get_blob by namespace + hash, 1 KiB and 64 KiB
- Blob encode/decode: FlatBuffer round-trip, 1 KiB payload

**Network Operations:**
- PQ handshake: full initiator + responder handshake dance (in-memory, no TCP)
  - Create HandshakeInitiator and HandshakeResponder with two NodeIdentity instances
  - Run the full 4-step protocol: start() -> receive_kem_pubkey() -> receive_kem_ciphertext() -> create_auth_message() -> verify_peer_auth() for both sides
  - This measures ML-KEM encaps/decaps + HKDF + ML-DSA sign/verify combined

3. Implementation approach:
- Use a `run_bench` helper function that takes a name, iteration count, and callable. Runs the callable N times, measures total wall-clock time with `std::chrono::high_resolution_clock`, computes ops/sec and avg latency.
- Warm-up: 2 iterations before timing (discarded).
- Iteration counts: crypto ops ~100 iterations, data path ~100 iterations, handshake ~10 iterations (it's expensive). SHA3-256 for large inputs ~100. Keygen ~10.
- Output format: markdown table with columns: | Benchmark | Iterations | Total (ms) | Avg (us) | Ops/sec |
- Print group headers as markdown `### Group Name`
- Use a temporary directory for storage (std::filesystem::temp_directory_path / "chromatindb_bench_XXXXXX"), clean up on exit.
- Include `<iostream>`, `<chrono>`, `<vector>`, `<random>`, `<filesystem>`, `<functional>` and all chromatindb headers needed.

4. Add to CMakeLists.txt:
```cmake
# =============================================================================

# =============================================================================

target_link_libraries(chromatindb_bench PRIVATE chromatindb_lib)
```
