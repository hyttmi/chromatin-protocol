<objective>
Add configurable max_ttl to Config, add ttl_exceeded IngestError variant, implement Step 0c TTL validation in BlobEngine::ingest(), remove BLOB_TTL_SECONDS constexpr, and wire max_ttl through main.cpp.

Purpose: Writers control blob lifetime via the TTL field already in the wire format. Operators bound it with a configurable max_ttl. This replaces the hardcoded BLOB_TTL_SECONDS constexpr with a flexible per-node policy.

Output: Config fields for max_ttl and tombstone_ttl, IngestError::ttl_exceeded, Step 0c TTL check in ingest(), updated main.cpp wiring, passing tests.
