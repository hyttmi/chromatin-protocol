  <action>
    Create directory `db/util/` and file `db/util/hex.h`. This is a header-only utility in namespace `chromatindb::util`.

    The header must contain these inline functions:

    1. `inline std::string to_hex(std::span<const uint8_t> bytes)` -- primary overload. Converts bytes to lowercase hex string. Uses `static constexpr char hex_chars[] = "0123456789abcdef"` lookup table, reserves `bytes.size() * 2`, iterates with `(b >> 4) & 0xF` and `b & 0xF`. This single overload covers all callers that pass `std::array<uint8_t, N>`, `std::span<const uint8_t, 32>`, or dynamic `std::span<const uint8_t>` because C++20 `std::span<const uint8_t>` implicitly constructs from all of these.

    2. `inline std::string to_hex(std::span<const uint8_t> bytes, size_t max_bytes)` -- truncating overload for logging. Same logic but only processes `std::min(bytes.size(), max_bytes)` bytes. Used by peer_manager.cpp for abbreviated key logging.

    3. `inline std::string from_hex(const std::string& hex_str)` -- returns `std::vector<uint8_t>`. Validates even length (throws `std::invalid_argument` with message "from_hex: odd-length hex string (N chars)" if odd). For each pair of hex chars, converts to byte using a hex_char_to_nibble helper. Throws `std::invalid_argument` with message "from_hex: invalid hex char 'X'" on bad input.

    4. `template <size_t N> inline std::array<uint8_t, N> from_hex_fixed(const std::string& hex_str)` -- returns fixed-size array. Validates length == N*2 (throws `std::invalid_argument` with message "from_hex_fixed: expected M hex chars, got K" if wrong). Uses same nibble conversion logic.

    Required includes: `<algorithm>`, `<array>`, `<cstdint>`, `<span>`, `<stdexcept>`, `<string>`, `<vector>`.

    Use `#pragma once` guard. All functions must be `inline` (or template, which is implicitly inline) to avoid ODR violations since this header will be included in multiple translation units that link into the same binary (chromatindb_tests links all test .cpp files).

    No CMake changes needed -- `chromatindb_lib` already has include directory `$<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/..>` which makes `#include "db/util/hex.h"` work from any target linking chromatindb_lib.
  </action>
