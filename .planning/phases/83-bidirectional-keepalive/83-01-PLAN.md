<objective>
Add bidirectional keepalive to chromatindb: the node sends Ping to every connected TCP peer every 30 seconds and disconnects any peer that produces 60 seconds of silence (2 missed keepalive cycles). Replace the existing passive inactivity_check_loop with this active probing mechanism.

Purpose: Dead TCP connections (half-open sockets where the remote host crashed or the network path died) can persist indefinitely without application-level keepalive. OS-level TCP keepalive has unpredictable timing across platforms. Active Ping/Pong heartbeat detects dead connections within a bounded 60-second window.

Output: Modified Connection class with last_recv_time_ tracking, new keepalive_loop() coroutine in PeerManager replacing inactivity_check_loop, unit tests proving both requirements.

```cpp
asio::awaitable<void> PeerManager::keepalive_loop() {
    static constexpr auto KEEPALIVE_INTERVAL = std::chrono::seconds(30);
    static constexpr auto KEEPALIVE_TIMEOUT  = std::chrono::seconds(60);

    while (!stopping_) {
        asio::steady_timer timer(ioc_);
        keepalive_timer_ = &timer;
        timer.expires_after(KEEPALIVE_INTERVAL);
        auto [ec] = co_await timer.async_wait(
            asio::as_tuple(asio::use_awaitable));
        keepalive_timer_ = nullptr;
        if (ec || stopping_) co_return;

        auto now = std::chrono::steady_clock::now();

        // Snapshot connections -- peers_ may change across co_await points
        std::vector<net::Connection::Ptr> tcp_peers;
        for (const auto& peer : peers_) {
            if (!peer.connection->is_uds()) {
                tcp_peers.push_back(peer.connection);
            }
        }

        // Phase 1: Check for dead peers (before sending new Pings)
        std::vector<net::Connection::Ptr> to_close;
        for (const auto& conn : tcp_peers) {
            auto silence = now - conn->last_recv_time();
            if (silence > KEEPALIVE_TIMEOUT) {
                auto secs = std::chrono::duration_cast<std::chrono::seconds>(silence).count();
                spdlog::warn("keepalive: disconnecting {} ({}s silent)",
                             conn->remote_address(), secs);
                to_close.push_back(conn);
            }
        }
        for (auto& conn : to_close) {
            conn->close();  // D-04: immediate TCP close, no Goodbye
        }

        // Phase 2: Send Ping to remaining live TCP peers
        for (const auto& conn : tcp_peers) {
            if (std::find(to_close.begin(), to_close.end(), conn) != to_close.end()) {
                continue;  // Already closed
            }
            std::span<const uint8_t> empty{};
            co_await conn->send_message(wire::TransportMsgType_Ping, empty);
        }
    }
}
```

Key implementation details:
- Uses `steady_clock` (monotonic, no NTP jumps) per research discretion recommendation
- Snapshots Connection::Ptr vector before iterating (Pattern 2 from research -- prevents iterator invalidation across co_await)
- Checks silence BEFORE sending new Pings (dead peers get closed first, then Pings go to live peers only)
- Skips UDS connections (clients, not peers -- Pattern 3)
- Uses `conn->close()` per D-04 (immediate close, no Goodbye)
- Uses `spdlog::warn` per research discretion recommendation (abnormal but expected in deployments)
