<!-- Prior art: the established safe pattern for variant access in coroutines -->
<!-- Source: relay/ws/ws_session.cpp lines 238-263 (commit 16e6caf) -->

Pattern for variant access in coroutine context (SAFE):
```cpp
// Use std::get_if for first alternative, std::get for known-safe fallback
auto* tcp_sock = std::get_if<asio::ip::tcp::socket>(&stream_);
if (tcp_sock) {
    // ... use tcp_sock directly ...
} else {
    auto& tls_stream = std::get<asio::ssl::stream<asio::ip::tcp::socket>>(stream_);
    // ... use tls_stream directly ...
}
```

Pattern that is UNSAFE in coroutine context:
```cpp
// DON'T DO THIS in a coroutine body:
std::visit([](auto& stream) -> asio::awaitable<void> {
    co_await stream.async_read(...);  // Lambda stack frame destroyed!
}, stream_);
```

Pattern that IS safe (synchronous, non-coroutine):
```cpp
// OK in non-coroutine function:
std::visit([this](auto& stream) {
    stream.shutdown(...);  // Synchronous, no co_await
}, stream_);
```
