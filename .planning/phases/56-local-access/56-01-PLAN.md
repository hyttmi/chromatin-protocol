**Connection refactor (db/net/connection.cpp):**
- Change constructor to accept `asio::generic::stream_protocol::socket` instead of `tcp::socket`. The existing `create_inbound`/`create_outbound` factories move their tcp::socket into a generic socket via the generic socket's converting constructor: `asio::generic::stream_protocol::socket(std::move(tcp_socket))`. Note: asio's generic socket constructor from a typed protocol socket may need explicit construction. Use: `asio::generic::stream_protocol::socket generic_sock(std::move(socket));` or `asio::generic::stream_protocol::socket(socket.get_executor(), asio::generic::stream_protocol(socket.local_endpoint().protocol().family(), socket.local_endpoint().protocol().type()), socket.release())`.

  Actually, the cleanest approach: change the private constructor to take `asio::generic::stream_protocol::socket`. For `create_inbound`/`create_outbound`, convert the tcp::socket by releasing the native handle and constructing a generic socket:
  ```cpp
  auto proto = asio::generic::stream_protocol(
      socket.local_endpoint().protocol().family(),
      socket.local_endpoint().protocol().type());
  asio::generic::stream_protocol::socket generic(socket.get_executor(), proto, socket.release());
  ```
  For `create_uds_inbound` and `create_uds_outbound`, do the same with the local::stream_protocol::socket.

- Implement `create_uds_outbound`: same pattern as `create_uds_inbound` but passes `is_initiator = true` to the Connection constructor (or sets it after construction). Sets `remote_addr_ = "uds"` and `is_uds_ = true`. This factory is needed by Plan 02's UDS test for the client side of a UDS connection.

- Update `close()`: change `socket_.shutdown(asio::ip::tcp::socket::shutdown_both, ec)` to `socket_.shutdown(asio::socket_base::shutdown_both, ec)` (base class constant, works for all protocols).

- Update constructor's `remote_endpoint` capture: for TCP sockets, `socket_.remote_endpoint()` returns a generic endpoint. Extract address via casting back to TCP. For UDS, set `remote_addr_` to `"uds"`. Use the `is_uds_` flag:
  ```cpp
  if (is_uds_) {
      remote_addr_ = "uds";
  } else {
      asio::error_code ec;
      auto ep = socket_.remote_endpoint(ec);
      if (!ec) {
          // generic endpoint -> convert to tcp
          asio::ip::tcp::endpoint tcp_ep(
              asio::ip::address::from_string(/* ... */));
          // Actually, with generic::stream_protocol, remote_endpoint() returns
          // asio::generic::stream_protocol::endpoint. We need to extract the IP.
          // Use the sockaddr data directly or just format the endpoint.
      }
  }
  ```

  Simpler approach for remote_addr_: Set it from the *caller* (the factory) before the connection runs. In `create_inbound`, capture `socket.remote_endpoint()` before moving. In `create_uds_inbound`/`create_uds_outbound`, set to `"uds"`. Modify factories:
  ```cpp
  Ptr Connection::create_inbound(asio::ip::tcp::socket socket,
                                  const identity::NodeIdentity& identity) {
      std::string addr;
      asio::error_code ec;
      auto ep = socket.remote_endpoint(ec);
      if (!ec) addr = ep.address().to_string() + ":" + std::to_string(ep.port());
      // Convert to generic socket
      // ...
      auto conn = Ptr(new Connection(std::move(generic), identity, false));
      conn->remote_addr_ = std::move(addr);
      return conn;
  }
  ```
  Remove the remote_endpoint capture from the Connection constructor since it won't work on generic sockets.

- Update `do_handshake()`: The trust check uses `socket_.remote_endpoint()` to get the peer's IP. For UDS connections (`is_uds_`), skip this and set `peer_is_trusted = true` unconditionally (UDS is always trusted per CONTEXT.md decision). This is clean — UDS = always trusted, always TrustedHello path:
  ```cpp
  bool peer_is_trusted;
  if (is_uds_) {
      peer_is_trusted = true;
  } else {
      asio::error_code ec;
      asio::ip::tcp::endpoint tcp_ep;
      // Reconstruct TCP endpoint from generic endpoint
      auto gen_ep = socket_.remote_endpoint(ec);
      // ... extract address ...
      peer_is_trusted = !ec && trust_check_ && trust_check_(/* address */);
  }
  ```

  Actually, the simplest approach: capture the remote address as `asio::ip::address` in the factory, store it as `std::optional<asio::ip::address> remote_ip_;` private member. For TCP: set it. For UDS: leave as nullopt. In do_handshake:
  ```cpp
  bool peer_is_trusted = is_uds_ || (remote_ip_ && trust_check_ && trust_check_(*remote_ip_));
  ```
  This avoids extracting IP from generic socket entirely. Add `std::optional<asio::ip::address> remote_ip_;` private member.

**IMPORTANT**: The `send_raw` / `recv_raw` / `send_encrypted` / `recv_encrypted` methods use `asio::async_write(socket_, ...)` and `asio::async_read(socket_, ...)` — these work unchanged with `generic::stream_protocol::socket` because it satisfies AsyncReadStream/AsyncWriteStream concepts.

**Create db/net/uds_acceptor.h:**
```cpp
#pragma once

#include "db/identity/identity.h"
#include "db/net/connection.h"

#include <asio.hpp>
#include <asio/local/stream_protocol.hpp>

#include <functional>
#include <string>
#include <vector>

namespace chromatindb::net {

/// Unix Domain Socket acceptor: listens on a local socket path.
/// Produces Connection objects using TrustedHello (always-trusted, no PQ).
class UdsAcceptor {
public:
    using ConnectionCallback = std::function<void(Connection::Ptr)>;
    using AcceptFilter = std::function<bool()>;

    UdsAcceptor(const std::string& path,
                const identity::NodeIdentity& identity,
                asio::io_context& ioc);

    /// Start accepting UDS connections. Unlinks stale socket if needed.
    void start();

    /// Stop accepting and unlink the socket file.
    void stop();

    /// Set callback for new authenticated connections.
    void set_on_connected(ConnectionCallback cb);

    /// Set callback for disconnected connections.
    void set_on_disconnected(ConnectionCallback cb);

    /// Set accept filter (connection limit).
    void set_accept_filter(AcceptFilter cb);

    /// Set thread pool for crypto offload (lightweight keys still need HKDF).
    void set_pool(asio::thread_pool& pool);

    /// Number of active UDS connections.
    size_t connection_count() const;

    /// Whether acceptor is running.
    bool is_running() const { return running_; }

private:
    asio::awaitable<void> accept_loop();
    void remove_connection(Connection::Ptr conn);

    std::string path_;
    const identity::NodeIdentity& identity_;
    asio::io_context& ioc_;
    asio::local::stream_protocol::acceptor acceptor_;
    std::vector<Connection::Ptr> connections_;
    bool running_ = false;

    ConnectionCallback on_connected_;
    ConnectionCallback on_disconnected_;
    AcceptFilter accept_filter_;
    asio::thread_pool* pool_ = nullptr;
};

} // namespace chromatindb::net
```

**Create db/net/uds_acceptor.cpp:**
- Constructor: initialize acceptor with ioc, store path, identity.
- `start()`:
  1. Try to bind. If bind fails with `address_in_use`, try `::unlink(path_.c_str())` then bind again. Log: `"unlinking stale UDS socket: {}"`. If second bind fails, throw.
  2. Set permissions: `::chmod(path_.c_str(), 0660)` after bind.
  3. Listen (backlog default).
  4. Set `running_ = true`.
  5. Log: `"UDS listening on {}"`.
  6. `co_spawn(ioc_, accept_loop(), detached)`.

- `accept_loop()`: Similar to Server::accept_loop but for UDS:
  1. While running:
  2. `co_await acceptor_.async_accept(use_nothrow)`
  3. Check accept_filter_ (same pattern as TCP server).
  4. Create connection via `Connection::create_uds_inbound(std::move(socket), identity_)`.
  5. Do NOT set trust_check — UDS connections have `is_uds_ = true` which bypasses trust check.
  6. Set pool_ if available.
  7. Add to connections_.
  8. Set on_close callback (remove_connection + on_disconnected_).
  9. Set on_ready callback (on_connected_).
  10. co_spawn conn->run().

- `stop()`:
  1. Set running_ = false.
  2. Close acceptor (cancel pending accept).
  3. Close all active connections.
  4. Unlink socket file: `::unlink(path_.c_str())`.
  5. Log: `"UDS acceptor stopped"`.

- `remove_connection()`: Same pattern as Server — erase from connections_ vector.
- `connection_count()`: return connections_.size().
