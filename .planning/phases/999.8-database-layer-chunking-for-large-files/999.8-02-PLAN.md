**Part B: store_chunked implementation in engine.cpp**

Add `#include "db/engine/chunking.h"` at top.

Implement `BlobEngine::store_chunked()`:

1. **Split data into chunk payloads** (per D-01, D-02):
   ```
   for offset in [0, CHUNK_SIZE, 2*CHUNK_SIZE, ...]:
     len = min(CHUNK_SIZE, data.size() - offset)
     chunk_payloads.push_back(data[offset..offset+len])
   ```

2. **Build, sign, encode each chunk** (per D-06):
   For each chunk payload:
   - Build `wire::BlobData` with namespace_id, pubkey, chunk_payload, ttl, timestamp
   - Compute `signing_input = wire::build_signing_input(namespace_id, chunk_payload, ttl, timestamp)`
   - Sign via callback: `signature = co_await crypto::offload(pool_, [&]{ return signature_fn(signing_input); });`
   - Set blob.signature
   - Encode: `auto encoded = wire::encode_blob(chunk_blob)`
   - Hash: `auto hash = wire::blob_hash(encoded)`
   - Store in `std::vector<storage::PrecomputedBlob>` and collect hash in `chunk_hashes`

   IMPORTANT: Each signing offload is a co_await. The signature_fn captures the identity's sign method. Since signing is non-deterministic (ML-DSA-87), each call produces a unique signature, and thus a unique blob_hash. This is fine -- chunks are content-addressed by their FlatBuffer encoding which includes the signature.

3. **Build manifest** (per D-03, D-07):
   - `auto manifest_data_bytes = make_manifest_data(chunk_hashes)`
   - Build manifest BlobData with same namespace_id, pubkey, ttl, timestamp, data=manifest_data_bytes
   - Sign manifest same way as chunks
   - Encode and hash manifest
   - Append to precomputed vector

4. **Atomic store** (per D-11):
   - `auto [byte_limit, count_limit] = effective_quota(namespace_id)`
   - `auto results = storage_.store_blobs_atomic(precomputed, max_storage_bytes_, byte_limit, count_limit)`
   - Check the LAST result (manifest) for success. If any result is CapacityExceeded/QuotaExceeded/Error, return rejection with appropriate IngestError.
   - On success, build IngestResult::success with WriteAck containing manifest blob_hash and manifest seq_num.

**Part C: read_chunked implementation in engine.cpp**

Implement `BlobEngine::read_chunked()`:

1. **Fetch manifest blob**: `auto manifest_opt = storage_.get_blob(namespace_id, manifest_hash)`
   - If nullopt: return error "manifest not found"

2. **Parse manifest**: `auto hashes_opt = parse_manifest(manifest_opt->data)`
   - If nullopt: return error "not a manifest blob" or "corrupt manifest"

3. **Set chunks_expected**: `result.chunks_expected = hashes_opt->size()`

4. **Fetch each chunk in order**:
   ```
   std::vector<uint8_t> reassembled;
   for (const auto& chunk_hash : *hashes_opt) {
       auto chunk_opt = storage_.get_blob(namespace_id, chunk_hash);
       if (!chunk_opt) {
           result.error = "missing chunk " + to_hex(chunk_hash);
           return result;  // chunks_found already set
       }
       result.chunks_found++;
       reassembled.insert(reassembled.end(), chunk_opt->data.begin(), chunk_opt->data.end());
   }
   ```

5. **Return success**: set result.success=true, result.data=reassembled

Note: read_chunked is NOT a coroutine (no crypto offload needed). Storage::get_blob is synchronous. This matches the existing get_blob pattern in BlobEngine.

```
### Manifest (Chunked File)

Magic prefix: `0x43 0x48 0x4E 0x4B` ("CHNK")

Data format: `[magic:4][chunk_count:4 BE][chunk_hash_1:32][chunk_hash_2:32]...`

A manifest blob is a regular signed blob whose data field contains an ordered list of chunk content hashes. Each chunk is also a regular signed blob containing a portion of the original file data. Chunks are 1 MiB (1,048,576 bytes) except for the last which may be smaller.

All chunks and the manifest share the same namespace, TTL, and timestamp. The manifest's blob_hash serves as the handle for the entire chunked file.

Maximum chunk count: 100,000 (~97.6 GiB files).
```
