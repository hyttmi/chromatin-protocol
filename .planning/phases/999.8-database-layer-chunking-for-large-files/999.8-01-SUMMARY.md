## Decisions Made
- Manifest magic placed in chromatindb::engine namespace (not wire/codec) since chunking is an engine-layer convention, not a wire-format primitive like tombstones/delegations
- store_blobs_atomic accumulates quota per-namespace then writes once at end (not per-blob), matching the batch optimization pattern
- Mixed dedup results: duplicate blobs in a batch get Duplicate status while new blobs still store successfully -- consistent with single-blob store_blob behavior
