<objective>
Create the foundation layer for chunked blob storage: manifest format utilities and atomic multi-blob storage.

Purpose: Before the engine can split/reassemble large files, it needs (1) a manifest format to list chunk hashes, and (2) a storage method that writes N blobs atomically in a single MDBX transaction. These are independent primitives that the engine's store_chunked/read_chunked will consume in Plan 02.

Output: chunking.h/cpp with manifest encode/decode, storage.h/cpp with store_blobs_atomic(), and unit tests for both.

**Part B: Atomic multi-blob storage (CHUNK-03)**

Add to `db/storage/storage.h`:
```cpp
/// Pre-computed blob for atomic multi-blob storage.
/// Caller has already encoded and hashed the blob.
struct PrecomputedBlob {
    wire::BlobData blob;
    std::array<uint8_t, 32> content_hash;
    std::vector<uint8_t> encoded;  // FlatBuffer-encoded bytes
};

/// Store multiple blobs atomically in a single write transaction.
/// All-or-nothing: if any blob fails capacity/quota check, none are stored.
/// Dedup is per-blob (duplicate blobs get Duplicate status, others still stored).
/// @param blobs Pre-computed blobs to store (chunks + manifest).
/// @param max_storage_bytes Capacity limit (0 = unlimited).
/// @param quota_byte_limit Namespace byte quota (0 = unlimited).
/// @param quota_count_limit Namespace blob count quota (0 = unlimited).
/// @return Vector of StoreResults (one per blob, same order as input).
std::vector<StoreResult> store_blobs_atomic(
    const std::vector<PrecomputedBlob>& blobs,
    uint64_t max_storage_bytes = 0,
    uint64_t quota_byte_limit = 0,
    uint64_t quota_count_limit = 0);
```

Implement `store_blobs_atomic` in `db/storage/storage.cpp`:
1. Open ONE write transaction: `auto txn = impl_->env.start_write();`
2. First pass: check dedup for each blob. Mark duplicates (find existing seq_num same as store_blob does). Count non-duplicate blobs.
3. Capacity check: if max_storage_bytes > 0, check `used_bytes() >= max_storage_bytes` (tombstones exempt). If fails, abort txn, return all CapacityExceeded.
4. Quota check: sum encrypted sizes of all non-duplicate non-tombstone blobs. Read current quota from quota_map. If byte or count limit exceeded, abort txn, return all QuotaExceeded.
5. For each non-duplicate blob: encrypt via `impl_->encrypt_value()`, upsert into blobs_map, assign seq_num via `impl_->next_seq_num()`, upsert into seq_map, upsert into expiry_map if TTL > 0, handle delegation_map/tombstone_map if applicable.
6. Update quota_map once at end with accumulated totals.
7. Commit transaction.
8. Return results vector (Stored with seq_num+hash for new, Duplicate with existing seq_num for dedup).

CRITICAL: The encrypt_value and next_seq_num are private Impl methods. store_blobs_atomic must be a member of Storage (not a free function) to access impl_->. Follow the exact same patterns as the existing store_blob(precomputed, limits) overload (lines 381-540 of storage.cpp).
