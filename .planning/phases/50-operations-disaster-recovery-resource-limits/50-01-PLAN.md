- Phase 2: Edit Node1 config to set rate_limit_bytes_per_sec=1024, rate_limit_burst=2048 (very tight: 1KB/s). Send SIGHUP. Verify "SIGHUP received" in logs. Verify "config reload" in logs (rate limit updated).
- Phase 3: Run high-rate loadgen (--size 10240 --rate 50 --count 100) targeting Node1. This should exceed 1KB/s immediately. Verify Node1 logs show "rate limit" or "rate_limited" disconnect. Verify rate_limited counter > 0 via SIGUSR1 metrics.
- Follow ACL-05 patterns: tmpdir configs, named volumes, trap cleanup EXIT.
- Use named volumes for identity persistence.

- Verify blobs > 0 (we ingested 10), uptime > 0 (running for some time).
