<objective>
Ship the read side of chunked large files plus post-reassembly integrity verification plus the end-to-end live-node checkpoint. After this plan, `cdb get <manifest_hash>` on a CPAR blob transparently fans out N pipelined ReadRequests across the Phase 120 primitives (depth=8), pwrites each decrypted chunk at its correct offset as it arrives (D-12, out-of-order arrival safe by construction), and after all chunks land re-reads the output file through a `Sha3Hasher` to confirm the plaintext matches `manifest.plaintext_sha3` (CHUNK-05 defense-in-depth). Any failure — send retry exhausted, decrypt failure, SHA3 mismatch — leaves the filesystem clean via `unlink()`. Dispatch into the chunked path is a 4-byte prefix check in `cmd::get` before envelope::decrypt.

This plan depends on Plan 01 (manifest codec, Sha3Hasher, chunked.h scaffolding, connection.cpp total_size clamp, cmd::put + cmd::rm dispatch). The three existing user-visible chunked entry points (`cdb put`, `cdb rm`) were delivered in Plan 01; `cdb get` is delivered here. The live-node checkpoint at the end is the first moment the full loop (put a 5 GiB file, get it back byte-identical, rm the manifest and its 320 chunks) runs against a real node — no automated test can exercise that, and the research document explicitly flagged this as a blocking gate (Focus Area 9 and the project-memory "would I use this myself?" filter).

```cpp
std::vector<std::array<uint8_t, 32>> plan_chunk_read_targets(const ManifestData& manifest) {
    std::vector<std::array<uint8_t, 32>> out;
    out.reserve(manifest.segment_count);
    for (uint32_t i = 0; i < manifest.segment_count; ++i) {
        std::array<uint8_t, 32> h{};
        std::memcpy(h.data(), manifest.chunk_hashes.data() + i * 32, 32);
        out.push_back(h);
    }
    return out;
}
```

Implement `verify_plaintext_sha3`:

```cpp
bool verify_plaintext_sha3(const std::string& path,
                           std::span<const uint8_t, 32> expected) {
    std::ifstream f(path, std::ios::binary);
    if (!f) return false;
    Sha3Hasher hasher;
    std::vector<uint8_t> buf(CHUNK_SIZE_BYTES_DEFAULT);
    while (f) {
        f.read(reinterpret_cast<char*>(buf.data()),
               static_cast<std::streamsize>(buf.size()));
        auto got = static_cast<std::size_t>(f.gcount());
        if (got == 0) break;
        hasher.absorb(std::span<const uint8_t>(buf.data(), got));
    }
    if (f.bad()) return false;
    auto actual = hasher.finalize();
    return std::memcmp(actual.data(), expected.data(), 32) == 0;
}
```

Implement `get_chunked`:

```cpp
int get_chunked(
    const Identity& id,
    std::span<const uint8_t, 32> ns,
    const ManifestData& manifest,
    const std::string& out_path,
    bool force_overwrite,
    Connection& conn,
    const cmd::ConnectOpts& opts) {

    namespace fs = std::filesystem;

    // Pre-flight: overwrite guard via the factored helper (D-12 semantics).
    // MUST be the first statement — no Connection or fd access before this.
    // The helper is the SINGLE place this check lives; if it returns false the
    // caller must not touch `conn` or `out_path` further than unlink-on-error
    // (which is moot because nothing was written yet).
    if (!refuse_if_exists(out_path, force_overwrite)) {
        return 1;
    }

    // Open + preallocate (Focus Area 3: ftruncate up front so we learn about
    // disk-full before we start fetching).
    int flags = O_WRONLY | O_CREAT | (force_overwrite ? O_TRUNC : O_EXCL);
    int fd = ::open(out_path.c_str(), flags, 0644);
    if (fd < 0) {
        std::fprintf(stderr, "Error: cannot open %s for write: %s\n",
                     out_path.c_str(), std::strerror(errno));
        return 1;
    }
    if (::ftruncate(fd, static_cast<off_t>(manifest.total_plaintext_bytes)) != 0) {
        std::fprintf(stderr, "Error: cannot preallocate %s (%llu bytes): %s\n",
                     out_path.c_str(),
                     (unsigned long long)manifest.total_plaintext_bytes,
                     std::strerror(errno));
        ::close(fd);
        ::unlink(out_path.c_str());
        return 1;
    }

    auto fail_unlink = [&](const char* where, const std::string& detail = {}) -> int {
        std::fprintf(stderr, "Error: %s%s%s\n", where,
                     detail.empty() ? "" : ": ", detail.c_str());
        ::close(fd);
        ::unlink(out_path.c_str());
        return 1;
    };

    const auto targets = plan_chunk_read_targets(manifest);
    const uint32_t N   = manifest.segment_count;

    // Two-phase pipeline mirroring cmd::get (commands.cpp:665-809) and
    // chunked::put_chunked (chunked.cpp). rid_to_chunk_index binds rid to
    // chunk_index at send time; on retry a new rid is allocated but the same
    // chunk_hash is resent.
    uint32_t rid = 1;
    std::unordered_map<uint32_t, uint32_t> rid_to_chunk_index;
    // attempts_per_chunk lets us cap retries per D-15 without global state.
    std::vector<int> attempts(N, 0);
    uint32_t next = 0;
    uint32_t completed = 0;

    auto send_chunk_read = [&](uint32_t chunk_idx) -> std::optional<uint32_t> {
        std::vector<uint8_t> payload(64);
        std::memcpy(payload.data(), ns.data(), 32);
        std::memcpy(payload.data() + 32, targets[chunk_idx].data(), 32);
        for (int attempt = 0; attempt < RETRY_ATTEMPTS; ++attempt) {
            if (attempt > 0) {
                std::this_thread::sleep_for(
                    std::chrono::milliseconds(RETRY_BACKOFF_MS[attempt]));
            }
            uint32_t this_rid = rid++;
            if (conn.send_async(MsgType::ReadRequest, payload, this_rid)) {
                return this_rid;
            }
        }
        return std::nullopt;
    };

    auto retry_chunk = [&](uint32_t chunk_idx) -> bool {
        if (++attempts[chunk_idx] >= RETRY_ATTEMPTS) return false;
        std::this_thread::sleep_for(
            std::chrono::milliseconds(RETRY_BACKOFF_MS[attempts[chunk_idx]]));
        auto r = send_chunk_read(chunk_idx);
        if (!r) return false;
        rid_to_chunk_index[*r] = chunk_idx;
        return true;
    };

    while (completed < N) {
        // Phase A: greedy fill.
        if (next < N && rid_to_chunk_index.size() < Connection::kPipelineDepth) {
            auto r = send_chunk_read(next);
            if (!r) {
                return fail_unlink("initial send failed", std::to_string(next));
            }
            rid_to_chunk_index[*r] = next;
            ++next;
            continue;
        }

        // Phase B: drain one reply in arrival order (D-08).
        auto resp = conn.recv();
        if (!resp) {
            return fail_unlink("connection lost during chunked download");
        }
        auto it = rid_to_chunk_index.find(resp->request_id);
        if (it == rid_to_chunk_index.end()) {
            spdlog::debug("get_chunked: discarding reply for unknown rid {}",
                          resp->request_id);
            continue;
        }
        const uint32_t chunk_idx = it->second;
        rid_to_chunk_index.erase(it);

        // Response validation.
        if (resp->type != static_cast<uint8_t>(MsgType::ReadResponse) ||
            resp->payload.empty() || resp->payload[0] != 0x01) {
            if (retry_chunk(chunk_idx)) continue;
            return fail_unlink("chunk read failed", std::to_string(chunk_idx));
        }
        auto blob_bytes = std::span<const uint8_t>(
            resp->payload.data() + 1, resp->payload.size() - 1);
        auto blob = decode_blob(blob_bytes);
        if (!blob) {
            if (retry_chunk(chunk_idx)) continue;
            return fail_unlink("chunk decode failed", std::to_string(chunk_idx));
        }

        // D-13 outer magic sanity: blob.data starts with CDAT_MAGIC. Defense
        // in depth; if the node returned the wrong blob_type, bail.
        if (blob->data.size() < 4 ||
            std::memcmp(blob->data.data(), CDAT_MAGIC.data(), 4) != 0) {
            return fail_unlink("chunk not tagged CDAT", std::to_string(chunk_idx));
        }
        auto cenv_bytes = std::span<const uint8_t>(
            blob->data.data() + 4, blob->data.size() - 4);
        auto pt = envelope::decrypt(cenv_bytes, id.kem_seckey(), id.kem_pubkey());
        if (!pt) {
            return fail_unlink("chunk decrypt failed", std::to_string(chunk_idx));
        }

        // Bounds sanity: plaintext length must match the expected chunk size
        // (or total_plaintext_bytes minus prior-chunks for the last chunk).
        const uint64_t expected_off = static_cast<uint64_t>(chunk_idx) *
                                       manifest.chunk_size_bytes;
        uint64_t expected_len = manifest.chunk_size_bytes;
        if (chunk_idx + 1 == N) {
            expected_len = manifest.total_plaintext_bytes - expected_off;
        }
        if (pt->size() != expected_len) {
            return fail_unlink("chunk plaintext length mismatch",
                               std::to_string(chunk_idx));
        }

        // D-12: pwrite at offset; out-of-order arrival safe.
        ssize_t w = ::pwrite(fd, pt->data(), pt->size(),
                             static_cast<off_t>(expected_off));
        if (w < 0 || static_cast<uint64_t>(w) != pt->size()) {
            return fail_unlink("pwrite failed", std::strerror(errno));
        }
        ++completed;
        if (!opts.quiet) {
            std::fprintf(stderr, "chunk %u/%u saved\n", completed, N);
        }
    }

    // Flush + close before re-reading for the integrity check.
    if (::fsync(fd) != 0) {
        // fsync failure is unusual; fall through to the re-read anyway — if
        // the data is genuinely wrong the sha3 check will catch it.
        spdlog::warn("get_chunked: fsync {} returned errno={}", out_path, errno);
    }
    ::close(fd);

    // CHUNK-05 / D-04 defense-in-depth: recompute SHA3-256 over the output
    // file and compare to manifest.plaintext_sha3. Mismatch => unlink.
    if (!verify_plaintext_sha3(out_path, manifest.plaintext_sha3)) {
        std::fprintf(stderr, "Error: plaintext_sha3 mismatch — aborting and removing %s\n",
                     out_path.c_str());
        ::unlink(out_path.c_str());
        return 1;
    }

    if (!opts.quiet) {
        std::fprintf(stderr, "saved: %s (%llu bytes, %u chunks)\n",
                     out_path.c_str(),
                     (unsigned long long)manifest.total_plaintext_bytes, N);
    }
    return 0;
}
```

```cpp
TEST_CASE("chunked: pwrite out-of-order lands chunks at correct offsets", "[chunked]") {
    auto tmp = std::filesystem::temp_directory_path() / "chunked_pwrite_test";
    std::filesystem::remove(tmp);
```

<threat_model>
## Trust Boundaries

Inherits all trust boundaries from Phase 120 (pipelining substrate) and Plan 01 (streaming reader, chunked-framing clamp, outer-magic placement). Plan 02 adds:

| Boundary | Description |
|----------|-------------|
| node -> cdb process (chunk payloads) | Every ReadResponse payload is AEAD-decrypted by Connection::recv before reaching get_chunked. The envelope::decrypt inside get_chunked provides a second AEAD layer (per-recipient key wrap). |
| cdb process -> local disk (output file) | get_chunked holds an open fd for the duration of the download; pwrites at chunk_size-aligned offsets; fsyncs before integrity verify. |
| node -> cdb process (manifest chunk_hashes) | The node's stored CPAR manifest could in principle point at ANY chunk_hashes, including hashes for blobs outside the caller's namespace. |

## STRIDE Threat Register (Plan 02 additions)

Inherits T-119-01..T-119-09 from Plan 01. New threats introduced by the read path:

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-119-10 | Tampering | Mid-download attacker swaps one chunk for a valid-but-different chunk (P-119-03) | mitigate | Per-chunk envelope AEAD authenticates the sender; the attacker must have been a valid recipient of some chunk to produce a passing CDAT blob. Even then, the whole-file SHA3-256 check against manifest.plaintext_sha3 catches any content substitution at the end. On mismatch, get_chunked unlinks the output and returns non-zero. Test: `chunked: verify_plaintext_sha3 rejects wrong hash`. |
| T-119-11 | Denial of Service (disk) | Malicious manifest.total_plaintext_bytes drives ftruncate too large | mitigate | decode_manifest_payload (Plan 01) clamps `total_plaintext_bytes <= segment_count * chunk_size_bytes`; segment_count <= MAX_CHUNKS (65536); chunk_size_bytes <= CHUNK_SIZE_BYTES_MAX (256 MiB). Product is bounded by 256 MiB * 65536 = 16 TiB theoretical, 1 TiB enforced by put-side D-14. If the user's filesystem runs out of space during ftruncate, ENOSPC surfaces cleanly: unlink + error + return 1. |
| T-119-12 | Information disclosure (local) | Partial output on disk between pwrite and sha3 verify | accept | The output file is writable only by the user who invoked cdb (0644, standard umask assumed). A concurrent local process that reads the output mid-write sees an inconsistent view, but local filesystem integrity is outside chromatindb's threat model. Documented here for traceability, not mitigated. |
| T-119-13 | Tampering (integrity bypass) | A CPAR manifest that claims segment_count=1 and plaintext_sha3 = SHA3(attacker_chosen_bytes) | mitigate | The manifest itself is ML-DSA-87 signed by the original uploader (same as every signed blob). An attacker who can produce a signed manifest also controls the namespace — at which point the integrity question is moot (they can trivially put any file they want). The threat is a server-side tamper, not a crypto break; addressed by the signature check in Connection's existing AEAD + signature validation pipeline. |
| T-119-14 | DoS (user disk) | Download consumes all free space before failing | accept | ftruncate up-front surfaces ENOSPC before any chunk is fetched (Focus Area 3 recommendation). User sees a clean error and no bandwidth is wasted. |
| T-119-15 | Repudiation | Mid-download cdb crash leaves a partial file with the final size (sparse where chunks weren't written) | accept | D-12 says unlink on mid-download failure, which is wired into get_chunked's fail_unlink lambda. A HARD crash (SIGKILL) skips the unlink and leaves a partial file. User detects this via a retry or re-run; the output is overwritten by --force or fails on O_EXCL. Documented. |
</threat_model>
