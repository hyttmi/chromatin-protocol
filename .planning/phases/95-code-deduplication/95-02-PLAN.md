    ```cpp
    #include "db/util/blob_helpers.h"
    ```

    **Category 1: BE shift patterns (~23 sites)**
    Replace all inline big-endian encoding/decoding patterns:

    - Inline `for (int i = 7; i >= 0; --i) buf.push_back(...)` loops -> `chromatindb::util::write_u64_be(buf, val)`
    - Inline 4-byte `buf.push_back(val >> 24)` patterns -> `chromatindb::util::write_u32_be(buf, val)`
    - Inline `(static_cast<uint64_t>(p[0]) << 56) | ...` reads -> `chromatindb::util::read_u64_be(p)` (raw pointer overload)
    - Inline `(static_cast<uint32_t>(p[0]) << 24) | ...` reads -> `chromatindb::util::read_u32_be(p)` (raw pointer overload)
    - For reads where the payload is available as a span and the offset is known, prefer the span overload with subspan: `chromatindb::util::read_u64_be(std::span{payload}.subspan(offset, 8))`
    - For writes to pre-allocated fixed buffers (uint8_t arrays), use `chromatindb::util::store_u64_be(out, val)` / `store_u32_be(out, val)`

    **Category 2: Namespace/hash extraction (~37 memcpy-32 sites)**
    Replace inline `std::memcpy(ns.data(), payload.data(), 32)` and `std::memcpy(hash.data(), payload.data() + 32, 32)` patterns:

    - Paired namespace+hash extraction (memcpy at offset 0 and offset 32): Replace both memcpy calls with `auto [ns, hash] = chromatindb::util::extract_namespace_hash(std::span{payload});` using structured bindings
    - Standalone namespace extraction (memcpy at offset 0 only): Replace with `auto ns = chromatindb::util::extract_namespace(std::span{payload});`
    - Where the existing code has already validated payload size before the memcpy (e.g., `if (payload.size() < 64) return;`), the helper's internal bounds check is redundant but harmless (per D-01, bounds checking is a feature)
    - For memcpy patterns writing INTO a result buffer (not extracting FROM a payload), use `encode_namespace_hash` instead

    **Category 3: Blob reference encoding (~8 sites)**
    Replace inline patterns that construct BlobNotify-style payloads (namespace:32 + hash:32 + seq_num:8 + size:4 + tombstone:1):
    - Replace the manual buffer construction with `chromatindb::util::encode_blob_ref(ns_span, hash_span, seq_num, size, tombstone)`
    - The 77-byte result matches the BlobNotify wire format exactly

    **IMPORTANT NOTES:**
    - Do NOT change any logic or control flow -- this is pure mechanical replacement
    - Preserve all existing log messages, error handling, and early returns
    - Some memcpy patterns may be at offsets other than 0 and 32 (e.g., extracting from the middle of a larger payload). For those, use raw memcpy or subspan the payload before calling the helper.
    - If a pattern does not cleanly map to a helper (e.g., writing a hash at a non-standard offset in a pre-allocated buffer), leave it as-is rather than forcing a bad fit
