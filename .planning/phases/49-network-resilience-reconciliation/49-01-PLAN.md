Write three integration test scripts following the established pattern (source helpers.sh, custom cleanup, FAILURES counter, pass/fail at end). All three use docker-compose.recon.yml (2-node topology).

**test_net03_large_blob_integrity.sh (NET-03):**
- Override COMPOSE to use docker-compose.recon.yml
- Start 2-node topology, wait healthy
- For each blob size (1024, 102400, 1048576, 10485760, 104857600 bytes):
  - Ingest 1 blob of that size to node1 via `run_loadgen` with `--count 1 --size SIZE --ttl 3600`
  - Wait for sync to node2 with appropriate timeout (scale timeout with size: 30s for small, 120s for 10M, 300s for 100M)
  - Verify blob count on node2 matches expected running total
  - Run `run_verify chromatindb-test-node2 hash-fields` and check that hash verification succeeds (exit code 0)
- Check for OOM kill on either container via `docker inspect --format '{{.State.OOMKilled}}'`
- IMPORTANT: The 100M blob test may take significant time. Set adequate timeouts.

**test_net04_cursor_resumption.sh (NET-04):**
- Override COMPOSE to use docker-compose.recon.yml
- Start 2-node topology, wait healthy
- Ingest 500 blobs to node1 (`--count 500 --size 256 --rate 100 --ttl 3600`)
- Wait for sync to node2 (500 blobs, 120s timeout)
- Stop node2: `docker stop chromatindb-test-node2`
- Ingest 100 more blobs to node1
- Start node2: `docker start chromatindb-test-node2`, wait healthy
- Wait for sync to node2 (600 blobs, 60s timeout)
- Verify cursor_hits > 0 via SIGUSR1 metrics: `docker kill -s USR1 chromatindb-test-node2`, parse `cursor_hits=N` from logs
- Verify blob count on node2 is >= 600
- CRITICAL: full_resync_interval must be 0 in configs (already set in Task 1) to prevent periodic full resyncs from defeating cursor detection

**test_net05_crash_recovery.sh (NET-05):**
- Override COMPOSE to use docker-compose.recon.yml
- Start 2-node topology, wait healthy
- Ingest 200 blobs to node1 (`--count 200 --size 1024 --rate 50 --ttl 3600`)
- Wait for sync to node2 (200 blobs)
- Start continuous ingest to node1 in background: `run_loadgen 172.28.0.2 --count 500 --size 1024 --rate 10 --ttl 3600 &` (slow rate so sync is actively happening)
- Wait 5 seconds for some blobs to be in-flight
- Kill node2 with SIGKILL: `docker kill --signal=KILL chromatindb-test-node2`
- Verify container exited (not OOM): `docker inspect --format '{{.State.ExitCode}}' chromatindb-test-node2` should be 137
- Wait for background loadgen to finish (or kill it)
- Get node1 blob count
- Restart node2: `docker start chromatindb-test-node2`, wait healthy
- Check logs for "integrity scan" on startup (proves clean recovery)
- Wait for sync convergence: node2 blob count should match node1
- Verify no duplicate blobs (blob count on node2 should equal node1's count, not exceed it)
