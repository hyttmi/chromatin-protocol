Update deploy/run-benchmark.sh to add two new scenario functions and a helper, then wire them into the main function. Also create the compose override file for trusted mode.

ADD TO run-benchmark.sh -- new helper function:

1. `generate_trusted_configs()` - Resolves container IPs at runtime via `docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'` for chromatindb-node1, node2, node3. Then generates 3 JSON config files in deploy/configs/:
   - node1-trusted.json: `{"bind_address":"0.0.0.0:4200","log_level":"info","sync_interval_seconds":10,"trusted_peers":["$ip2","$ip3"]}`
   - node2-trusted.json: `{"bind_address":"0.0.0.0:4200","bootstrap_peers":["node1:4200"],"log_level":"info","sync_interval_seconds":10,"trusted_peers":["$ip1","$ip3"]}`
   - node3-trusted.json: `{"bind_address":"0.0.0.0:4200","bootstrap_peers":["node2:4200"],"log_level":"info","sync_interval_seconds":10,"trusted_peers":["$ip1","$ip2"]}`
   Each node trusts the other two nodes' IPs. Use cat heredoc with variable interpolation (not sed/jq on existing configs -- research says heredoc is cleaner for JSON generation).

ADD TO run-benchmark.sh -- new scenario functions:

2. `run_scenario_latejoin()` - Implements PERF-04:
   a. reset_topology (clean 3-node chain)
   b. Load blobs to node1: `run_loadgen node1 --count $BLOB_COUNT --rate $RATE --size 1024 --drain-timeout 10`
   c. Wait for node3 convergence first (ensures full chain has data): `wait_for_convergence chromatindb-node3 $BLOB_COUNT 120`
   d. capture_stats "pre" "latejoin"
   e. Start node4: `$COMPOSE --profile latejoin up -d node4`
   f. Wait for node4 health: poll `docker inspect --format '{{.State.Health.Status}}' chromatindb-node4` until "healthy" with 30s timeout
   g. Record start time: `start_ns=$(date +%s%N)`
   h. Wait for node4 convergence: `catchup_ms=$(wait_for_convergence chromatindb-node4 $BLOB_COUNT 300)`
   i. capture_stats "post" "latejoin"
   j. Build result JSON with jq: `jq -n --arg blobs "$BLOB_COUNT" --arg catchup "$catchup_ms" '{scenario:"late-joiner",blob_count:($blobs|tonumber),catchup_ms:($catchup|tonumber),note:"Time from node4 healthy to full convergence"}'`
   k. Write to `$RESULTS_DIR/scenario-latejoin.json`
   l. Log summary

3. `run_scenario_trusted_vs_pq()` - Implements PERF-05:
   a. RUN 1 -- PQ mode (default):
      - reset_topology (clean 3-node chain, default PQ handshake)
      - capture_stats "pre" "pq"
      - `pq_result=$(run_loadgen node1 --count $BLOB_COUNT --rate $RATE --size 1024 --drain-timeout 10)`
      - capture_stats "post" "pq"

   b. GENERATE trusted configs:
      - generate_trusted_configs (creates node*-trusted.json with resolved IPs)

   c. RUN 2 -- Trusted mode:
      - Full restart with trusted configs: `$COMPOSE down -v 2>/dev/null`
      - Start with override: `docker compose -f "$COMPOSE_FILE" -f deploy/docker-compose.trusted.yml -p chromatindb up -d --wait`
      - (The override file mounts node*-trusted.json instead of node*.json)
      - capture_stats "pre" "trusted"
      - `trusted_result=$(run_loadgen node1 --count $BLOB_COUNT --rate $RATE --size 1024 --drain-timeout 10)`
      - capture_stats "post" "trusted"
      - Restore default topology: `$COMPOSE down -v 2>/dev/null` (cleanup trusted volumes)

   d. Build comparison JSON: `jq -n --argjson pq "$pq_result" --argjson trusted "$trusted_result" '{scenario:"trusted-vs-pq",pq_mode:$pq,trusted_mode:$trusted}'`
   e. Write to `$RESULTS_DIR/scenario-trusted-vs-pq.json`
   f. Log comparison summary: PQ p50/p95 vs Trusted p50/p95

UPDATE main function:
- After `run_scenario_sync`, add:
  - `run_scenario_latejoin`
  - `run_scenario_trusted_vs_pq`
- After all scenarios, add a final cleanup: `$COMPOSE down -v 2>/dev/null || true`
- Add summary: count JSON files in $RESULTS_DIR, log "Benchmark complete: N result files in $RESULTS_DIR"

CREATE deploy/docker-compose.trusted.yml as a Docker Compose override file:
```yaml
services:
  node1:
    volumes:
      - node1-data:/data
      - ./configs/node1-trusted.json:/config/node1.json:ro
  node2:
    volumes:
      - node2-data:/data
      - ./configs/node2-trusted.json:/config/node2.json:ro
  node3:
    volumes:
      - node3-data:/data
      - ./configs/node3-trusted.json:/config/node3.json:ro
```
This override replaces the config file mounts so nodes use trusted configs. The volume mounts must be fully re-specified (compose override replaces the entire volumes list for a service, it does not merge). Keep the named data volumes so libmdbx works.

IMPORTANT notes:
- For PERF-05, a FULL restart (compose down -v + up) is required between PQ and trusted runs. SIGHUP can reload trusted_peers at runtime, but existing connections keep their handshake type. Fresh handshakes require fresh connections, which means fresh containers.
- trusted_peers requires IP addresses, NOT Docker DNS names. Must resolve via docker inspect at runtime.
- The compose override mounts the trusted config at the SAME path inside the container (/config/nodeN.json) as the original -- the command line does not change.
- Cleanup generated trusted config files at script end or leave them (they are in deploy/configs/ so add a comment that they can be cleaned up manually).
