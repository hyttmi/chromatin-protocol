## Common Pitfalls

### Pitfall 1: Forgetting Dockerfile cleanup after bench removal
**What goes wrong:** Docker build fails because `COPY bench/ bench/` references a deleted directory
**Why it happens:** CMake changes are obvious; Dockerfile changes are easy to forget
**How to avoid:** Bench removal and Dockerfile cleanup MUST be in the same commit
**Warning signs:** Docker build failure

### Pitfall 2: Message type count drift
**What goes wrong:** README says "24 message types" but protocol has 27
**Why it happens:** New message types added in Phases 25 (TrustedHello, PQRequired) and 35 (QuotaExceeded) without updating docs
**How to avoid:** Always verify count against transport.fbs enum, never hardcode
**Warning signs:** grep for "message types" in all .md files, compare against transport.fbs count

### Pitfall 3: Inconsistent README updates
**What goes wrong:** db/README.md is updated but top-level README.md diverges
**Why it happens:** Top-level README is easy to forget since it's mostly a pointer
**How to avoid:** Top-level README.md is intentionally a thin pointer to db/README.md -- verify it still links correctly after changes
**Warning signs:** Any content duplication between the two READMEs

### Pitfall 4: Accidentally gitignoring committed configs
**What goes wrong:** Adding broad gitignore patterns that match committed configs
**Why it happens:** `deploy/configs/*.json` would match both committed and generated configs
**How to avoid:** Use specific pattern `deploy/configs/*-trusted.json` to only ignore generated ones
**Warning signs:** `git status` shows committed files as modified after gitignore change

### v0.7.0 Config Fields (from db/config/config.h)

| Field | Type | Default | SIGHUP Reloadable | Purpose |
|-------|------|---------|-------------------|---------|
| `full_resync_interval` | uint32_t | 10 | Yes | Full hash-diff resync every Nth sync round |
| `cursor_stale_seconds` | uint64_t | 3600 | Yes | Force full resync after this time gap (seconds) |
| `namespace_quota_bytes` | uint64_t | 0 | Yes | Global default byte limit per namespace (0 = unlimited) |
| `namespace_quota_count` | uint64_t | 0 | Yes | Global default blob count limit per namespace (0 = unlimited) |
| `namespace_quotas` | map | {} | Yes | Per-namespace overrides: `{"hex_ns": {"max_bytes": N, "max_count": N}}` |

### v0.7.0 Feature Additions for README

1. **Sync Resumption** -- Per-peer per-namespace sequence number cursors skip unchanged namespaces during sync. Only new blobs are transferred (O(new) instead of O(total)). Cursors persist across restarts. Periodic full resync as safety net.
2. **Namespace Quotas** -- Per-namespace byte and blob count limits enforced at ingest. Quota-exceeded rejection signaled to peers via QuotaExceeded message. Per-namespace overrides allow different limits for different namespaces.
