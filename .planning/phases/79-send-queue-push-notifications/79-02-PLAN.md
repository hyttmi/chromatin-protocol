The send queue is the safety foundation for all subsequent push notification work.
Output: Modified Connection class with queue-based send_message(), drain coroutine, and bounded queue with disconnect policy.

```cpp
asio::awaitable<bool> Connection::enqueue_send(std::vector<uint8_t> encoded) {
    if (closed_ || closing_) co_return false;
    if (send_queue_.size() >= MAX_SEND_QUEUE) {
        spdlog::warn("send queue full ({} messages), disconnecting {}",
                     MAX_SEND_QUEUE, remote_addr_);
        close();
        co_return false;
    }
    bool result = false;
    asio::steady_timer completion(socket_.get_executor());
    completion.expires_after(std::chrono::hours(24));

    send_queue_.push_back({std::move(encoded), &completion, &result});
    send_signal_.cancel();  // Wake drain coroutine

    auto [ec] = co_await completion.async_wait(use_nothrow);
    // ec == operation_aborted means drain coroutine processed our message
    co_return result;
}
```

3. **Implement `drain_send_queue()`** -- the single writer coroutine:
```cpp
asio::awaitable<void> Connection::drain_send_queue() {
    auto self = shared_from_this();
    drain_running_ = true;

    while (!closed_) {
        while (!send_queue_.empty() && !closed_) {
            auto msg = std::move(send_queue_.front());
            send_queue_.pop_front();

            bool ok = co_await send_encrypted(msg.encoded);

            // Signal the waiting caller
            if (msg.result_ptr) *msg.result_ptr = ok;
            if (msg.completion) {
                auto ec = asio::error::operation_aborted;
                msg.completion->cancel();
            }

            if (!ok) {
                // Write failed -- drain remaining messages as failures
                while (!send_queue_.empty()) {
                    auto& m = send_queue_.front();
                    if (m.result_ptr) *m.result_ptr = false;
                    if (m.completion) m.completion->cancel();
                    send_queue_.pop_front();
                }
                break;
            }
        }

        if (closed_) break;

        // Wait for new messages (timer-cancel wakeup pattern)
        send_signal_.expires_after(std::chrono::hours(24));
        auto [ec] = co_await send_signal_.async_wait(use_nothrow);
        // ec == operation_aborted means new message was enqueued (cancel woke us)
    }

    // Drain remaining on close: signal all waiters as failed
    while (!send_queue_.empty()) {
        auto& m = send_queue_.front();
        if (m.result_ptr) *m.result_ptr = false;
        if (m.completion) m.completion->cancel();
        send_queue_.pop_front();
    }

    drain_running_ = false;
}
```

4. **Modify `run()`** to start drain coroutine alongside message_loop using awaitable_operators:
```cpp
asio::awaitable<bool> Connection::run() {
    auto self = shared_from_this();
    bool hs_ok = co_await do_handshake();
    if (!hs_ok) {
        close();
        if (close_cb_) close_cb_(self, false);
        co_return false;
    }
    if (ready_cb_) ready_cb_(self);

    using namespace asio::experimental::awaitable_operators;
    co_await (message_loop() && drain_send_queue());

    if (close_cb_) close_cb_(self, received_goodbye_);
    co_return true;
}
```
Note: `&&` in awaitable_operators waits for BOTH to complete. When message_loop exits (socket closed/error), the drain coroutine also exits because `closed_` is set and `send_signal_` is cancelled.

7. **Modify `close()`** to cancel the send_signal_ timer so drain coroutine exits:
Add `send_signal_.cancel();` to the `close()` method, before or alongside the existing socket cancel/close. This wakes the drain coroutine which checks `closed_` and exits.

**CRITICAL -- Pitfall 2 (dangling pointer):** When `close()` is called, the drain coroutine's cleanup loop must NOT write through `result_ptr`/`completion` if the corresponding `enqueue_send` coroutine may have already been destroyed. This is safe in our design because: (a) `close()` sets `closed_ = true`, then cancels `send_signal_`; (b) the drain loop's cleanup signals waiters by cancelling their timers; (c) all coroutines run on the same io_context strand, so there's no race between close() setting closed_ and drain accessing the queue. The `completion` timer cancel in the cleanup unblocks the enqueue_send waiter, which then reads `result = false` and returns.

**CRITICAL -- Handshake bypass:** Handshake runs BEFORE `run()` calls `drain_send_queue()`. Handshake uses `send_raw()` (unencrypted) and direct `send_encrypted()` for auth. This is safe because no concurrent sends happen during handshake. No handshake code changes needed.
