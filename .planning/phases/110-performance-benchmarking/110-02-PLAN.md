    - .planning/phases/110-performance-benchmarking/110-01-SUMMARY.md (any issues or changes from Plan 01)

  <action>
Install httpx if not already installed:
```
pip install httpx
```

Run the benchmark tool with default settings (per D-04, D-05, D-08, D-15):
```
python3 tools/relay_benchmark.py --skip-uds --output tools/benchmark_report.md
```

This will:
1. Generate an ephemeral ML-DSA-87 keypair
2. Run PERF-01: throughput at 1, 10, 100 concurrent clients (1 KB blobs)
3. Run PERF-02: relay-only latency for write/read/exists/stats (UDS deferred)
4. Run PERF-03: large blob write+read at 1, 10, 50, 100 MiB
5. Run PERF-04: mixed workload with concurrent small queries + large blobs
6. Generate tools/benchmark_report.md

If the script fails:
- Check stderr for connection errors (relay not running?)
- Check for auth errors (allowed_client_keys might need to be empty array or include the ephemeral key)
- If 100 MiB blob times out, increase request_timeout_seconds in relay config
- If the script has bugs, fix them in tools/relay_benchmark.py and re-run

After successful run, verify the report:
- tools/benchmark_report.md must exist
- Must have PERF-01 throughput table (blobs/sec at 1, 10, 100 clients)
- Must have PERF-02 latency table (p50/p95/p99 for each operation)
- Must have PERF-03 large blob table (MiB/sec write and read at each size)
- Must have PERF-04 mixed workload table (latency with/without large blob load)
- Must have hardware info and test date
  </action>

    - Contains hardware info and test date
