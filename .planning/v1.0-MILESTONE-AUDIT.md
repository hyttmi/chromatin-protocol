## Critical Integration Issues

### P0: Sync Receive Side Missing (SYNC-01, SYNC-02, SYNC-03)

`PeerManager::run_sync_with_peer()` and `handle_sync_as_responder()` implement only the **send side** of the sync protocol:
- Sends `TransportMsgType_NamespaceList` ✓
- Sends `TransportMsgType_HashList` ✓
- Receives and parses peer's NamespaceList ✗
- Receives and parses peer's HashList ✗
- Computes diff via `SyncProtocol::diff_hashes()` ✗
- Sends `TransportMsgType_BlobRequest` ✗
- Handles `TransportMsgType_BlobTransfer` ✗
- Calls `SyncProtocol::ingest_blobs()` ✗

All building blocks exist in `sync_protocol.cpp`. The orchestration glue in `peer_manager.cpp` is missing.

### P1: Peer Discovery Missing (DISC-02)

No peer exchange mechanism. Nodes connect to bootstrap peers from config but never discover additional peers.

### P2: Handshake API Orphaned (TRNS-03 — working but via duplicate code)

`connection.cpp` re-implements auth inline rather than calling `HandshakeInitiator::create_auth_message()` / `verify_peer_auth()`. Dead `HandshakeInitiator hs2` at line 153.
